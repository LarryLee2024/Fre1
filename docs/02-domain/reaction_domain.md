## 1. 统一术语

| 术语 | 定义 | 职责边界 |
|------|------|----------|
| Reaction | 反应，单位在回合外进行的特殊行动，每回合仅限一次 | 负责：反应时机的检测与执行管理；不负责：反应的具体效果 |
| ReactionState | 反应状态组件，标记单位在当前回合是否已使用反应 | 负责：反应槽位的管理（已用/未用）；不负责：反应的内容 |
| ReactionQueue | 反应队列，当多个反应同时满足触发条件时，按优先级排队执行 | 负责：反应触发的排队与顺序控制；不负责：反应的执行逻辑 |
| OpportunityAttack | 机会攻击/借机攻击，当敌方单位离开己方威胁范围时触发的近战攻击 | 负责：机会攻击的触发检测与执行；不负责：攻击的伤害计算（归 Combat 领域） |
| Counterspell | 法术反制，以反应消耗法术位打断对方施法 | 负责：法术反制的触发与环阶判定；不负责：反制后的处理 |
| ShieldReaction | 护盾术反应，以反应施放护盾术临时提升 AC | 负责：护盾术的触发时机与 AC 加成；不负责：护盾术的持续时间管理 |
| GuardianReaction | 援护格挡/守护，替相邻友方单位承受攻击 | 负责：援护的触发与转移伤害；不负责：援护者的减伤计算 |

### 反应类型一览

```
反应类型                           触发时机                     每回合限制   占用反应槽
 ├── 机会攻击（Opportunity Attack）   敌方离开威胁区                 1次         是
 ├── 法术反制（Counterspell）         敌方施法时                     1次         是
 ├── 护盾术（Shield）                 被攻击命中前                    1次         是
 ├── 援护格挡（Guardian）             相邻友方被攻击时                 1次         是
 ├── 直觉法术（Reaction Spell）       多种触发条件                    1次         是
 └── 特殊反应（Special Reaction）     由 Domain 注册的自定义反应       按配置        按配置
```

### 反应触发优先级

```
当多个反应同时满足触发条件时（如：敌方离开威胁区，多个单位都满足机会攻击条件）：
1. 防御型反应（护盾术/援护格挡）> 进攻型反应（机会攻击/法术反制）
2. 同类型反应：按先攻值从高到低
3. 先攻相同：按敏捷从高到低
4. 仍然相同：掷骰决定
```

### 已对齐项目术语

- **Combat**：反应在 Combat 的回合流程中插入执行（回合外触发）
- **Spell**：法术反制和护盾术涉及法术位的消耗
- **Tactical**：机会攻击依赖 Tactical 的威胁区/移动判定
- **Ability**：反应本质是一种特殊的 Ability（触发条件 = Reaction 触发时机）

## 2. 反应状态机

### 反应槽位状态

```
Available（可用——反应槽位已就绪）
   │  [触发条件满足]
   ▼
Triggered（已触发——正在执行反应）
   │  [选择使用/不使用]
   │
   ├──→ [使用反应] → 执行反应内容 → 消耗反应槽位
   │
   └──→ [不使用反应] → 反应槽位保持可用
           │
           ▼
      Available（回到可用）
```

### 机会攻击触发流程

```
[单位尝试离开威胁区]
    │
    ├──→ 检测所有对该单位有威胁的敌方单位
    │
    ▼
[威胁单位反应检测]
    │
    ├──→ 反应槽位可用？
    │      ├──→ 是 → 触发机会攻击 → 近战攻击一次
    │      └──→ 否 → 无法触发
    │
    ▼
[机会攻击执行]
    │
    ├──→ 攻击命中？
    │      ├──→ 命中 → 造成伤害 → 目标速度变为 0（移动停止）
    │      └──→ 未命中 → 目标继续移动
    │
    ▼
[反应槽位消耗]
    │
    └──→ 标记反应为已使用（直到下回合开始重置）
```

## 3. 不变量（Invariants）

### 3.1 每回合反应次数上限
- **条件**：任何回合中
- **不变量**：每个单位每回合最多只能使用 1 次反应（除非有特殊能力/专长允许额外反应）
- **违反后果**：单回合多次反应破坏 D&D 5e 行动经济体系

### 3.2 反应不可在己方回合触发（除非特殊规则）
- **条件**：单位的己方回合行动中
- **不变量**：反应默认在回合外触发（非己方回合或己方回合的特定时机如"被攻击时"）
- **违反后果**：己方回合中多次触发反应破坏回合秩序

### 3.3 机会攻击触发条件——离开威胁区
- **条件**：机会攻击触发检测时
- **不变量**：机会攻击只在目标单位主动离开己方威胁区时触发（不因被推开/传送/强制移动触发）
- **违反后果**：强制移动触发机会攻击，规则理解困难

### 3.4 法术反制的环阶匹配
- **条件**：法术反制生效时
- **不变量**：反制法术后，目标法术的环级 > 反制法术的环级时，需要施法属性检定（DC = 10 + 目标环级 - 反制环级）
- **违反后果**：低环反制高环法术必定成功，破坏法术对抗规则

### 3.5 援护距离限制
- **条件**：援护格挡触发时
- **不变量**：援护者必须在被援护目标的相邻格内（四角网格四连通，六角网格六连通）
- **违反后果**：远距离援护使战术定位失去意义

## 4. 禁止事项（Forbidden）

- 🟥 禁止：反应在己方回合主动触发（除非法术/能力声明"可在己方回合以反应施放"） — 理由：反应是回合外机制，己方回合有标准动作和附赠动作
- 🟥 禁止：机会攻击消耗单位的反应后，单位在本回合还能再触发其他反应 — 理由：反应槽位每回合只有 1 个
- 🟥 禁止：法术反制在非施法动作时触发 — 理由：法术反制只针对施法行为
- 🟥 禁止：援护者替目标承受伤害时绕过减伤/抗性 — 理由：援护者承受的伤害应同样经过其自身的减伤/抗性计算

## 5. 流程定义

### 5.1 反应触发与队列管理

- **输入**：反应触发事件（如"单位移动出威胁区"、"施法开始"、"攻击命中前"）
- **处理**：
  1. 检测所有满足触发条件的单位
  2. 检查每个候选单位的反应槽位状态
  3. 如果多个反应同时触发，按优先级排序插入 ReactionQueue
  4. 按队列顺序询问是否使用反应（玩家控制时弹出选择界面，AI 控制时自动决策）
  5. 使用反应 → 执行反应内容 → 消耗反应槽位
  6. 不使用 → 从队列移除，反应槽位保持可用
- **输出**：ReactionTriggered 事件（触发者、反应类型、目标）
- **失败处理**：队列中某一反应的执行失败不影响队列中其他反应

### 5.2 机会攻击

- **输入**：单位主动离开威胁区的移动意图
- **处理**：
  1. 在移动意图生效前插入机会攻击检查点
  2. 检测离开单位的当前位置 —— 是否在某个敌方的威胁区内
  3. 对每个有威胁的敌方，检查其反应槽位
  4. 触发机会攻击：
     a. 攻击者发动一次近战攻击
     b. 命中判定（走 Combat 伤害结算流程）
     c. 如果命中，目标速度在当前移动中变为 0（停在当前格）
  5. 消耗攻击者的反应槽位
- **输出**：攻击结果（命中/未命中、伤害值），OpportunityAttackExecuted 事件
- **失败处理**：多个威胁单位都触发机会攻击时，先攻高者先执行；第一个命中阻止移动后，后续机会攻击取消

### 5.3 法术反制

- **输入**：敌方施法事件触发
- **处理**：
  1. 检查候选反制者的反应槽位
  2. 候选反制者选择是否使用法术反制（消耗反应+法术位）
  3. 确定反制环级：默认使用与目标法术同环的法术位，可升环
  4. 环级判定：
     a. 反制环级 >= 目标法术环级 → 自动成功
     b. 反制环级 < 目标法术环级 → 施法属性检定 DC = 10 + 目标环级
  5. 反制成功 → 目标法术被中断，不消耗目标法术位
  6. 反制失败 → 目标法术正常施放
  7. 消耗反制者的反应槽位和法术位
- **输出**：CounterspellResult（成功/失败/环级判定），CounterspellExecuted 事件
- **失败处理**：反制者法术位不足时无法反制

### 5.4 护盾术反应

- **输入**：单位被攻击命中前（命中判定完成后、伤害计算前）
- **处理**：
  1. 检查被攻击者是否有护盾术已准备且反应槽位可用
  2. 被攻击者选择是否施放护盾术（消耗反应+1 环法术位）
  3. 护盾术效果：AC +5 直到下回合开始
  4. 重新计算命中判定（以 +5 AC 后的值为准）
  5. 如果原命中变为未命中，攻击无效
  6. 消耗反应槽位和 1 环法术位
- **输出**：ShieldResult（是否生效、新命中判定），ShieldUsed 事件
- **失败处理**：没有 1 环法术位时无法施放护盾术

### 5.5 援护格挡

- **输入**：相邻友方单位被攻击命中时
- **处理**：
  1. 检测所有与目标友方相邻的己方单位
  2. 检查候选援护者的反应槽位
  3. 援护者选择是否援护（消耗反应）
  4. 援护生效：援护者代替原目标承受此次攻击的全部伤害
  5. 伤害计算以援护者的属性为准（AC/减伤/抗性）
  6. 消耗援护者的反应槽位
- **输出**：GuardianResult（援护者、转移的伤害），GuardianUsed 事件
- **失败处理**：援护者不在相邻格时无法触发

## 6. 领域事件

| 事件名 | 触发时机 | 携带数据 | 订阅者 |
|--------|----------|----------|--------|
| ReactionTriggered | 反应满足触发条件时 | reactor_id, reaction_type, trigger_event, context | UI（显示反应选择界面/AI 自动决策）、Combat（等待反应执行后再继续） |
| ReactionExecuted | 反应执行完毕时 | reactor_id, reaction_type, result | Combat（继续原流程）、Cue（反应特效） |
| ReactionDeclined | 单位选择不使用反应时 | reactor_id, reaction_type, reason | Combat（继续原流程，跳过此反应） |
| OpportunityAttackExecuted | 机会攻击执行完毕时 | attacker_id, target_id, hit_result, damage | Combat（更新目标位置/伤害）、Cue（攻击特效） |
| CounterspellExecuted | 法术反制执行完毕时 | caster_id, target_spell, counter_level, success | Spell（法术被中断/继续）、Cue（反制特效） |

### 事件订阅关系图

```
ReactionTriggered
    │
    ├──→ 玩家控制：弹出反应选择界面（"是否使用机会攻击？"）
    ├──→ AI 控制：自动决策（按策略规则选择使用/不使用）
    ├──→ Combat：等待反应执行或取消后再继续战斗流程
    └──→ Cue：反应触发提示特效

ReactionExecuted
    │
    ├──→ Combat：反应执行完成 → 继续原流程
    ├──→ Cue：反应特效
    └──→ 日志：记录反应使用

OpportunityAttackExecuted
    │
    ├──→ Tactical：如果命中→目标位置不变（被拦截）
    ├──→ Combat：伤害结算
    ├──→ Cue：攻击特效
    └──→ UI：显示机会攻击动画

CounterspellExecuted
    │
    ├──→ Spell：法术被反制→中断法术效果
    ├──→ Cue：法术反制特效（法术消散）
    ├──→ UI：显示"法术反制"文字提示
    └──→ 日志：记录法术反制
```

## 7. 与已有架构的对齐校验

- ✅ 架构边界：Reaction 域位于 `core/domains/reaction/`，components.rs 定义 ReactionState/ReactionQueue/OpportunityAttack/CounterSpell，systems/ 实现机会攻击/法术反制/护盾/援护/队列管理系统，rules/ 定义触发/限制/反制规则
- ✅ 反应槽机制对齐 D&D 5e：每回合 1 次反应，在回合外使用
- ✅ 反应队列解决"多人同时触发"问题：按优先级排序执行
- ✅ 机会攻击与 Combat 伤害结算复用：不另建攻击系统
- ✅ 法术反制与 Spell 领域协作：反制消耗法术位、环级判定涉及 Spell 的数据
