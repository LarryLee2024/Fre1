## 1. 统一术语

| 术语 | 定义 | 职责边界 |
|------|------|----------|
| RestState | 休息状态，标记队伍当前是否正在休息以及休息的进行阶段 | 负责：休息流程的状态管理；不负责：休息的资源恢复量 |
| ShortRest | 短休，约 1 小时的短暂休息，恢复部分资源 | 负责：短休流程的定义与执行；不负责：短休触发的剧情事件 |
| LongRest | 长休，约 8 小时的充足休息，恢复全部资源 | 负责：长休流程的定义与执行；不负责：长休触发的营地事件 |
| HitDice | 生命骰，短休时可消耗来恢复 HP 的资源 | 负责：生命骰的消耗与恢复规则；不负责：具体的恢复量计算 |
| CampNPC | 营地 NPC，在长休时出现在营地中的可交互角色 | 负责：营地 NPC 的管理；不负责：NPC 的对话内容 |
| CampEvent | 营地事件，长休时根据条件触发的剧情/随机事件 | 负责：营地事件的触发条件与执行；不负责：事件的具体剧情 |

### 休息规则（D&D 5e）

```
短休（Short Rest）：
- 时长：约 1 小时
- 效果：
  ├── 可消耗生命骰（Hit Dice）来恢复 HP
  ├── 部分能力/资源恢复（声明"短休恢复"的能力）
  └── 不恢复法术位
- 限制：每天短休次数不限，但生命骰数量有限

长休（Long Rest）：
- 时长：约 8 小时（至少 6 小时睡眠 + 2 小时轻活动）
- 效果：
  ├── 恢复全部 HP
  ├── 恢复全部法术位
  ├── 恢复全部生命骰（上限为等级数的一半）
  ├── 恢复声明"长休恢复"的能力
  └── 可能触发营地事件/剧情
- 限制：每 24 小时仅能进行 1 次长休
- 中断：长休中断（战斗/环境干扰）超过 1 小时，不获得长休效果
```

### 已对齐项目术语

- **Combat**：战斗结束后可进入休息；休息被打断可能重新进入战斗
- **Spell**：长休后法术位全恢复
- **Progression**：恢复生命骰依赖角色等级
- **Party**：营地的队伍管理（换人/调整配置）
- **Narrative**：营地事件通常伴随对话和剧情选择

## 2. 休息状态机

### 短休状态

```
Idle（未休息）
   │  [选择短休]
   ▼
ShortResting（短休中）
   │  [消耗生命骰恢复 HP]
   │  [恢复短休能力]
   │  [1 小时（可加速/跳过）]
   ▼
ShortRestComplete（短休完成）
```

### 长休状态

```
Idle（未休息）
   │  [选择长休]
   ▼
LongResting（长休中）
   │  [6 小时睡眠中]
   │      │
   │      ├──→ [睡眠被中断] → 中断计时开始
   │      │       │
   │      │       ├──→ [中断持续 < 1 小时] → 继续睡眠
   │      │       │
   │      │       └──→ [中断持续 >= 1 小时] → 长休失败
   │      │
   │      └──→ [6 小时睡眠完成]
   │
   ├──→ [2 小时轻活动（可选：触发营地事件/NPC 对话/调整队伍)]
   │
   ▼
LongRestComplete（长休完成）
   │
   ├──→ HP 全恢复
   ├──→ 法术位全恢复
   ├──→ 生命骰恢复（上限为等级/2）
   ├──→ 能力恢复
   └──→ 营地事件触发
```

## 3. 不变量（Invariants）

### 3.1 长休频率限制
- **条件**：任何长休开始时
- **不变量**：每 24 小时最多只能进行 1 次长休
- **违反后果类型**：🔴 规则失败
- **违反后果**：过于频繁的长休使游戏失去资源管理策略

### 3.2 短休安全要求
- **条件**：短休开始时
- **不变量**：短休需要在安全区域或非战斗状态下进行
- **违反后果类型**：🔴 规则失败
- **违反后果**：战斗中的短休违反 D&D 5e 规则

### 3.3 长休安全要求
- **条件**：长休开始时
- **不变量**：长休需要在安全区域进行（有营地/安全的避难所）
- **违反后果类型**：🔴 规则失败
- **违反后果**：在危险区域的"长休"无法获得充足休息效果

### 3.4 生命骰上限
- **条件**：长休恢复生命骰时
- **不变量**：生命骰的恢复上限为角色等级的一半（向下取整），已有生命骰数不得超过总等级数
- **违反后果类型**：🔴 规则失败
- **违反后果**：生命骰无限积累破坏恢复资源平衡

### 3.5 长休中断可导致休息失败
- **条件**：长休期间遭遇战斗/干扰时
- **不变量**：如果长休中断累计超过 1 小时，本次长休不获得任何效果，不消耗长休次数
- **违反后果类型**：🔴 程序错误
- **违反后果**：中断后仍获得长休效果

## 4. 禁止事项（Forbidden）

- 🟥 禁止：战斗中进行短休/长休 — 理由：休息必须在非战斗状态进行
- 🟥 禁止：长休后资源未完全恢复（除特殊模式如"硬核模式"外） — 理由：D&D 5e 标准规则下长休全恢复
- 🟥 禁止：一天多次长休 — 理由：24 小时 1 次长休的频率限制
- 🟥 禁止：生命骰自然恢复（除长休外） — 理由：生命骰只在长休时恢复（上限 = 等级/2）

## 5. 流程定义

### 5.1 短休

- **输入**：休息请求（队伍/个人）、当前状态（必须在非战斗状态）
- **处理**：
  1. 检查是否在安全区域/非战斗状态（不变量 3.2）
  2. 如有必要，消耗生命骰：
     a. 玩家选择消耗几个生命骰
     b. 每个生命骰恢复量 = 等级对应的 Hit Dice + 体质调整值
     c. HP += 恢复量
  3. 恢复所有标记为"短休恢复"的能力
  4. 发布 ShortRestCompleted 事件
- **输出**：ShortRestCompleted 事件（消耗的生命骰数、HP 恢复量、恢复的能力列表）
- **失败处理**：非安全区域时短休失败 → 这是**规则失败**（预期业务分支，短休需要安全环境）

### 5.2 长休

- **输入**：休息请求（必须非战斗状态）、当前场景（是否有营地）
- **处理**：
  1. 检查 24 小时内是否已进行过长休（不变量 3.1）
  2. 检查是否在安全区域（不变量 3.3）
  3. 进入长休状态（LongResting）
  4. 睡眠阶段（6 小时）：
     a. 如果遭遇战斗/干扰，启动中断计时
     b. 中断小于 1 小时 → 继续睡眠
     c. 中断 >= 1 小时 → 长休失败（不变量 3.5）
  5. 轻活动阶段（2 小时）：
     a. 可触发营地事件（CampEvent）
     b. 可与 CampNPC 交互
     c. 可调整队伍配置（换人）
  6. 长休完成：
     a. HP = MaxHP（全恢复）
     b. SpellSlots = 全恢复
     c. 生命骰恢复量 = max(当前生命骰数, 总等级/2)
     d. 所有声明"长休恢复"的能力重置
  7. 发布 LongRestCompleted 事件
- **输出**：LongRestCompleted 事件（恢复量明细、触发的营地事件）
- **失败处理**：不安全区域/24 小时内已长休/睡眠中断超过 1 小时均导致长休失败 → 这是**规则失败**（预期业务分支，长休条件不满足或受到干扰）

### 5.3 营地事件触发

- **输入**：长休的轻活动阶段开始
- **处理**：
  1. 遍历所有注册的营地事件，检查触发条件（队伍状态/剧情进度/阵营声望/随机概率）
  2. 满足条件的事件排队
  3. 按优先级执行（剧情事件 > 角色事件 > 随机事件）
  4. 每个事件：
     a. 展示事件描述
     b. 如需要玩家选择，提供选项
     c. 根据选择影响后续剧情/属性/关系
- **输出**：CampEventTriggered 事件（事件 Id、类型、参与者、选择结果）
- **失败处理**：无满足条件的事件时不触发 → 这是**规则失败**（预期业务分支，非每次长休都有营地事件）

## 6. 领域事件

| 事件名 | 触发时机 | 携带数据 | 订阅者 |
|--------|----------|----------|--------|
| ShortRestCompleted | 短休完成时 | entity[ ], hit_dice_used, hp_healed, abilities_restored[ ] | Ability（重置短休能力）、UI（显示短休结果）、日志（LogCode: CNR001） |
| LongRestStarted | 长休开始时 | entity[ ], camp_location | Narrative（准备营地事件）、NPC（激活营地 NPC）、日志（LogCode: CNR002） |
| LongRestCompleted | 长休完成时 | entity[ ], hp_restored, spell_slots_restored, hit_dice_restored, events_triggered[ ] | Spell（法术位恢复）、Ability（重置所有能力）、UI（显示长休结果）、Cue（日出/恢复特效）、日志（LogCode: CNR003） |
| LongRestInterrupted | 长休被中断时 | entity[ ], interruption_source（战斗/环境）, cumulative_interrupt_time | Combat（如中断原因为战斗）、UI（显示中断警告）、日志（LogCode: CNR004） |
| CampEventTriggered | 营地事件触发时 | event_id, event_type, participants[ ], choices_available[ ] | Narrative（推动剧情）、Quest（检查任务进度）、UI（显示事件界面）、日志（LogCode: CNR005） |

### 事件订阅关系图

```
ShortRestCompleted
    │
    ├──→ Ability：重置标记为"短休恢复"的能力
    ├──→ UI：显示短休结果（消耗了多少生命骰、恢复了多少 HP）
    └──→ 日志：记录短休

LongRestCompleted
    │
    ├──→ Spell：恢复全部法术位
    ├──→ Ability：重置所有能力
    ├──→ Effect：移除所有应被长休清除的临时效果
    ├──→ Inventory：无特殊动作（装备不会被长休卸下）
    ├──→ Progression：确认长休不影响经验/等级
    ├──→ UI：显示长休结果界面
    ├──→ Cue：日出/休息完成特效
    └──→ 日志：记录长休详情

LongRestInterrupted
    │
    ├──→ Combat：如被战斗中断，开始战斗
    ├──→ UI：显示中断告警
    └──→ 日志：记录中断
```

## 7. 与已有架构的对齐校验

- ✅ 架构边界：CampRest 域位于 `core/domains/camp_rest/`，components.rs 定义 RestState/CampNPC/CampEvent，systems/ 实现休息/恢复/营地 NPC/营地事件系统，rules/ 定义休息/恢复/营地规则
- ✅ 短休/长休规则对齐 D&D 5e：短休 1h+生命骰恢复，长休 8h+全恢复（每 24h 1 次）
- ✅ 长休中断规则：中断累积超 1h 不获得效果
- ✅ 营地事件作为叙事入口：CampEvent 触发后推动 Narrative 领域的对话/剧情
