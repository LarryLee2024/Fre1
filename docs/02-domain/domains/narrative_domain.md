## 1. 统一术语

| 术语 | 定义 | 职责边界 |
|------|------|----------|
| DialogueState | 对话状态，管理当前对话的节点位置和已选择的分支 | 负责：对话流程的控制（当前节点/可选分支/已选分支）；不负责：对话节点的内容 |
| DialogueNode | 对话节点，是对话树的基本单位，包含 NPC 文本和玩家可选的分支 | 负责：对话内容与分支选项的结构化定义；不负责：分支选项的条件判断 |
| StoryFlag | 故事标记，记录玩家在剧情中做出的关键选择或达成的状态 | 负责：二值/枚举状态的记录（如"救了村民"=true）；不负责：状态变化的业务影响 |
| CutsceneState | 演出状态，控制过场动画/镜头/音效的播放 | 负责：演出的阶段管理（播放中/暂停/完成）；不负责：演出的具体内容（资产/脚本由 Infra 层管理） |
| ChoiceCondition | 分支选项的条件判定，决定对话选项是否可见/可用 | 负责：对话分支的条件过滤；不负责：条件的结果处理 |
| DialogueHistory | 对话历史，记录玩家已看过的对话和选择，支持对话回溯/跳过 | 负责：已阅对话的记录；不负责：对话的交互流程 |

### 对话树结构

```
DialogueNode（入口节点）
    │  [NPC 说话："需要帮助吗？"]
    │
    ├── Choice A: "告诉我更多" ──→ DialogueNode A
    │    │                            [NPC 说话："事情是这样的..."]
    │    │                            ├── Choice A1: "我帮你" ──→ Node A1
    │    │                            └── Choice A2: "我没兴趣" ──→ Node A2
    │    │
    ├── Choice B: "我路过而已" ──→ DialogueNode B
    │    │                            [对话结束]
    │    │
    └── Choice C: "需要条件可见" ──→ 基于 StoryFlag/Faction/Quest 的条件判定
         │  ┌──────────────────────────────────────────────────┐
         │  │ 条件：Tag(Quest."帮忙任务") == Completed          │
         │  │ 可见：仅当条件满足时此选项才出现                    │
         │  └──────────────────────────────────────────────────┘
         │
         └──→ DialogueNode C（条件满足时才可到达）
```

### 已对齐项目术语

- **Faction**：对话分支的可见性可能依赖 Faction 的声望等级
- **Quest**：对话分支可能接受/推进/完成 Quest
- **Condition**：ChoiceCondition 使用 Condition 领域检查对话条件的满足情况
- **Inventory**：对话选项可能消耗/给予物品
- **Progression**：对话选项可能给予经验奖励

## 2. 对话状态机

```
NotStarted（未开始）
   │  [触发对话（与 NPC 交互/自动触发）]
   ▼
Speaking（对话中）
   │  [显示当前节点 NPC 文本]
   │  [展示可选分支]
   │      │
   │      ├──→ [玩家选择分支]
   │      │      │
   │      │      ├──→ [分支对应新节点] → 继续 Speaking
   │      │      │
   │      │      └──→ [分支对应结束对话] → 进入 End
   │      │
   │      └──→ [玩家跳过/加速]
   │
   ▼
End（对话结束——记录选择，触发事件）
```

## 3. 不变量（Invariants）

### 3.1 对话树无环
- **条件**：对话树定义时
- **不变量**：对话树的节点之间不得形成循环引用（玩家无法通过对话选项回到已经过的节点）
- **违反后果类型**：🔴 程序错误
- **违反后果**：对话陷入死循环，玩家永远无法退出

### 3.2 分支条件可预测
- **条件**：对话分支过滤时
- **不变量**：相同状态下，同一对话分支的条件判定结果必须一致（确定性）
- **违反后果类型**：🔴 程序错误
- **违反后果**：同一存档加载后对话选项不同，玩家困惑

### 3.3 故事标记只增不减
- **条件**：任何 StoryFlag 设置时
- **不变量**：StoryFlag 一旦设置为 true/某个值，不可被恢复为 false/其他值（仅可通过新剧情线覆盖）
- **违反后果类型**：🔴 规则失败
- **违反后果**：关键剧情选择可被撤销，叙事一致性被破坏

### 3.4 对话不可跳过（首次/关键对话）
- **条件**：关键剧情对话/首次对话时
- **不变量**：标记为"重要"的对话不可被跳过，玩家必须阅读/选择
- **违反后果类型**：🔴 规则失败
- **违反后果**：玩家跳过了理解后续剧情所必需的信息

### 3.5 多分支互斥性
- **条件**：对话分支展示时
- **不变量**：同一节点的多个分支选项中，互斥条件的分支只能出现一个（如"救 A"和"救 B"不同时出现）
- **违反后果类型**：🔴 程序错误
- **违反后果**：玩家看到两个互斥条件选项，选择逻辑混乱

## 4. 禁止事项（Forbidden）

- 🟥 禁止：StoryFlag 被多次设置后恢复到初始值 — 理由：剧情选择是有重量的，不可逆
- 🟥 禁止：对话树运行时动态修改 — 理由：对话树应在内容加载阶段确定，运行时不修改结构
- 🟥 禁止：对话选项影响非对话系统（直接修改属性/给予物品） — 理由：对话选项通过事件通知其他系统，不走 bypass
- 🟥 禁止：对话节点不存在时试图跳转 — 理由：跳转到不存在的节点导致对话卡死

## 5. 流程定义

### 5.1 对话开始

- **输入**：对话触发（与 NPC 交互/自动触发/物品触发）
- **处理**：
  1. 加载对话树定义（从 content/ 加载）
  2. 确定入口节点
  3. 检查入口节点的触发条件（如需要特定 StoryFlag）
  4. 初始化 DialogueState
  5. 展示入口节点文本
  6. 展示可选分支（过滤不可见分支）
  7. 发布 DialogueStarted 事件
- **输出**：DialogueStarted 事件
- **失败处理**：入口节点触发条件不满足时对话不可用 → 这是**规则失败**（预期业务分支，对话需要满足触发条件）

### 5.2 分支选择

- **输入**：玩家选择的分支 ID
- **处理**：
  1. 检查分支是否在可选列表中（防止玩家选择不可见分支）
  2. 记录选择到 DialogueHistory
  3. 执行分支的副作用：
     a. 设置 StoryFlag（如"选择了帮助 A"）
     b. 如果分支关联了任务，通过 Event 通知 Quest 领域
     c. 如果分支关联了声望变化，通知 Faction 领域
     d. 如果分支关联了物品变化，通知 Inventory 领域
  4. 跳转到分支指向的下一节点（如节点存在）或结束对话
  5. 发布 ChoiceMade 事件
- **输出**：ChoiceMade 事件（选择的分支、设置的 StoryFlag、触发的副作用）
- **失败处理**：分支 ID 无效时对话不跳转，停留在当前节点 → 这是**程序错误**（系统异常，UI 不应提供无效选项，分支 ID 异常应记 Bug）

### 5.3 对话结束

- **输入**：结束信号（到达结束节点/玩家主动退出/分支指向结束）
- **处理**：
  1. 记录对话完成状态到 DialogueHistory
  2. 发布 StoryFlagSet 事件（如有新设置的标记）
  3. 清理 DialogueState
- **输出**：对话结束确认
- **失败处理**：无

## 6. 领域事件

| 事件名 | 触发时机 | 携带数据 | 订阅者 |
|--------|----------|----------|--------|
| DialogueStarted | 对话开始时 | npc_id, dialogue_tree_id, entry_node_id, available_choices[ ] | UI（打开对话界面）、Quest（检查对话相关任务）、日志（LogCode: NAR001） |
| ChoiceMade | 玩家选择分支时 | entity_id, dialogue_id, choice_id, story_flags_set[ ], side_effects[ ] | Quest（接受/推进任务）、Faction（声望变化）、StoryFlag（记录选择）、UI（推进对话）、日志（LogCode: NAR002） |
| StoryFlagSet | 故事标记被设置时 | entity_id, flag_id, value, source（对话/任务/事件） | Quest（检查任务条件）、Narrative（解锁新对话分支）、UI（显示状态变化）、日志（LogCode: NAR003） |
| CutsceneStarted | 过场动画开始时 | cutscene_id, duration, participants[ ] | UI（切换播放模式/隐藏 HUD）、Cue（过场音效）、日志（LogCode: NAR004） |
| CutsceneEnded | 过场动画结束时 | cutscene_id | UI（恢复 HUD/交互模式）、日志（LogCode: NAR005） |

### 事件订阅关系图

```
DialogueStarted
    │
    ├──→ UI：打开对话界面/显示 NPC 头像和文本
    ├──→ Quest：检查是否有对话相关的任务
    └──→ 日志：记录对话开始

ChoiceMade
    │
    ├──→ StoryFlag：记录选择的标记
    ├──→ Quest：接受新任务或推进已有任务
    ├──→ Faction：更新声望
    ├──→ Inventory：给予/移除对话奖励物品
    ├──→ Progression：给予对话经验奖励
    ├──→ UI：推进对话到下一节点
    └──→ 日志：记录对话选择

StoryFlagSet
    │
    ├──→ Narrative：解锁新的对话分支/节点
    ├──→ Quest：检查任务条件的满足状态
    ├──→ UI：显示状态变化通知（如"你救了村民"）
    └──→ 日志：记录剧情标记
```

## 7. 与已有架构的对齐校验

- ✅ 架构边界：Narrative 域位于 `core/domains/narrative/`，components.rs 定义 DialogueState/StoryFlag/CutsceneState，systems/ 实现对话/分支/演出/标记系统，rules/ 定义条件/分支规则
- ✅ 对话通过 Condition 过滤分支：复用了 Capabilities 的 Condition 领域，不另建条件系统
- ✅ StoryFlag 是叙事核心：记录玩家选择（只增不减），后续对话/任务/阵营通过检查 StoryFlag 决定状态
- ✅ 副作用通过 Event 通知：对话选择不直接修改属性/物品，通过事件解耦
