## 1. 统一术语

| 术语 | 定义 | 职责边界 |
|------|------|----------|
| Experience | 经验值，通过战斗/任务/探索获得的角色成长点数 | 负责：经验的获取与累计；不负责：经验的消耗（等级提升自动消耗） |
| Level | 角色等级，反映角色的整体实力水平 | 负责：等级的定义与管理；不负责：等级带来的具体能力提升 |
| ClassLevel | 职业等级映射，记录角色在各职业上的等级分布（多职） | 负责：多职业等级的分别管理；不负责：职业特性的解锁 |
| TalentTree | 天赋树，记录角色在各个天赋上的解锁状态 | 负责：天赋的解锁状态管理；不负责：天赋的具体效果 |
| SubclassChoice | 子职选择，记录角色在特定等级选择的子职业 | 负责：子职的选择记录；不负责：子职的特性 |
| ProficiencyBonus | 熟练加值，基于总等级的基础攻击/技能检定加值 | 负责：熟练加值的计算；不负责：熟练加值对具体技能的影响 |
| ASI（Ability Score Improvement） | 属性值提升，特定等级时角色可提升属性值 | 负责：ASI 的触发时机与规则；不负责：属性值的实际修改 |

### 等级成长体系

```
总等级 = 各职业等级之和（D&D 5e 多职规则）

等级范围：1-20 级

经验表（示例，累计 XP）：
  1→2：300 XP
  2→3：900 XP
  3→4：2,700 XP
  4→5：6,500 XP
  5→6：14,000 XP
  ...
  19→20：355,000 XP

熟练加值：
  1-4 级：+2
  5-8 级：+3
  9-12 级：+4
  13-16 级：+5
  17-20 级：+6

ASI / 专长时机：
  4 级、8 级、12 级、16 级、19 级
  每次可选择：+2 点属性值 / 或选择一个专长
```

### 多职业规则

```
多职业（Multiclassing）：
- 角色升级时可以选择提升已有职业或开始一个新职业
- 总等级 = 所有职业等级之和
- 熟练加值基于总等级计算
- 职业特性（如施法能力、额外攻击）按各职业等级分别结算
- 属性需求：开始新职业需要满足该职业的最低属性要求（如兼职圣武士需力量≥13且魅力≥13）
```

### 已对齐项目术语

- **Attribute**：Progression 通过 ASI 修改 Primary 属性的 BaseValue
- **Spec**：Progression 给技能升级（SkillSpec 等级变更）
- **Condition**：Progression 检查升级/天赋解锁的前置条件
- **Combat**：Progression 在 CombatEnded 时发放战斗经验
- **Quest**：Progression 在 QuestCompleted 时发放任务经验

## 2. 成长状态机

### 角色成长状态

```
Level N（当前等级 N）
   │  [获得经验]
   ▼
Experience Accumulated（经验累计中）
   │  [累计经验 >= 升级所需]
   │
   ├──→ [升级到 Level N+1]
   │      │
   │      ├──→ 增加属性值（如适用 ASI 等级）
   │      ├──→ 获得职业特性
   │      ├──→ 解锁天赋选项
   │      ├──→ 增加法术位（如施法职业）
   │      └──→ 更新熟练加值
   │
   │  [多职：可选择提升已有职业或开始新职业]
   │
   ▼
Level N+1（升级完成）
```

### 天赋解锁状态

```
Locked（锁定）
   │  [满足前置条件（等级/其他天赋/属性）]
   ▼
Available（可选）
   │  [选择解锁]
   ▼
Unlocked（已解锁/生效中）
```

## 3. 不变量（Invariants）

### 3.1 等级上限
- **条件**：任何角色升级时
- **不变量**：角色总等级不得超过 20
- **违反后果类型**：🔴 规则失败
- **违反后果**：超出 20 级时经验继续累计但不再升级（D&D 5e 上限规则）

### 3.2 经验不可消耗
- **条件**：任何经验变更时
- **不变量**：经验值只增不减（且仅通过升级"消耗"——即经验条扣除升级所需量）
- **违反后果类型**：🔴 程序错误
- **违反后果**：经验被扣除导致降级，违反 D&D 5e 无降级规则

### 3.3 天赋前置链完整性
- **条件**：任何天赋解锁时
- **不变量**：天赋的前置条件（等级/属性/前置天赋）必须全部满足
- **违反后果类型**：🔴 规则失败
- **违反后果**：跳跃解锁天赋导致天赋树结构被破坏

### 3.4 子职选择的唯一性
- **条件**：子职选择时
- **不变量**：同一职业只能选择一个子职，选择后不可更改
- **违反后果类型**：🔴 规则失败
- **违反后果**：同一职业选择多个子职导致特性冲突

### 3.5 ASI 时机不可跳过
- **条件**：角色到达 ASI 等级（4/8/12/16/19）时
- **不变量**：每次到达 ASI 等级时，角色必须分配属性提升或选择专长，不可跳过
- **违反后果类型**：🔴 规则失败
- **违反后果**：跳过 ASI 导致角色属性低于预期

## 4. 禁止事项（Forbidden）

- 🟥 禁止：经验值被任何系统消耗（除升级自动消耗外） — 理由：经验是成长资源，不可用于交易/制造等非成长用途
- 🟥 禁止：降级（等级逆向变化） — 理由：D&D 5e 无降级机制，等级只升不降
- 🟥 禁止：同一职业选择多个子职 — 理由：子职是职业分支，唯一选择
- 🟥 禁止：跳过前置条件解锁天赋 — 理由：天赋树的可选性依赖前置条件完整性

## 5. 流程定义

### 5.1 经验获取

- **输入**：经验来源（战斗胜利/完成任务/探索发现/对话奖励）、获得经验量
- **处理**：
  1. 如果当前等级 >= 20，不增加经验（不变量 3.1）
  2. 增加经验值
  3. 检查是否达到升级所需经验
  4. 如果达到升级经验，触发升级流程
- **输出**：ExperienceGained 事件
- **失败处理**：已满级时经验不增加（可记录溢出但不影响） → 这是**规则失败**（预期业务分支，角色已达等级上限）

### 5.2 升级

- **输入**：升级通知（经验达到升级线）
- **处理**：
  1. 扣除升级所需的经验值
  2. 总等级 +1
  3. 让玩家/系统选择：提升已有职业 or 开始新职业（多职）
  4. 根据职业：
     a. 增加职业等级
     b. 解锁该职业等级的特性
     c. 更新 HP 上限（按职业 Hit Dice 增加）
     d. 如果是施法职业：更新法术位
  5. 更新熟练加值
  6. 检查是否为 ASI 等级 → 触发 ASI 流程
  7. 检查是否有可解锁的天赋 → 通知天赋选择
  8. 发布 LevelUp 事件
- **输出**：LevelUp 事件（新等级、获得的特性、属性变化摘要）
- **失败处理**：升级所需经验不足时拒绝升级 → 这是**规则失败**（预期业务分支，经验未达升级阈值）

### 5.3 属性提升（ASI）

- **输入**：ASI 触发通知
- **处理**：
  1. 展示可提升的属性列表
  2. 玩家选择：提升 2 点单一属性 / 提升 1+1 点两项属性 / 选择一个专长
  3. 如果是属性提升，修改对应 Attribute 的 BaseValue
  4. 如果是专长，注册专长相关的 Ability/Spec
  5. 确保所有属性不超过 20（D&D 5e 属性上限）
  6. 发布 ASICompleted 事件
- **输出**：ASICompleted 事件（选择结果）
- **失败处理**：属性值已到 20 上限时不能继续提升 → 这是**规则失败**（预期业务分支，属性已达 D&D 5e 上限）

### 5.4 天赋解锁

- **输入**：天赋解锁请求（玩家选择/系统自动解锁）
- **处理**：
  1. 检查前置条件（等级/前置天赋/属性要求）
  2. 如果满足前置条件，解锁天赋
  3. 应用天赋效果（通常为被动 Modifier 或新 Ability）
  4. 发布 TalentUnlocked 事件
- **输出**：TalentUnlocked 事件
- **失败处理**：前置条件不满足时解锁失败 → 这是**规则失败**（预期业务分支，天赋前置条件未满足）

## 6. 领域事件

| 事件名 | 触发时机 | 携带数据 | 订阅者 |
|--------|----------|----------|--------|
| ExperienceGained | 角色获得经验时 | entity_id, amount, source, total_xp | UI（显示经验获取动画）、Progression（检查是否可升级）、日志（LogCode: PRG001） |
| LevelUp | 角色升级时 | entity_id, old_level, new_level, class_gained, features[ ] | Attribute（升级带来的属性变化）、UI（升级动画/选择界面）、日志（LogCode: PRG002） |
| TalentUnlocked | 天赋解锁时 | entity_id, talent_id, tier | Ability（注册天赋相关能力）、UI（天赋树更新）、日志（LogCode: PRG003） |
| SubclassChosen | 子职选择时 | entity_id, class_id, subclass_id | Spell（更新法术列表）、Ability（注册子职能力）、日志（LogCode: PRG004） |
| ASICompleted | 属性提升完成时 | entity_id, level, choices（提高的属性/选取的专长） | Attribute（修改 BaseValue）、UI（属性变化显示）、日志（LogCode: PRG005） |
| ClassGained | 获得新职业等级时 | entity_id, class_id, new_level, features | Spell（更新法术位/已知法术）、日志（LogCode: PRG006） |

### 事件订阅关系图

```
ExperienceGained
    │
    ├──→ UI：显示经验获取动画/进度条
    ├──→ Progression：检查经验阈值 → 触发升级
    └──→ 日志：记录经验获取来源

LevelUp
    │
    ├──→ Attribute：应用升级带来的属性变化
    ├──→ Spell：更新法术位/已知法术上限
    ├──→ Spec：更新技能等级
    ├──→ Ability：注册升级解锁的新能力
    ├──→ UI：显示升级选择界面（职业/天赋/ASI）
    │
    ├──→ Condition：重新检查新解锁的能力/装备条件
    ├──→ Quest：检查等级相关任务条件
    ├──→ Cue：升级特效/音效
    └──→ 日志：记录升级

TalentUnlocked
    │
    ├──→ Modifier：应用天赋的被动效果
    ├──→ Ability：注册天赋提供的主动能力
    ├──→ UI：更新天赋树显示
    └──→ Cue：天赋解锁特效

ASICompleted
    │
    ├──→ Attribute：修改对应属性 BaseValue
    ├──→ Aggregator：触发属性重算（派生属性受影响）
    ├──→ UI：显示属性变化
    └──→ 日志：记录 ASI 选择
```

## 7. 与已有架构的对齐校验

- ✅ 架构边界：Progression 域位于 `core/domains/progression/`，components.rs 定义 Level/Experience/ClassLevels/TalentTree/SubclassChoice，systems/ 实现经验/职业/天赋/子职/ASI 系统，rules/ 定义等级表/多职/ASI/熟练规则
- ✅ 多职业规则：ClassLevels 组件分别记录各职业等级，熟练加值按总等级计算
- ✅ 属性提升通过 Modifier 管线：ASI 修改 BaseValue 后经 Aggregator 重算，不 bypass
- ✅ 天赋通过 Condition 做前置检查，解锁后注册 Ability/Modifier
