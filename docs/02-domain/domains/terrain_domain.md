## 1. 统一术语

| 术语 | 定义 | 职责边界 |
|------|------|----------|
| Tile | 战场地图的最小格子单元，携带地形属性 | 负责：格子的地形类型/高度/通行性/遮蔽度定义，Terrain 的 LocalizationKey（name_key/desc_key）；不负责：格子上的单位管理 |
| TileProperties | 格子的地形属性集合，描述该格子的地形特征 | 负责：地形类型及其属性的定义；不负责：地形上的动态效果 |
| TerrainEffect | 地形效果，单位进入特定地形格子时被施加的效果 | 负责：地形与游戏效果的映射关系；不负责：效果的生命周期管理 |
| HazardZone | 危险区域/陷阱，定义特定区域的触发条件和伤害/效果 | 负责：陷阱/危险区域的定义与触发；不负责：触发后的效果执行 |
| SurfaceType | 地面表面类型枚举，描述格子表面的物质状态 | 负责：表面物质分类（正常/冰面/油面/水面/毒池/灼烧）；不负责：表面的交互规则 |
| TerrainInteraction | 地形交互规则，定义技能/效果与地形的互动方式 | 负责：地形改变规则（点火/冻结/破坏）；不负责：交互的技能消耗 |

### 地形分类体系

```
TileProperties
 ├── TerrainType（地形类型）
 │    ├── 平地（Normal）：基础地形，无特殊效果
 │    ├── 高地（Highground）：提供视野/命中优势
 │    ├── 障碍（Obstacle）：不可通行，提供掩体
 │    ├── 水域（Water）：可通行但消耗翻倍
 │    ├── 丛林（Bush）：提供隐蔽/遮蔽
 │    ├── 冰面（Ice）：移动消耗翻倍，冰面行走判定
 │    ├── 毒池（Poison）：进入后施加中毒效果
 │    ├── 灼烧（Burning）：每回合燃烧伤害
 │    ├── 油面（Oil）：可被点燃，燃烧扩散
 │    └── 岩浆（Lava）：极高伤害，部分单位免疫
 │
 ├── Height（层高）
 │    └── 整数值（0 = 地面层，1+ = 高地层）
 │
 ├── Passability（通行性）
 │    ├── Walkable（可行走）：单位可进入
 │    ├── Blocked（阻挡）：单位不可进入（障碍/边界）
 │    ├── Flyable（可飞行）：飞行单位可越过
 │    └── Impassable（不可通行）：所有单位不可进入
 │
 └── Concealment（遮蔽度）
      ├── None（无遮蔽）：完全可见
      ├── Half（半遮蔽）：隐蔽 -2 命中
      └── Full（全遮蔽）：不可见，无法作为目标
```

### 已对齐项目术语

- **Tactical**：Tactical 领域的移动/掩体判定依赖 Terrain 的通行性和遮蔽度数据
- **Effect**：TerrainEffect 通过 Effect 领域的机制实现（如毒池=Effect(Periodic)+Modifier）
- **Combat**：Combat 领域在伤害计算时引用 Terrain 的遮蔽度和高地数据
- **Targeting**：Targeting 的视野校验依赖 Terrain 的遮蔽度数据

## 2. 地形交互状态机

### 表面状态变化

```
Default（默认表面）
   │  [技能/效果作用]
   ▼
Modified（表面被改变）
   │
   ├──→ 冰面（Ice）← 水系法术/冻结
   ├──→ 油面（Oil）← 油脂术/油瓶
   ├──→ 灼烧（Burning）← 火系法术/点燃
   └──→ 毒池（Poison）← 毒系法术/毒气
           │
           │  [持续时间到期 / 驱散]
           ▼
      Reverting（恢复中）
           │
           │  [恢复到原来的表面类型]
           ▼
      Default（恢复为默认）
```

## 3. 不变量（Invariants）

### 3.1 通行性一致性
- **条件**：所有移动/路径计算时
- **不变量**：Terrain 提供的通行性数据必须与地图配置一致，运行时不可被移动逻辑修改
- **违反后果**：移动逻辑自行修改通行性导致地图数据不一致
- **违反后果类型**：🔴 程序错误

### 3.2 地形效果绑定格子
- **条件**：TerrainEffect 被触发时
- **不变量**：每个地形效果必须绑定到具体的 Tile（格子）上，不绑定到单位
- **违反后果**：地形效果跟随单位移动（应通过 Effect 领域实现，而非 Terrain）
- **违反后果类型**：🔴 程序错误

### 3.3 表面变化可逆
- **条件**：任何表面类型变化时
- **不变量**：每个表面变化必须有对应的恢复机制（到期恢复/驱散恢复）
- **违反后果**：表面变化永久化，地图状态被不可逆修改
- **违反后果类型**：🔴 程序错误

### 3.4 高度数据一致性
- **条件**：地图网格构建时
- **不变量**：相邻格的高度差不得超过预设的最大值（防止悬崖/不合理地形）
- **违反后果**：高度差过大的相邻格导致寻路/移动异常
- **违反后果类型**：🔴 程序错误

### 3.5 陷阱触发可预期
- **条件**：HazardZone 定义时
- **不变量**：每个陷阱必须有明确的触发条件、影响范围和触发后的效果定义
- **违反后果**：触发条件模糊导致陷阱被多次触发或从不触发
- **违反后果类型**：🔴 程序错误

## 4. 禁止事项（Forbidden）

- 🟥 禁止：Terrain 领域包含单位的网格位置信息 — 理由：单位位置归 Tactical 领域，Terrain 只描述格子的属性
- 🟥 禁止：地形效果永久改变地图配置 — 理由：所有表面变化必须可恢复（Infinite 类型效果例外，但需显式声明）
- 🟥 禁止：Terrain 直接修改单位属性 — 理由：地形通过 TerrainEffect 间接影响单位，走 Effect→Modifier 管线
- 🟥 禁止：同一格堆叠多种互斥的表面类型（如同时是冰面和灼烧） — 理由：互斥表面类型同时存在导致效果冲突
- 🟥 禁止：TerrainDef 中直接存储用户可见文本的自然语言 — 理由：必须使用 name_key/desc_key: LocalizationKey 引用。违反宪法 §22 Localization First。

## 5. 流程定义

### 5.1 进入地形格

- **输入**：进入格子的单位、目标格子
- **处理**：
  1. 检查目标格子的通行性（不变量 3.1）— 不可通行时阻止进入
  2. 检查目标格子的表面类型
  3. 如果有 TerrainEffect 定义（如毒池/灼烧），调用 Effect 领域施加地形效果
  4. 如果有 HazardZone 触发条件满足，触发陷阱
  5. 更新单位位置到目标格
- **输出**：TerrainEffect 触发事件（如有）
- **失败处理**：通行性检查不通过时进入被阻止 → 这是**程序错误**（通行性数据不一致属于地图配置问题，应记 Bug）

### 5.2 表面变化（地形交互）

- **输入**：目标格子、目标表面类型、持续时间/条件
- **处理**：
  1. 检查当前表面是否可被目标表面覆盖（互斥检查）
  2. 创建表面变化记录（原始表面类型、变化后类型、持续时间和恢复机制）
  3. 更新格子的表面类型
  4. 如果有对应的 TerrainEffect，注册到格子
  5. 设置到期恢复定时器
- **输出**：SurfaceChanged 事件
- **失败处理**：互斥表面类型冲突时变化失败 → 这是**程序错误**（互斥检查应在调用前由调用方保证，属于数据/配置问题，应记 Bug）

### 5.3 陷阱触发

- **输入**：进入陷阱区域的单位、HazardZone 定义
- **处理**：
  1. 检查触发条件（阵营/单位类型/是否飞行等）
  2. 如果触发条件满足，执行陷阱效果（调用 Execution 或 Effect 领域）
  3. 如果陷阱为消耗型（如一次性的捕兽夹），触发后标记为已消耗
- **输出**：HazardTriggered 事件（包含伤害/效果数据）
- **失败处理**：触发条件不满足时陷阱不触发 → 这是**规则失败**（预期业务分支，陷阱触发机制的正常流程，触发条件未满足是合法结果）

## 6. 领域事件

| 事件名 | 触发时机 | 携带数据 | 订阅者 |
|--------|----------|----------|--------|
| TileEntered | 单位进入格子时 | entity_id, tile_pos, tile_properties, surface_type | Effect（施加地形效果）、Tactical（更新位置）、日志（LogCode: TER001） |
| SurfaceChanged | 格子表面类型变化时 | tile_pos, old_surface, new_surface, duration | Terrain（更新显示）、Cue（表面变化特效）、日志（LogCode: TER002） |
| HazardTriggered | 陷阱触发时 | tile_pos, target_id, hazard_id, damage/effect | Combat（处理伤害）、Effect（施加效果）、日志（LogCode: TER003） |
| TerrainEffectApplied | 地形效果施加时 | entity_id, tile_pos, effect_id | UI（显示地形状态图标）、日志（LogCode: TER004） |

### 事件订阅关系图

```
TileEntered
    │
    ├──→ Effect：施加格子上的 TerrainEffect（如进入毒池→施加中毒）
    ├──→ Resource: 触发格子上 HazardZone 的检测
    └──→ Trigger：触发"进入地形"相关触发器

SurfaceChanged
    │
    ├──→ Tactical：更新格子的通行性/遮蔽度数据
    ├──→ Cue：播放表面变化特效（如冻结/燃烧动画）
    └──→ UI：更新格子视觉效果

HazardTriggered
    │
    ├──→ Execution：计算陷阱伤害
    ├──→ Effect：施加陷阱附带的效果
    └──→ Cue：陷阱触发特效
```

## 7. 与已有架构的对齐校验

- ✅ 架构边界：Terrain 域位于 `core/domains/terrain/`，components.rs 定义 TileProperties/TerrainEffect/HazardZone，systems/ 实现地形效果/陷阱/表面/交互系统，rules/ 定义伤害/移动/交互规则
- ✅ 职责明确：Terrain 只描述"格子有什么属性/效果"，不管理"单位在哪个格子上"（Tactical 的职责）
- ✅ 表面可变可逆：所有表面变化都设计了恢复机制（到期/驱散），防止地图状态被不可逆破坏
- ✅ 地形效果通过 Effect/Modifier 管线传递，不走 bypass
- ✅ LocalizationKey：本领域涉及的用户可见文本使用 LocalizationKey 而非硬编码文本（宪法 §22）
