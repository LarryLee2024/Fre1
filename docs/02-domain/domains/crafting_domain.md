## 1. 统一术语

| 术语 | 定义 | 职责边界 |
|------|------|----------|
| Recipe | 配方，定义制作所需材料、工艺和产出 | 负责：配方数据的定义（材料/工具/技能要求）；不负责：制作流程的执行 |
| CraftingStation | 制作台，不同类型的制作设施（锻造台/附魔台/炼金台） | 负责：制作台的类型定义与使用许可；不负责：制作的工序 |
| Enchantment | 附魔/词条，额外的魔法效果附加到装备上 | 负责：附魔的定义与规则；不负责：附魔效果的数值实现（归 Modifier 管线） |
| Material | 材料/原料，制作消耗的资源 | 负责：材料的分类与定义；不负责：材料的来源（采集/掉落/购买） |
| UpgradeLevel | 升级等级，装备的强化等级（如 +1 → +2 → +3） | 负责：升级等级的规则与上限；不负责：升级的数值加成 |

### 制作类型

```
制作类型
 ├── 锻造（Smithing）：制作/升级金属装备（武器/铠甲）
 │    示例：铁匠铺打造精钢长剑 +1
 ├── 附魔（Enchanting）：为装备添加魔法词条
 │    示例：给长剑附加"烈焰"附魔（+1d6 火焰伤害）
 ├── 炼金（Alchemy）：制作药水/毒药/炼金炸弹
 │    示例：制作治疗药水（恢复 2d4+2 HP）
 ├── 裁缝（Tailoring）：制作布甲/披风/背包
 │    示例：制作+1 智力属性的法师袍
 └── 工程（Engineering）：制作机关/陷阱/特殊装置
       示例：制作闪光弹（致盲范围内敌人 1 回合）
```

### 配方结构

```
Recipe {
    recipe_id: "longsword_+1",
    name: "精钢长剑 +1",
    station: CraftingStation::Forge（需要锻造台）,
    skill_requirement: { proficiency: "smithing_tools", dc: 13 },
    materials: [
        { item: "铁锭", quantity: 3 },
        { item: "木材", quantity: 1 },
        { item: "磨刀石", quantity: 1 },
    ],
    output: {
        item_template: "longsword_+1",
        quantity: 1,
        enchantment_slots: 2（锻造时预留 2 个附魔槽位）,
    },
    craft_time: 2（单位：小时/回合/交互次数）,
}
```

### 附魔规则

```
EnchantmentSlot（附魔槽位）
 ├── 武器：最多 3 个词条
 ├── 铠甲：最多 2 个词条
 ├── 饰品（戒指/项链）：最多 1 个词条
 └── 普通物品：无附魔槽位（不可附魔）

 附魔类型：
 ├── 元素附魔：+1d6 火焰/寒冰/雷电/强酸伤害
 ├── 属性附魔：+1/2/3 属性值（如力量腰带）
 ├── 防御附魔：+1/2/3 AC（如防御戒指）
 └── 特殊附魔：特定能力（如"对亡灵额外 +2d8 伤害"）

 互斥规则：
 - 同类型词条互斥（不能同时附魔"烈焰"和"冰霜"）
 - 互斥词条后附魔的会覆盖先附魔的
```

### 已对齐项目术语

- **Inventory**：制作的产出（物品/装备）进入背包；材料从背包消耗
- **Effect**：炼金产出的药水/毒药使用后走 Effect 管线
- **Modifier**：附魔/升级装备的数值加成通过 Modifier 管线实现
- **Economy**：制作材料可在商店购买，成品可出售

## 2. 制作状态机

```
RecipeSelected（配方已选择）
   │  [检查材料 + 检查制作台 + 检查技能]
   ├──→ 条件全部满足
   │
   ▼
Crafting（制作中）
   │  [消耗材料]
   │  [等待制作时间完成]
   │      │
   │      ├──→ [成功] → 产出物品
   │      │
   │      └──→ [失败（如技能检定未通过）] → 部分材料损失
   │
   ▼
Crafted（制作完成——物品已产出）
```

## 3. 不变量（Invariants）

### 3.1 材料充足性
- **条件**：任何制作开始时
- **不变量**：玩家背包中的材料数量必须满足配方的全部材料需求
- **违反后果类型**：🔴 规则失败
- **违反后果**：材料不足时制作无法开始

### 3.2 制作台匹配
- **条件**：制作开始时
- **不变量**：制作必须使用配方指定的 CraftingStation 类型
- **违反后果类型**：🔴 规则失败
- **违反后果**：在错误类型的工作台上无法制作对应物品

### 3.3 附魔槽位上限制
- **条件**：任何附魔操作时
- **不变量**：装备上已附魔的词条数不得超过该装备类型的槽位上限制
- **违反后果类型**：🔴 规则失败
- **违反后果**：超出槽位上限的附魔未被应用

### 3.4 升级等级上限
- **条件**：装备升级时
- **不变量**：装备的升级等级不得超过装备稀有度对应的上限（普通≤+0，非凡≤+1，稀有≤+2，史诗≤+3）
- **违反后果类型**：🔴 规则失败
- **违反后果**：升级超过上限导致装备属性异常

### 3.5 互斥词条防冲突
- **条件**：附魔新词条时
- **不变量**：新附魔的词条如果与已有词条属于互斥类型，则新词条替换旧词条
- **违反后果类型**：🔴 规则失败
- **违反后果**：互斥词条共存导致效果冲突或叠加

## 4. 禁止事项（Forbidden）

- 🟥 禁止：制作直接产出 Modifier 或 Effect — 理由：制作产出的是物品（ItemInstance），物品的 Modifier/Effect 由装备/使用流程触发
- 🟥 禁止：制作过程中消耗背包中不存在的材料 — 理由：材料必须在开始制作时从背包移除
- 🟥 禁止：跳过制作时间直接获得成品 — 理由：制作时间是游戏体验的一部分（除非有"立即完成"的特殊能力）

## 5. 流程定义

### 5.1 制作

- **输入**：角色、Recipe 选择、CraftingStation
- **处理**：
  1. 检查配方所需的材料是否在背包中且数量充足（不变量 3.1）
  2. 检查制作台类型是否匹配（不变量 3.2）
  3. 检查角色是否满足技能要求（如武器匠熟练度）
  4. 消耗材料（从背包移除）
  5. 开始制作（等待制作时间）
  6. 制作完成：
     a. 技能检定（DC 由配方定义，d20 + 工具熟练度）
     b. 检定成功 → 产出物品到背包
     c. 检定失败 → 部分材料损失（配方定义"成功时可回收"和"失败时损失"的比例）
  7. 发布 ItemCrafted 事件
- **输出**：ItemCrafted 事件（制作者、配方、产出物品、技能检定结果）
- **失败处理**：材料不足/制作台不匹配/技能不足时制作无法开始 → 这是**规则失败**（预期业务分支，制作条件不满足）

### 5.2 附魔

- **输入**：角色、目标装备、附魔配方、附魔台
- **处理**：
  1. 检查目标装备是否有空闲的附魔槽位（不变量 3.3）
  2. 检查附魔材料是否充足
  3. 如果新词条与已有一词条互斥（不变量 3.5），替换旧词条
  4. 如果新词条不互斥且有槽位，添加新词条
  5. 消耗附魔材料
  6. 更新装备的 Modifier 列表
  7. 发布 EnchantmentApplied 事件
- **输出**：EnchantmentApplied 事件（装备、旧词条（如有替换）、新词条）
- **失败处理**：无空闲槽位且无互斥替换时附魔失败 → 这是**规则失败**（预期业务分支，附魔槽位已满需先移除旧词条）

### 5.3 装备升级

- **输入**：角色、目标装备、升级材料、锻造台
- **处理**：
  1. 检查当前升级等级是否已达上限（不变量 3.4）
  2. 检查升级材料是否足够
  3. 消耗升级材料
  4. 升级等级 +1
  5. 更新装备的 Modifier（+1 装备 → 命中+1、伤害+1）
  6. 发布 ItemUpgraded 事件
- **输出**：ItemUpgraded 事件（装备、旧等级、新等级、新 Modifier）
- **失败处理**：已达上限时无法继续升级 → 这是**规则失败**（预期业务分支，装备已达当前稀有度的最高升级等级）

## 6. 领域事件

| 事件名 | 触发时机 | 携带数据 | 订阅者 |
|--------|----------|----------|--------|
| ItemCrafted | 制作完成时 | entity_id, recipe_id, output_item, materials_consumed[ ], skill_check_result | Inventory（物品入背包）、Quest（检查制作任务）、UI（显示制作结果） |
| EnchantmentApplied | 附魔完成时 | entity_id, equipment_item, old_enchantment（如有替换）, new_enchantment | Modifier（更新装备 Modifier）、UI（显示附魔结果） |
| ItemUpgraded | 装备升级时 | entity_id, equipment_item, old_level, new_level, new_modifiers[ ] | Modifier（更新装备 Modifier）、UI（显示升级结果） |
| CraftingFailed | 制作失败时 | entity_id, recipe_id, fail_reason（材料不足/技能不足/失败）, materials_lost[ ] | UI（显示失败原因）、日志 |

### 事件订阅关系图

```
ItemCrafted
    │
    ├──→ Inventory：制作的物品进入背包
    ├──→ Quest：检查制作相关任务进度
    ├──→ UI：显示制作成功界面
    └──→ Cue：锻造/制作完成特效

EnchantmentApplied
    │
    ├──→ Modifier：更新装备的 Modifier 列表
    ├──→ Aggregator：触发属性重算（如果装备已穿戴）
    ├──→ UI：显示附魔效果
    └──→ Cue：附魔特效

ItemUpgraded
    │
    ├──→ Modifier：更新装备的 Modifier
    ├──→ Aggregator：触发属性重算
    ├──→ UI：显示升级成功效果
    └──→ Cue：升级特效
```

## 7. 与已有架构的对齐校验

- ✅ 架构边界：Crafting 域位于 `core/domains/crafting/`，components.rs 定义 RecipeBook/CraftingStation/EnchantmentSlot，systems/ 实现制作/附魔/升级/分解系统，rules/ 定义配方/附魔/材料规则
- ✅ 制作产出通过 Inventory 进入背包：不 bypass 背包系统
- ✅ 附魔/升级通过 Modifier 管线：装备属性变化走 Modifier→Aggregator，不走 bypass
- ✅ 有失败机制：技能检定失败导致部分材料损失，增加制作深度
