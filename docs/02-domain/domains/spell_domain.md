## 1. 统一术语

| 术语 | 定义 | 职责边界 |
|------|------|----------|
| Spell | 法术，一种特殊的需消耗法术位释放的能力 | 负责：法术的施放条件与流程，Spell 的 LocalizationKey（name_key/desc_key）；不负责：法术的技能效果（归 Ability 领域） |
| SpellSlot | 法术位，记录各环级法术位的总数与已用量 | 负责：法术位的消耗与恢复管理；不负责：法术位的获取（归 Progression 领域） |
| Concentration | 专注状态，施法者正在维持一个需要专注的法术 | 负责：专注的建立、维持与打断；不负责：专注法术的具体效果 |
| Spellbook | 法术书/法术列表，记录角色已知和已准备的法术 | 负责：法术的已知/已准备管理；不负责：法术的施放流程 |
| SaveDC | 法术豁免难度等级，目标抵抗法术效果的难度值 | 负责：豁免 DC 的计算（8 + 熟练 + 施法属性调整值）；不负责：豁免检定的执行 |
| UpcastRule | 升环施法规则，使用高环法术位施放低环法术时的效果增强 | 负责：升环的增益计算规则；不负责：增益的具体数值实现 |

### 法术环阶体系（D&D 5e）

```
Spell Level（法术环阶）
 ├── 0 环（戏法，Cantrip）：不消耗法术位，可随意施放
 ├── 1 环（一级法术）
 ├── 2 环（二级法术）
 ├── 3 环（三级法术）—— 关键分水岭，3 环法术强度显著提升
 ├── 4 环（四级法术）
 ├── 5 环（五级法术）
 ├── 6 环（六级法术）
 ├── 7 环（七级法术）
 ├── 8 环（八级法术）
 └── 9 环（九级法术）—— 最高环阶

 施法者等级与法术位：
 ┌──────────────────────────────────────┐
 │ 等级│1环│2环│3环│4环│5环│6环│7环│8环│9环│
 ├──────────────────────────────────────┤
 │ 1级 │ 2 │ - │ - │ - │ - │ - │ - │ - │ - │
 │ 3级 │ 4 │ 2 │ - │ - │ - │ - │ - │ - │ - │
 │ 5级 │ 4 │ 3 │ 2 │ - │ - │ - │ - │ - │ - │
 │ 7级 │ 4 │ 3 │ 3 │ 1 │ - │ - │ - │ - │ - │
 │ 9级 │ 4 │ 3 │ 3 │ 3 │ 1 │ - │ - │ - │ - │
 │ ... │...│...│...│...│...│...│...│...│...│
 │ 20级│ 4 │ 3 │ 3 │ 3 │ 3 │ 2 │ 2 │ 1 │ 1 │
 └──────────────────────────────────────┘
```

### Speall：施法组件

```
SpellComponent（施法组件）
 ├── Verbal（语言成分 V）：必须能说话才能施法。沉默状态禁止施法
 ├── Somatic（姿势成分 S）：必须能自由活动一只手。束缚状态禁止施法
 └── Material（材料成分 M）：需要特定的施法材料或法器。无材料时可用施法法器代替

 规则：
 - 法术的组件需求在法术定义中固定
 - 缺少任一组件时施法失败
 - 语言/姿势成分会被"沉默术"和"定身术"等效果封锁
```

### 已对齐项目术语

- **Ability**：Spell 是一种特殊类型的 Ability，复用了 Ability 领域的技能生命周期（消耗→施法→生效→冷却）
- **Combat**：Spell 在 Combat 的回合框架内执行（标准动作施法）
- **Reaction**：部分法术是反应法术（护盾术 +5AC、法术反制），由 Reaction 领域管理
- **Condition**：施法条件检查（沉默/束缚状态）使用 Condition 领域
- **Effect**：法术产生的持续性效果归 Effect 领域管理
- **Progression**：法术位数量由职业等级决定（归 Progression 领域）

## 2. 施法状态机

```
Known（已习得——在法术书中）
   │  [准备法术]
   ▼
Prepared（已准备——可随时施放）
   │  [选择施法 → 检查组件 → 检查法术位 → 检查专注]
   │
   ├──→ [戏法] ↓（无消耗，直接施法）
   │
   └──→ [1-9环法术] ↓
           │
           ▼
      Casting（施法中）
           │  [动作时长（标准动作/附赠动作/反应/长施法时间）]
           │
           ├──→ [施法完成]
           │      │
           │      ├──→ [消耗法术位]
           │      ├──→ [目标选择]
           │      ├──→ [豁免检定]
           │      ├──→ [效果执行 → 调用 Ability/Effect 领域]
           │      └──→ [如果是专注法术 → 建立专注连接]
           │
           └──→ [专注打断/施法被打断] → 法术失败，不消耗法术位
```

### 状态转换规则

| 转换 | 触发条件 | 动作 |
|------|---------|------|
| Known → Prepared | 长休后选择准备法术列表 | 从法术书中选择 Prepared 法术 |
| Prepared → Casting | 玩家/AI 选择施放法术 | 检查组件/法术位/专注（不变量检查） |
| Casting → Complete | 施法时间完成 | 消耗法术位，执行法术效果 |
| Casting → Failed | 被打断/专注断掉 | 法术失败，不消耗法术位 |
| Complete → 效果执行 | 法术效果结算完毕 | 法术书中的状态回到 Prepared |

## 3. 不变量（Invariants）

### 3.1 法术位不可透支
- **条件**：任何 1-9 环法术施放时
- **不变量**：当前可用法术位数量 >= 施放所需法术位数量（升环时以实际消耗环级为准）
- **违反后果类型**：🔴 规则失败
- **违反后果**：法术位不足时施法失败，法术位不消耗

### 3.2 专注唯一性
- **条件**：施法者正在维持一个专注法术时
- **不变量**：同一时间最多只能维持一个专注法术。施放新专注法术时，旧专注自动解除
- **违反后果类型**：🔴 规则失败
- **违反后果**：多个专注法术同时存在，破坏游戏平衡

### 3.3 施法组件必检查
- **条件**：任何法术施放前
- **不变量**：必须检查语言/姿势/材料三种施法组件的可用性
- **违反后果类型**：🔴 规则失败
- **违反后果**：缺少组件的法术被施放，违反法术规则

### 3.4 法术环阶与施法者等级匹配
- **条件**：法术施放时
- **不变量**：施法者等级必须满足该法术环阶的最低等级要求
- **违反后果类型**：🔴 规则失败
- **违反后果**：低等级角色施放高环法术（特殊情况如魔法物品/卷轴除外）

### 3.5 专注打断规则一致性
- **条件**：专注法术的施法者受到伤害时
- **不变量**：专注打断检定 DC = max(10, 所受伤害的一半)，每次受伤均需检定
- **违反后果类型**：🔴 规则失败
- **违反后果**：专注打断概率不符合 D&D 5e 规则

## 4. 禁止事项（Forbidden）

- 🟥 禁止：无法术位时仍可施放 1-9 环法术 — 理由：法术位是施法的消耗资源，法术位耗尽后只能施放戏法
- 🟥 禁止：同一角色同时维持多个专注法术 — 理由：专注唯一性是 D&D 5e 的核心规则之一
- 🟥 禁止：跳过法术组件检查的施法（除非法术声明"无需组件"） — 理由：组件是施法的基础约束
- 🟥 禁止：非施法者职业施放法术（除非通过魔法物品/卷轴/种族能力） — 理由：职业决定施法能力
- 🟥 禁止：SpellDef 中直接存储用户可见文本的自然语言 — 理由：必须使用 name_key/desc_key: LocalizationKey 引用。违反宪法 §22 Localization First。

## 5. 流程定义

### 5.1 施法流程

- **输入**：施法者、法术定义（SpellDef）、目标/位置、施法环级（是否升环）
- **处理**：
  1. 检查法术组件：语言（是否沉默）、姿势（是否束缚）、材料（是否有法器/材料包）
  2. 检查法术位：当前环级是否有可用法术位（升环时检查目标环级）
  3. 如果为专注法术，检查当前是否有其他专注阻止（不变量 3.2）
  4. 消耗施法动作类型（标准动作/附赠动作/反应/1 分钟+）
  5. 如果施法时间 > 1 动作，进入持续施法状态
  6. 施法完成时消耗法术位
  7. 执行法术效果（目标选择 → 豁免检定 → 调用 Ability/Effect 领域）
  8. 如果是专注法术，建立专注连接
  9. 发布 SpellCast 事件
- **输出**：SpellCast 事件（施法者、法术、目标、环级、结果）
- **失败处理**：组件检查/法术位/专注任一不满足时施法失败，不消耗法术位 → 这是**规则失败**（预期业务分支，施法条件不满足）
- **程序错误**：法术定义（SpellDef）在数据层不存在时 → 这是**程序错误**（系统异常，应记 Bug）

### 5.2 法术位恢复

- **输入**：恢复事件（长休完成/特殊能力触发）
- **处理**：
  1. 长休：所有法术位恢复到满（根据当前职业等级的法术位表）
  2. 短休：不恢复法术位（除非有特殊职业特性）
  3. 特殊能力：按能力描述恢复指定环级/数量的法术位
- **输出**：SpellSlotsRestored 事件
- **失败处理**：恢复量超过上限时 clamp 到最大法术位数 → 这是**规则失败**（预期业务分支，资源恢复的边界处理）

### 5.3 专注维持与打断

- **输入**：专注中的施法者受到伤害、专注法术的状态变化
- **处理**：
  1. 施法者受到伤害时：
     a. 计算专注检定 DC = max(10, 伤害值的一半)
     b. 施法者进行体质豁免检定
     c. 检定通过 → 专注维持
     d. 检定失败 → 专注打断 → 法术效果终止
  2. 施法者主动结束专注时 → 专注解除
  3. 专注法术的持续时间结束 → 专注自动解除
  4. 专注解除时发布 ConcentrationBroken 事件
- **输出**：专注状态变化通知，ConcentrationBroken 事件（如打断）
- **失败处理**：专注打断时法术效果终止，已产生的 Modifier 按 Effect 移除流程回退 → 这是**规则失败**（预期业务分支，专注打断是战斗的正常机制）

### 5.4 法术豁免

- **输入**：法术的目标、法术的豁免类型（力量/敏捷/体质/智力/感知/魅力）
- **处理**：
  1. 计算法术豁免 DC = 8 + 熟练加值 + 施法属性调整值 + 其他加值
  2. 目标进行对应属性的豁免检定（d20 + 属性调整值 + 熟练加值（如擅长））
  3. 如果豁免检定 >= DC → 豁免成功（通常是伤害减半或无效）
  4. 如果豁免检定 < DC → 豁免失败（法术完全生效）
  5. 自然 20 → 豁免必定成功；自然 1 → 豁免必定失败
- **输出**：SaveResult（成功/失败/数值）
- **失败处理**：豁免类型不合法时默认按"豁免成功"处理 → 这是**程序错误**（配置/数据问题，应记 Bug）

## 6. 领域事件

| 事件名 | 触发时机 | 携带数据 | 订阅者 |
|--------|----------|----------|--------|
| SpellCast | 法术施放完成时 | caster, spell_def, spell_level, targets[ ], save_results[ ] | Combat（记录行动）、Effect（法术效果执行）、Cue（施法特效）、日志（LogCode: SPR001） |
| SpellSlotChanged | 法术位数量/已用量变化时 | entity_id, slot_level, total_slots, used_slots, remaining | UI（更新法术位显示）、日志（LogCode: SPR002） |
| ConcentrationStarted | 专注法术建立时 | entity_id, spell_def, concentration_duration | UI（显示专注图标）、Combat（标记专注状态） |
| ConcentrationBroken | 专注被打断时 | entity_id, broken_by（伤害/主动/持续时间到）, spell_def | Effect（终止专注法术效果）、Cue（专注打断特效） |
| SaveRolled | 豁免检定完成时 | entity_id, save_type, roll_value, modifier, dc, result | UI（显示豁免结果）、日志 |

### 事件订阅关系图

```
SpellCast
    │
    ├──→ Ability：执行法术的技能效果
    ├──→ Effect：应用法术产生的持续性效果
    ├──→ Combat：记录行动消耗
    ├──→ Cue：施法特效/音效
    ├──→ UI：更新法术位显示
    └──→ 日志：记录施法

ConcentrationStarted
    │
    ├──→ Combat：标记专注状态（被打断时发送伤害通知）
    ├──→ UI：显示专注图标
    └──→ Spell：记录当前专注法术

ConcentrationBroken
    │
    ├──→ Effect：终止专注法术的所有效果
    ├──→ Cue：专注消散特效
    ├──→ UI：移除专注图标
    └──→ 日志：记录专注打断原因
```

## 7. 与已有架构的对齐校验

- ✅ 架构边界：Spell 域位于 `core/domains/spell/`，components.rs 定义 SpellSlots/Concentration/Spellbook/CastingComponent，systems/ 实现施法/法术位/专注/豁免/反应法术系统，rules/ 定义环阶/专注DC/豁免DC/升环规则
- ✅ 施法流程复用 Ability 领域：Spell 复用 Ability 的生命周期（消耗激活→执行→完成），不另建施法引擎
- ✅ 专注规则对齐 D&D 5e：唯一专注 + 受伤时体质豁免检定 DC = max(10, 伤害/2)
- ✅ 不造新系统：法术位是资源属性（Attribute），专注是状态标签（Tag+Condition），均复用 Capabilities
- ✅ LocalizationKey：本领域涉及的用户可见文本使用 LocalizationKey 而非硬编码文本（宪法 §22）
