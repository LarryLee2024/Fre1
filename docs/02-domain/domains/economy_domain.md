## 1. 统一术语

| 术语 | 定义 | 职责边界 |
|------|------|----------|
| Wallet | 钱包，角色的货币持有量 | 负责：货币的存储与增删查改；不负责：货币的购买力计算 |
| Currency | 货币体系，定义游戏中使用的货币类型和换算关系 | 负责：货币单位及汇率的定义；不负责：货币的产出 |
| ShopInventory | 商店库存，定义商店出售的物品、价格和补货规则 | 负责：商品的供应与补货管理；不负责：商品的定价策略 |
| PriceModifier | 价格修正系数，基于声望/供需/特殊事件等影响最终价格 | 负责：价格修正的计算因子；不负责：基础价格的制定 |
| TransactionRecord | 交易记录，记录每一次交易行为的明细 | 负责：交易的可追溯记录；不负责：交易的执行流程 |

### 货币体系

```
Currency（货币体系）
 ├── 金币（Gold Piece, GP）：主货币
 ├── 银币（Silver Piece, SP）：1 GP = 10 SP
 ├── 铜币（Copper Piece, CP）：1 SP = 10 CP（1 GP = 100 CP）
 └── 特殊货币（由特定场景/活动产出，如"荣誉点数"、"龙鳞币"）

 换算规则：
 - 向下兑换：1 GP = 10 SP = 100 CP（无条件可兑换）
 - 向上兑换：100 CP = 10 SP = 1 GP（通常需要找 NPC 兑换）
 - 特殊货币不参与标准汇率体系
```

### 价格计算

```
最终价格 = 基础价格 × 声望折扣 × 供需系数 × 其他修正

声望折扣：
  中立（Neutral）：× 1.0（标准价）
  友好（Friendly）：× 0.9（-10%）
  尊敬（Honored）：× 0.8（-20%）
  崇敬（Revered）：× 0.7（-30%）
  敌对（Hostile）：× 2.0（+100%）
  仇恨（Hated）：不交易

供需系数：
  过剩（物品 > 需求）：× 0.8
  平衡（物品 = 需求）：× 1.0
  紧缺（物品 < 需求）：× 1.5
  短缺（极度稀缺）：× 2.0（或限购）

赃物（Stolen Goods）：
  赃物售价 × 0.5（正常商人拒收，仅黑市商人以半价收购）
```

### 已对齐项目术语

- **Inventory**：Economy 的交易结果影响 Inventory（购买获得物品/出售移除物品）
- **Faction**：价格折扣依赖 Faction 的声望数据
- **Quest**：任务可用货币作为奖励
- **Crafting**：制作需要购买材料，成品可出售

## 2. 交易状态机

```
Browsing（浏览中——查看商品列表和价格）
   │  [选择购买/出售]
   ▼
Negotiating（交易协商中——选择数量，计算总价）
   │  [确认交易]
   │
   ├──→ [购买]：检查钱包 → 检查背包空间 → 扣款 → 获得物品
   │
   └──→ [出售]：检查物品是否存在 → 计算价格 → 移除物品 → 增加货币
           │
           ▼
TransactionComplete（交易完成——货币与物品更新完毕）
```

### 状态转换规则

| 转换 | 触发条件 | 动作 |
|------|---------|------|
| Browsing → Negotiating | 选择购买/出售 | 计算价格，显示确认 |
| Negotiating → 购买完成 | 确认购买 + 钱包充足 + 背包空间 | 扣款，物品入背包 |
| Negotiating → 出售完成 | 确认出售 + 物品存在 | 物品从背包移除，加款 |
| 任一 → Failed | 条件不满足 | 显示失败原因，回到 Browsing |

## 3. 不变量（Invariants）

### 3.1 货币非负
- **条件**：任何货币变更后
- **不变量**：所有货币类型的持有量 >= 0
- **违反后果类型**：🔴 程序错误
- **违反后果**：负货币（负债）破坏经济体系

### 3.2 交易物存在性
- **条件**：任何交易确认前
- **不变量**：购买时商店必须有库存；出售时背包必须有该物品
- **违反后果类型**：🔴 规则失败
- **违反后果**：交易了不存在的物品

### 3.3 价格计算确定性
- **条件**：相同条件下的同一物品
- **不变量**：相同条件下（声望/供需/折扣）同一物品的价格必须一致
- **违反后果类型**：🔴 程序错误
- **违反后果**：同一物品在相同条件下价格波动

### 3.4 商店补货周期
- **条件**：商店库存管理时
- **不变量**：商店物品在补货周期内不刷新；补货周期到达时按规则恢复库存
- **违反后果类型**：🔴 程序错误
- **违反后果**：商店频繁/不规律补货破坏经济预期

### 3.5 赃物标记不可清除
- **条件**：赃物交易时
- **不变量**：被标记为"赃物"的物品在被卖给正常商人时自动触发特殊对话，卖给黑市商人时半价，赃物标记不会被游戏内常规方式清除
- **违反后果类型**：🔴 规则失败
- **违反后果**：赃物混入正常交易渠道

## 4. 禁止事项（Forbidden）

- 🟥 禁止：Economy 领域直接创建/销毁物品 — 理由：物品的创建/销毁归 Inventory 领域，Economy 只做货币与物品的交换
- 🟥 禁止：货币系统支持"贷款"或"透支" — 理由：货币非负原则，不支持透支
- 🟥 禁止：声望折扣在交易中途变化 — 理由：声望按交易时刻快照，不在一次交易流程中动态变化

## 5. 流程定义

### 5.1 购买

- **输入**：买家、商店、目标物品、数量
- **处理**：
  1. 检查商店库存是否充足
  2. 计算价格（基础价 × 声望折扣 × 供需系数）
  3. 检查买家钱包是否足够
  4. 检查买家背包是否有足够空位
  5. 从钱包扣款
  6. 从商店库存移除物品
  7. 物品加入买家背包
  8. 如有需要，更新商店的供需系数
  9. 发布 TransactionCompleted 事件
- **输出**：TransactionCompleted 事件（买家、物品、数量、总价、折扣明细）
- **失败处理**：库存不足/钱包不足/背包满时交易失败 → 这是**规则失败**（预期业务分支，交易条件不满足）

### 5.2 出售

- **输入**：卖家、商店/商人、目标物品、数量
- **处理**：
  1. 检查卖家背包中是否有该物品
  2. 计算出售价格（基础价 × 售价折扣系数（通常 0.5）× 声望折扣）
  3. 如果物品为赃物，应用赃物折扣（× 0.5）
  4. 从卖家背包移除物品
  5. 增加卖家钱包
  6. 物品加入商店库存（或根据商店类型，部分商人拒收赃物）
  7. 发布 TransactionCompleted 事件
- **输出**：TransactionCompleted 事件（卖家、物品、数量、所得金额、折扣明细）
- **失败处理**：物品不存在/商人拒收时交易失败 → 这是**规则失败**（预期业务分支，出售物品不存在或商人拒绝收购）

### 5.3 商店补货

- **输入**：补货周期触发
- **处理**：
  1. 检查当前时间是否到达补货时间点
  2. 按商店配置的补货规则恢复库存：
     - 固定补货：恢复至初始库存
     - 部分补货：按比例恢复（如每次补货恢复售出量的 50%）
     - 特殊补货：根据玩家行为/世界状态决定补货内容
  3. 更新供需系数
  4. 发布 PriceChanged 事件（如果价格有变化）
- **输出**：PriceChanged 事件（物品、旧价、新价、变化原因）
- **失败处理**：补货周期未到时不执行补货 → 这是**规则失败**（预期业务分支，补货遵循定时周期）

## 6. 领域事件

| 事件名 | 触发时机 | 携带数据 | 订阅者 |
|--------|----------|----------|--------|
| TransactionCompleted | 交易完成时 | entity_id, shop_id, item_id, quantity, total_price, price_breakdown, transaction_type（buy/sell） | Inventory（确认物品已增减）、Wallet（确认货币已变更）、UI（更新商店/钱包界面）、日志（LogCode: ECO001） |
| PriceChanged | 商店价格变化时 | shop_id, item_id, old_price, new_price, reason（补货/供需/事件） | UI（更新商店价格显示）、日志（LogCode: ECO002） |
| CurrencyChanged | 角色货币变化时 | entity_id, currency_type, old_amount, new_amount, delta, reason | UI（更新钱包显示）、日志（LogCode: ECO003） |

### 事件订阅关系图

```
TransactionCompleted
    │
    ├──→ Inventory：确认物品已在背包中/已移除
    ├──→ Wallet：确认货币已扣减/增加
    ├──→ UI：更新商店界面和钱包显示
    ├──→ Quest：检查"购买/出售特定物品"任务进度
    └──→ 日志：记录交易明细

CurrencyChanged
    │
    ├──→ UI：更新钱包显示
    └──→ 日志：记录货币变化原因
```

## 7. 与已有架构的对齐校验

- ✅ 架构边界：Economy 域位于 `core/domains/economy/`，components.rs 定义 Wallet/ShopInventory/PriceModifier，systems/ 实现商店/价格/交易系统，rules/ 定义定价/折扣/赃物规则
- ✅ 价格计算集成 Faction 声望：交易价格受 Faction 领域提供的声望数据影响
- ✅ 交易结果影响 Inventory：购买→物品入背包，出售→物品出背包
- ✅ 货币体系对齐 D&D 5e：GP/SP/CP 三级货币单位 + 标准汇率
