### 4. 核心设计规约（冻结·强制宪法条款）
#### 4.1 Buff 永久设计规约
**删除顶层Buff领域**，Buff属于**带回合时长的Effect**，统一归一：
1. 瞬时Effect：伤害、击退、即时治疗 → `Duration::Instant`
2. 回合Buff Effect：灼烧、眩晕、攻速增减 → `Duration::TurnLimited(u32)`
3. 永久常驻Effect：职业天赋、装备属性、被动光环 → `Duration::Permanent`

#### 4.2 Attribute 双分层强制规约
- Primary基础属性：力量、敏捷、智力、体质（原始只读、无运行时修改）
- Derived派生属性：攻击力、防御、暴击、移速、治疗加成（由Modifier统一计算）
- ❌ 禁止业务代码直接修改任意派生属性数值

#### 4.3 Stacking 全局枚举冻结（不新增、不修改）
```rust
enum StackingRule {
    // 覆盖原有同ID效果

    // 刷新剩余时长、保留层数
    RefreshDuration,
    // 层数叠加、无上限
    StackAdd,
    // 层数叠加、硬上限约束
    StackMax(u32)
}
```

#### 4.4 Execution 解耦强制规约
所有伤害/治疗/地形/百分比数值，全部抽离独立Execution Trait；
禁止Effect内部写公式、禁止业务match分支膨胀、禁止魔法数字硬编码。

#### 4.5 Cue 强制解耦规约
Cue仅下发**纯数据业务事件**，不携带特效、动画、UI资源；
表现层仅订阅Cue、反向零依赖业务战斗逻辑，严格遵守宪法分层。

#### 4.6 Registry 强制规约
技能/效果/算式/标签全部全局注册，代码只写通用逻辑；
新增玩法、数值、效果**纯配置完成、零业务代码改动**。
