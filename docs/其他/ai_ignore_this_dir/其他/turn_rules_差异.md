### 3. 不变量5：状态机不处理业务细节（新增）

**位置**：不变量

**变更类型**：新增

**新内容**：
```
## 不变量5：状态机不处理业务细节 🟥

宪法依据：2.1.2（数据与行为分离）、1.1.4（逻辑与表现分离）

任意时刻：
TurnPhase 的 OnEnter / OnExit 只负责流程转换，不包含业务逻辑。

违反表现：
OnEnter(ExecuteAction) 中包含伤害计算、Buff 应用等业务逻辑。

架构违规检测：
ARCHITECTURE VIOLATION: 状态机 OnEnter/OnExit 包含业务逻辑，违反"状态机只负责流程"原则。
```

**原因**：宪法 2.1.2 和 1.1.4 明确要求状态机与业务逻辑分离，这是 Turn 领域的核心不变量。
