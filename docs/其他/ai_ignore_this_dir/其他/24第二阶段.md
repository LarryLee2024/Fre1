# 二、下一步最关键的不是“功能”，而是3个升级

## 1. Skill → 数据化（最优先）

你现在大概率还是：

```rust
match skill_id {
    FireBall => ...
}
```

或者：

```rust
fn fireball_system() { ... }
```

### 下一步必须变成：

```text
Skill = Data
Effect = Data
System = Generic Executor
```

---

### 你要做的是：

#### Skill Definition化

```json
{
  "id": "fireball",
  "cost": 2,
  "range": 3,
  "effects": [
    {
      "type": "damage",
      "value": 120
    },
    {
      "type": "add_buff",
      "buff": "burn"
    }
  ]
}
```

---

### 核心变化

以前：

```text
技能 = 代码
```

现在：

```text
技能 = 数据 + 通用执行器
```

---

### 这一点的重要性：

它直接决定：

```text
你未来能不能做：
100角色 / 500技能 / mod系统
```

---

# 2. Map + Battle = “关卡数据化”


```yaml
    type: all_dead
```

---

### 关键变化

以前：

```text

```

现在：

```text
地图 = 数据驱动生成
```

# 3. Buff系统“事件化”

你现在Buff可能是：

```text
turn_start => damage
```

下一步升级：

```text
Buff = Event Listener + Effect Generator
```

---

### 标准模型

```text
Event → Buff → Effect → State Change
```

---

### 举例

```text
OnTurnStart
  ↓
BurnBuff触发
  ↓
DamageEffect
  ↓
HP减少
```

---

### 关键点

Buff不能再“主动改状态”

必须：

```text
响应事件 → 产生Effect
```
