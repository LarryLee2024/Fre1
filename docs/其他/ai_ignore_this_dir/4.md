这是很多 Godot 开发者转 Bevy 时最大的痛点之一。

因为：

```text
Godot
=
UI优先引擎

Bevy
=
ECS优先引擎
```

所以你在 Godot 里觉得理所当然的：

```text
VBoxContainer
HBoxContainer
GridContainer
Button
Panel
ScrollContainer
TabContainer
```

到了 Bevy 会变成：

```text
Node
Button
Text
```

然后自己拼。

很多人第一次接触会觉得：

> Godot 的 UI 怎么这么舒服，Bevy 怎么这么原始？

实际上 Bevy 的最佳实践和 Godot 完全不同。

---

# Godot 的本质

Godot UI：

```text
Control
│
├── VBoxContainer
├── HBoxContainer
├── GridContainer
├── Button
├── Label
└── ScrollContainer
```

属于：

```text
保留式UI（Retained UI）
```

你搭场景：

```text
CharacterPanel
├─ VBox
│  ├─ Name
│  ├─ HP
│  └─ MP
```

然后引擎维护状态。

---

```rust

```

```text

```

```text

```

立即模式 UI。


```text

```

# 如何获得 Godot Container 的体验？

这是关键。

很多人手写：

```rust
Node {
    width: px(100.),
    height: px(100.),
}
```

几十个节点。

非常痛苦。

---

实际上应该封装。

---

# 做自己的 VBox

例如：

```rust
pub fn vbox() -> Node {
    Node {
        flex_direction: FlexDirection::Column,
        row_gap: px(8.),
        ..default()
    }
}
```

以后：

```rust
parent.spawn(vbox());
```

---

就相当于：

```text
Godot VBoxContainer
```

---

# 做自己的 HBox

```rust
pub fn hbox() -> Node {
    Node {
        flex_direction: FlexDirection::Row,
        column_gap: px(8.),
        ..default()
    }
}
```

---

变成：

```text
Godot HBoxContainer
```

---

# 做自己的 Panel

```rust
pub fn panel() -> impl Bundle
```

统一：

```text
背景
圆角
边框
padding
```

---

以后：

```rust
parent.spawn(panel());
```

---

相当于：

```text
Godot Panel
```

---

# 做 Widget 库

大型项目都会这么干。

例如：

```text
ui/

widgets/

button.rs
panel.rs
vbox.rs
hbox.rs
label.rs
scroll.rs
window.rs
```

---

然后：

```rust
spawn_character_window()
```

内部：

```rust
panel
  -> vbox
      -> labels
      -> buttons
```

---

你写出来就像：

```text
Godot 场景树
```

一样。

# 推荐 DSL

Bevy 现在最舒服的是：

## BSN

Bevy Scene Notation。

例如：

```rust
bsn! {
    Node(
        flex_direction: Column
    ) {
        Text("角色")
        Button {
            Text("确定")
        }
    }
}
```

体验已经非常接近：

```text
Godot 场景树
```

# UI 最佳实践

不要：

```rust
spawn_button()
spawn_text()
spawn_node()
```

到处散落。

---

应该：

```text
CharacterPanel
InventoryPanel
SkillPanel
QuestPanel
```

每个面板一个模块。

---

例如：

```text
ui/

character_panel/
```

```text
mod.rs

layout.rs

events.rs

widgets.rs
```

---

# 数据驱动 UI

不要：

```rust
ui.label("HP");
ui.label("MP");
ui.label("ATK");
```

硬编码。

---

应该：

```yaml
character_sheet:
  sections:
    - attributes
    - skills
    - equipment
```

或者：

```rust
AttributeDefinition
```

驱动生成。

---

这样以后：

```text
增加士气
增加信仰
增加魅力
```

UI自动出现。

---

如果你是从 Godot 转过来，我的建议是：

> 不要试图在 Bevy 里寻找 VBoxContainer、GridContainer、PanelContainer 的一一对应物。

而是把 Bevy UI 理解成：

```text
CSS Flexbox
+
Widget封装
+
ECS数据绑定
```

当你自己封装出 `VBox`、`HBox`、`Panel`、`Window`、`ScrollView` 这些 Widget 后，开发体验会非常接近 Godot，甚至在大型项目里比 Godot 的场景树更容易模块化和复用。
