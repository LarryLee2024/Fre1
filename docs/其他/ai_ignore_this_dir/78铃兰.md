# 《铃兰之剑》全战斗系统模型解构
本文从属性、伤害、技能、Buff、装备、触发机制、时序管线七个维度完整拆解其战斗底层模型，所有机制均来自游戏实装规则与数值验证结论。

---

## 一、属性体系：双层结构 + 多源叠加 + 属性转换
铃兰的属性体系采用**「基础面板（黑字）+ 战斗修正（战斗内动态）」**两层架构，边界清晰，所有养成与Buff效果严格分层计算，避免数值混乱。

### 1.1 基础属性（战斗外固定，面板黑字）
属于角色固有属性，仅受养成系统影响，战斗中不会被直接修改，是所有百分比加成的计算基数。
- **核心五维**：物理攻击、魔法攻击、物理防御、魔法防御、最大生命值
- **次级属性**：暴击率、暴击伤害、移动范围、攻击范围、命中率、闪避率
- **养成来源**：角色等级/升星/升阶、装备基础值、刻印词条、塔罗词条、羁绊加成、天赋固定加成

### 1.2 战斗内属性修正（动态生效）
战斗中通过Buff、地形、技能、装备特效附加的属性调整，全部基于基础面板计算，分为两类：
- **无条件加成**：光环、常驻Buff、装备常驻百分比词条，开战即生效，加算叠加
- **条件加成**：高地加成、背击加成、濒死加成、血量阈值加成，满足条件时实时生效

### 1.3 特色机制：属性转换
这是铃兰职业差异化的核心设计，允许将防御、生命等属性按比例转换为攻击，支撑防战、血牛输出等特殊build。
- 典型例子：坦克角色「防御转攻击」、狂战「损失血量转攻击」
- 计算规则：转换后的属性可被后续百分比加成放大，但部分条件型加成不参与转换
- 归属：属于属性计算的中间层，在基础属性之后、最终属性之前结算

## 二、伤害计算模型：四段式分层计算，加减乘除边界严格
铃兰的伤害公式采用**分段独立计算**的设计，每一段的计算规则、叠加方式完全固定，不存在模糊区间，是典型的工业级SRPG数值架构。

### 2.1 伤害类型四大分类
| 伤害类型 | 防御减免 | 护盾抵挡 | 典型场景 |
|----------|----------|----------|----------|
| 物理伤害 | 受物防减免 | 物理护盾可挡 | 普攻、物理技能 |
| 魔法伤害 | 受魔防减免 | 魔法护盾可挡 | 法术技能、魔法反击 |
| 穿透伤害 | 无视防御 | 部分护盾可挡 | 斩杀、固定比例伤害 |
| 真实伤害 | 无视防御无视护盾 | 无法抵挡 | 灼烧、中毒、生命流失 |

### 2.2 完整四段伤害公式
#### 第一段：攻击值结算
```
最终攻击值 = (基础攻击 × (1 + 所有攻击百分比加算) + 固定攻击加成 + 属性转换攻击) × 属性克制倍率 × 高低地修正
```
- 攻击百分比：个性、装备、Buff、光环全部加算
- 属性克制：克制×1.3，被克制×0.7
- 高低地：高差≥2格时，高地+15%攻击，低地-10%攻击

#### 第二段：防御值结算
```
最终防御值 = (基础防御 × (1 - 所有降防效果乘积)) × (1 - 所有无视防御效果乘积)
```
- 核心规则：降防效果之间**乘算**，无视防御之间**乘算**，降防与无视防御之间也为乘算
- 示例：40%降防 + 40%无视防御，最终防御 = 基础防御 × 0.6 × 0.6 = 36%基础防御

#### 第三段：基础伤害结算
```
基础伤害 = (最终攻击值 - 最终防御值) × 技能倍率 × 暴击倍率
```
- 技能倍率：技能自带的伤害系数，部分技能可根据移动距离、目标数量动态变化
- 暴击倍率：默认150%，受爆伤加成加算叠加

#### 第四段：最终伤害修正
```
最终伤害 = 基础伤害 × (1 + 所有增伤效果加算) × (1 + 所有易伤效果乘积) × (1 - 所有减伤效果乘积)
```
- 增伤区间：背击、装备增伤、Buff增伤，全部**加算**
- 易伤区间：敌人debuff带来的受伤增加，多个易伤**乘算**
- 减伤区间：格挡、护盾、濒死减伤，多个减伤**乘算**

### 2.3 关键数值规则
1. **攻击%收益 > 增伤%收益**：因为攻击在防御前结算，会放大「攻-防」的整体基数
2. **减防/无视防御收益指数级增长**：乘算机制下，多层叠加收益远高于增伤
3. **穿透伤害单独计算**：不受防御影响，单独走穿透倍率，用于针对高防单位
4. **治疗/护盾同理分层**：治疗量基于攻击/治疗强度，受治疗加成、受治疗提升修正

## 三、技能系统：四维分类 + 标准化执行管线
铃兰的技能体系没有用复杂的GAS架构，而是采用**「类型 + 构成 + 目标 + 执行阶段」**的标准化模型，足够支撑丰富玩法，同时维护成本极低。

### 3.1 技能五大分类
| 技能类型 | 行动消耗 | 触发方式 | 典型例子 |
|----------|----------|----------|----------|
| 普攻 | 消耗1次行动 | 玩家主动释放 | 普通攻击、远程射击 |
| 主动技能 | 消耗能量+行动 | 玩家主动释放 | 火球、冲锋、治疗 |
| 反应技能 | 不消耗行动 | 事件触发 | 反击、援护、回击 |
| 支援技能 | 瞬发，剩余行动保留 | 玩家主动释放 | 加Buff、驱散、自身回血 |
| 被动技能 | 无消耗 | 常驻/条件触发 | 个性、天赋、装备被动 |

### 3.2 技能的标准化构成
每个技能都由以下固定字段组成，完全可配置化：
1. **基础信息**：ID、名称、技能类型、消耗（能量/冷却）、射程
2. **目标规则**：目标类型、选择范围、目标筛选条件（敌我、阵营、状态）
3. **前置校验**：消耗检查、冷却检查、标签检查（需具备/禁止具备的Tag）
4. **效果列表**：一个技能可挂载多个效果（伤害+加Buff+击退）
5. **特殊规则**：瞬发、移动后可释放、背击加成、多段攻击等

### 3.3 目标选择体系（独立领域）
目标选择完全从技能中解耦，是独立的算法模块，所有技能复用同一套目标类型：
- **单体目标**：选定单个敌方/友方单位
- **直线范围**：沿方向延伸的直线区域（穿刺、冲锋）
- **圆形范围**：以目标点为中心的圆形AOE
- **锥形范围**：扇形AOE
- **自身范围**：以自身为中心的周围区域
- **全局目标**：全场所有友军/敌军
- **路径目标**：移动路径上的所有单位

### 3.4 技能执行固定管线
所有技能严格按顺序执行，不可逆、不乱序，是战斗确定性和回放的基础：
```
1. 前置校验 → 检查冷却、消耗、目标合法性、标签限制
2. 扣除消耗 → 扣能量、标记冷却
3. 目标选取 → 调用目标算法得到最终目标列表
4. 效果执行 → 按顺序执行所有挂载效果（伤害→Buff→位移）
5. 事件广播 → 广播「技能释放完成」「造成伤害」等事件
6. 触发反应 → 检查目标的反击、援护等反应技
7. 结算收尾 → 更新UI、记录战斗日志
```

## 四、Buff与状态体系：三类状态 + 统一生命周期 + 堆叠规则
铃兰没有单独的「Buff模块」和「Debuff模块」，而是统一为**状态效果**，按时长、叠加方式、生效时机分类，逻辑高度统一。

### 4.1 状态三大分类
#### 1. 增益状态（橙色）
- 属性类：攻击提升、防御提升、暴击提升
- 功能类：护盾、免疫控制、无法被选中、反击
- 特殊类：能量回复、再行动、伤害吸收护盾

#### 2. 减益状态（红色/紫色）
- 属性类：攻击降低、防御降低、命中降低
- 控制类：眩晕、沉默、定身、嘲讽
- 持续伤害类：灼烧、中毒、感染（层数型）
- 功能类：易伤、减速、禁止治疗

#### 3. 特殊状态（蓝色/灰色）
不属于增益/减益，无法被驱散，比如：
- 濒死、受伤（血量阈值触发的状态）
- 斥力屏障、晶能防护（不可驱散的减伤）
- 召唤物归属、地形效果

### 4.2 统一生命周期
所有持续状态都遵循完全相同的生命周期节点：
1. **施加（Apply）**：添加到目标身上，触发入场效果，刷新持续时间/层数
2. **周期触发（Tick）**：按回合/按行动触发，比如中毒每回合掉血
3. **到期（Expire）**：持续时间归零，触发到期效果
4. **移除（Remove）**：被驱散/覆盖时触发，执行清理逻辑

### 4.3 层数型Buff机制
这是SRPG的核心玩法扩展，铃兰做了非常细致的分层：
- **层数独立计算**：每层效果独立，叠加层数提升效果强度
  - 例：晶能每层+5%攻防，最多10层；感染每层提升掉血，最多9层
- **层数衰减规则**：部分Buff触发后层数减半，而非直接清零
  - 例：感染行动结束掉血后，层数减半
- **不可驱散层数**：部分特殊层数Buff无法被驱散，只能靠时间衰减

## 五、堆叠与互斥规则：精细化的数值边界控制
这是铃兰数值体系最精髓的部分，通过明确的叠加规则，避免了数值膨胀和效果冲突，也是很多SRPG项目最容易出坑的地方。

### 5.1 同名与异名基础规则
1. **同名同效果Buff：取最大值，不叠加**
   - 例：两个「攻击提升20%」同时存在，只生效20%，不会变成40%
   - 高级Buff覆盖低级Buff，获得高等级后低等级自动失效
2. **异名同类型Buff：加算叠加**
   - 例：「攻击指挥+12%」+「阵营光环+10%」+「装备+6%」，合计+28%攻击
3. **同类型不同区间：分别计算，互不影响**
   - 攻击%、增伤%、易伤%属于不同区间，各自按自身规则叠加

### 5.2 关键类别堆叠规则
| 效果类型 | 叠加方式 | 备注 |
|----------|----------|------|
| 属性百分比加成 | 加算 | 所有来源全部相加 |
| 减防效果 | 乘算 | 多个降防相乘，收益递减 |
| 无视防御 | 乘算 | 多个穿甲效果相乘 |
| 增伤效果 | 加算 | 背击、装备、Buff全部相加 |
| 易伤效果 | 乘算 | 多个易伤debuff相乘 |
| 减伤效果 | 乘算 | 多个减伤buff相乘 |
| 暴击率 | 加算 | 所有暴击率加成相加 |
| 暴击伤害 | 加算 | 所有爆伤加成相加 |

### 5.3 特殊机制堆叠规则
#### 反击类效果
- 多个不同反击效果共存时，**只触发优先级最高的一个**（通常威力更高的优先级高）
- 「反击次数增加」效果对每个反击单独生效，分别加次数
- 例：回击+强力回击+反击次数+3，可分别触发，单回合最多8次反击

#### 光环类效果
- 同阵营同名光环不可叠加，取最高等级
- 不同功能的光环可同时生效（攻击光环+防御光环）

#### 护盾类效果
- 不同类型护盾（物理盾、魔法盾、全伤害盾）独立计算，同时生效
- 同类护盾叠加方式由配置决定：刷新时长、叠加数值、取最大值

## 六、装备与词条体系：四层结构 + 条件触发
铃兰的装备不是简单加属性，而是**「基础属性+词条+特效+刻印」**四层结构，既支撑养成深度，又严格遵守属性计算分层规则。

### 6.1 装备属性四层结构
#### 第一层：基础白值属性
- 武器加攻击、防具加防御生命，固定数值，随装备星级提升
- 计入角色基础面板，是所有百分比加成的基数

#### 第二层：常驻百分比词条
- 例：「物攻提升5%」「最大生命+10%」
- 计算时机：战斗开始时生效，基于基础面板计算，和其他百分比加算
- 属于无条件常驻加成，全程生效

#### 第三层：条件触发词条
- 例：「健康状态下物攻+6%」「释放技能后物攻+1%/能量」
- 计算时机：满足条件时实时生效，不满足时立即失效
- 基数同样为基础面板，和其他常驻百分比加算

#### 第四层：装备特效
- 纯机制效果，不直接加属性
- 例：「释放技能后回复能量」「攻击特定目标伤害提升」
- 通过事件触发，和技能、Buff体系打通

### 6.2 外围养成：刻印与塔罗
- 机制和装备词条完全一致，提供固定值+百分比属性
- 所有百分比加成全部计入「攻击百分比加算」区间，不搞特殊乘区
- 好处：养成系统再多，数值计算规则只有一套，不会失控

### 6.3 装备改属性的核心原则
1. **所有属性加成最终都汇入统一属性管线**，没有特殊计算的“私货”
2. **条件型词条不改变基础面板**，只在战斗内动态生效，不影响基数
3. **特效类全部走事件触发**，不侵入属性计算核心逻辑

## 七、被动触发与反应体系：事件驱动的反应技
这是铃兰战斗乐趣的核心，也是最能体现SRPG策略深度的部分，完全基于**事件-条件-效果**的触发器模型。

### 7.1 触发事件分类
所有触发器都挂载在固定的战斗事件上，事件是全局唯一的，所有被动、反应技、装备特效都监听同一套事件：
1. **回合事件**：回合开始、回合结束、行动开始、行动结束
2. **战斗事件**：造成伤害、受到伤害、击杀单位、被击杀
3. **技能事件**：释放技能前、释放技能后、技能命中
4. **状态事件**：获得Buff、失去Buff、控制生效、控制解除
5. **移动事件**：移动开始、移动结束、进入格子、离开格子

### 7.2 反应技核心规则
反应技是特殊的被动技能，有严格的触发限制，避免强度失控：
- **触发条件**：必须满足特定事件+特定条件（比如「被友军攻击时+自身濒死」）
- **次数限制**：每回合/每场战斗有最大触发次数
- **不消耗行动**：反应技不占用主动行动次数
- **独立伤害计算**：反应伤害单独走伤害公式，但通常有倍率衰减

典型反应技：反击（被攻击时普攻回击）、援护（友军被攻击时替伤）、追击（友军攻击时补刀）

### 7.3 触发优先级与冲突处理
1. **同事件多个触发器：按优先级顺序执行**
   - 高优先级先结算，比如减伤类先于伤害类
2. **同类反应只触发最高级**
   - 多个反击效果只触发优先级最高的一个，不重复触发
3. **触发链深度限制**
   - 反应技触发的伤害不能再触发新的反应，避免无限循环
   - 典型：反击不会触发对方的反击

## 八、战斗时序与阶段划分：回合制固定管线
铃兰的回合执行有严格的阶段划分，所有系统都在固定阶段运行，是战斗确定性和回放的基石。

### 8.1 完整回合时序
```
1. 回合开始阶段
   → 回合开始事件触发
   → Buff Tick（回合开始生效的效果）
   → 被动效果结算（回合开始触发的天赋、装备）

2. 单位行动阶段
   → 移动
   → 释放主动技能/普攻
   → 技能执行管线（校验→扣消耗→选目标→执行效果）
   → 反应技触发
   → 行动结束事件
   → Buff Tick（行动结束生效的效果，如中毒掉血）

3. 回合结算阶段
   → 所有Buff持续时间-1
   → 冷却时间-1
   → 到期Buff移除，触发到期效果
   → 回合结束事件触发
   → 被动效果结算（回合结束触发的效果）
```

### 8.2 关键时序原则
1. **同阶段效果按优先级执行**，不允许乱序
2. **所有状态变更都在固定阶段结算**，不会在技能执行中途突然刷新Buff
3. **反应技插入在「效果执行后」和「行动结束前」之间**，时序固定

## 九、设计精髓与对SRPG Lite-GAS的借鉴
### 9.1 铃兰之剑的核心设计优点
1. **分层极其清晰**：属性、伤害、技能、Buff、触发各层边界严格，没有跨层逻辑，维护成本极低
2. **规则高度统一**：所有Buff走同一生命周期、所有加成走同一套叠加规则、所有触发走同一套事件
3. **裁剪极其精准**：没有网络同步、预测回滚这些冗余内容，只做单机回合制需要的机制
4. **配置化程度高**：技能、Buff、效果几乎都可以配出来，不需要每次写新代码
5. **数值边界明确**：加算乘算区间划分清楚，不会出现数值膨胀失控

### 9.2 可直接落地到我们Lite-GAS架构的部分
1. **Attribute层**：直接采用「基础属性+战斗修正」双层结构，百分比加算、防御乘算的规则完全复用
2. **Effect层**：伤害、治疗、加Buff、位移、击退都做成独立Effect，和铃兰的效果列表模型一致
3. **Stacking层**：同名取最大、异名加算、乘算/加算分区的规则，直接照搬即可
4. **Trigger层**：全局事件+条件判断+执行效果的模型，和我们的触发器设计完全吻合
5. **执行管线**：技能七阶段执行流程，直接对应我们的Ability执行管线

### 9.3 不需要照搬的冗余部分
1. **多层养成系统**：刻印、塔罗、羁绊这些网游养成，单机项目不需要，只保留装备词条机制即可
2. **过于复杂的反应技分级**：我们不需要做太多反应技优先级，基础的触发+次数限制就够用
3. **属性转换**：属于职业特色机制，不需要作为核心层，放在Execution里做特殊算式即可
4. **复杂的护盾类型细分**：初期只做通用护盾，后续再拆分物理/魔法护盾

整体来看，铃兰之剑的战斗体系就是**单机回合制SRPG版的轻量化GAS**，它吸收了GAS的标签、效果、执行管线、事件触发的精髓，完全砍掉了网络相关的所有复杂度，非常适合我们的项目参考。

之前的解构已经覆盖了属性、伤害、技能、Buff、堆叠、装备、触发、时序八大核心主干，是完整的主链路模型。如果面向工业级落地、对标铃兰之剑的完整战斗深度，还有**12个细分维度**之前未提及，且全部属于「容易踩坑、但对长期数值平衡和玩法稳定性至关重要」的补充项，无任何重复内容。

## 一、状态与标签体系补全
### 1. 异常状态层级与免疫优先级
#### 铃兰实装规则
控制类状态严格分为三级，不同等级的覆盖、免疫规则完全独立：
- **软控（削弱层）**：减速、命中降低、攻击降低，不限制行动，仅削弱属性
- **硬控（行动限制层）**：定身、束缚、嘲讽，禁止移动/改变目标，但可释放技能、普攻
- **强控（完全失能层）**：眩晕、冰冻、石化，完全禁止所有行动，无法普攻/技能/反应

配套规则：
1. **高等级覆盖低等级**：施加强控时，自动覆盖目标身上同类型的硬控/软控，不会叠加存在
2. **免疫分级生效**：「免疫控制」免疫所有硬控+强控；「免疫行动限制」仅免疫定身类；「免疫眩晕」仅免疫单种状态
3. **控制递减规则**：部分PVE场景，连续对同一目标施加同类控制，持续时间会衰减，避免无限控死

#### 架构价值
对应Lite-GAS的**Tag层 + Stacking层**：给状态Tag增加「优先级权重」字段，解决多控制共存、免疫范围模糊的问题，避免后期出现“眩晕了还能放技能”的逻辑漏洞。

### 2. Tag分层分类与权限规则
#### 铃兰实装规则
所有游戏标签分为四大类，各自有独立的作用域与继承规则，不会混在一起：
- **伤害类型Tag**：物理、魔法、火焰、寒冰、真实，用于伤害判定、抗性匹配
- **状态Tag**：增益、减益、控制、无敌、不可选中，用于状态判定、互斥判断
- **阵营/身份Tag**：友方、敌方、召唤物、Boss、机械，用于目标筛选、效果判定
- **机制Tag**：可驱散、不可驱散、可反弹、不可触发，用于底层规则判定

配套规则：
1. 召唤物仅继承本体的「阵营Tag」和「机制Tag」，不继承状态Tag
2. 「无敌」「不可选中」属于最高权限机制Tag，会屏蔽所有伤害、选中类效果
3. 互斥Tag成对存在，比如「飞行」与「地面地形效果」互斥，不会触发地刺、毒沼

#### 架构价值
对应Lite-GAS的**Tag层**：避免Tag泛滥后语义混乱，AI生成代码时也有明确的分类标准，不会随便新增无意义标签。

### 3. 驱散与净化的精准结算规则
#### 铃兰实装规则
驱散不是单一效果，而是分类型、分优先级的精准操作，规则完全标准化：
1. **驱散类型拆分**
   - 随机驱散：随机移除1/2/3个增益/减益
   - 定向驱散：移除指定类型的效果（比如所有灼烧、所有控制）
   - 全体驱散：移除所有可驱散的增益/减益
   - 净化：仅移除控制类减益，不碰属性类减益
2. **驱散优先级**：高价值Buff优先被驱散（比如护盾、免疫），还是低价值优先，由配置决定
3. **不可驱散标记**：Boss核心Buff、剧情状态、装备常驻效果带「不可驱散」机制Tag，所有驱散效果对其无效
4. **驱散反向规则**：部分效果会「偷取增益」，属于驱散+给自己施加的组合效果，复用同一套驱散逻辑

#### 架构价值
对应Lite-GAS的**Effect层**：驱散是独立的一类Effect，不需要每个技能单独写逻辑，统一规则后可完全配置化。

## 二、战斗效果机制补全
### 4. 位移与地形交互的完整模型
#### 铃兰实装规则
位移分「主动位移」和「强制位移」两类，规则完全不同，是战棋策略性的核心：
- **主动位移**：角色自主移动、冲锋、突进，受地形、单位阻挡约束
  - 障碍、悬崖、水域等不可通行地形完全阻挡
  - 敌方单位阻挡路径，友方单位可穿越但不能停留
- **强制位移**：击退、拉扯、推送、换位，属于效果触发的位移
  - 部分强制位移可穿越障碍，不可通行地形也能生效
  - 撞击墙体/障碍物时，额外造成最大生命值百分比的撞墙伤害，并终止位移
  - 可推开路径上的敌方单位，单位卡位规则失效

配套规则：
1. 位移路径上的地形效果（毒沼、地刺、灼烧地面），每经过1格触发1次
2. 位移结束位置如果是不可通行地形，会回退到最近的合法格子
3. 「定身」状态禁止主动位移，但不禁止强制位移；「石化」状态禁止所有位移

#### 架构价值
对应Lite-GAS的**Effect层 + Targeting层**：位移是独立的Effect类型，需要和地形系统、Trigger系统打通，不能简单塞进伤害效果里。

### 5. 多段伤害与命中事件的拆分规则
#### 铃兰实装规则
连击、扫射、多段打击类技能，不是简单的“总伤害除以段数”，而是有严格的事件拆分规则：
1. **每段独立判定**：暴击、命中、闪避每段单独计算，可能第一段暴击、第二段不暴击
2. **被动触发规则**
   - 「受到伤害时触发」的被动，每段伤害都触发一次
   - 「被技能命中时触发」的被动，整个技能只触发一次
3. **护盾结算**：每段伤害单独扣护盾，而不是总伤害一次性扣除
4. **反伤/吸血规则**：按每段伤害单独计算，还是按总伤害计算，由效果配置决定

核心边界：**「技能命中」和「造成伤害」是两个独立事件**，不能合并。

#### 架构价值
对应Lite-GAS的**Trigger层 + Execution层**：明确事件粒度，避免多段技能触发被动次数异常的Bug，这是多段伤害类玩法最常见的数值漏洞。

### 6. 治疗与护盾的细分结算规则
#### 铃兰实装规则
治疗与护盾的复杂度不低于伤害，有独立的结算分层：
1. **治疗三类规则**
   - 基础治疗量 = 攻击×技能倍率 / 治疗强度
   - 受治疗提升：目标身上的「受到治疗量提升」效果，单独乘算
   - 治疗暴击：可暴击的治疗技能，单独走暴击判定，爆伤规则和伤害一致
   - 治疗衰减：AOE治疗对多个目标时，治疗量按目标数递减，避免群奶强度超标
   - 过量治疗：部分天赋支持过量治疗转化为护盾，转化率固定
2. **护盾四类规则**
   - 吸收优先级：先扣通用护盾，再扣物理/魔法专属护盾
   - 叠加规则：同类护盾取最大值+刷新时长，不同类护盾独立共存
   - 持续护盾：每回合回复固定值的护盾，有上限
   - 伤害吸收护盾：只吸收特定伤害类型的护盾，其余伤害穿透

#### 架构价值
对应Lite-GAS的**Execution层 + Effect层**：治疗和伤害共用同一套执行算式框架，不需要单独写一套管线，保持架构统一。

### 7. 死亡结算与濒死的完整链路
#### 铃兰实装规则
死亡是最容易出时序Bug的环节，铃兰有严格的结算窗口和链路：
1. **判定时机**：每次伤害结算完成后，立刻判定血量是否≤0，不是等阶段结束统一判定
2. **濒死窗口**：血量归零时，先触发「濒死」事件，所有回血、免死、假死效果在这个窗口生效
   - 如果回血后血量>0，取消死亡判定，单位存活
   - 如果没有免死效果，进入死亡流程
3. **死亡流程顺序**
   - 触发「即将死亡」事件，最后一次触发被动
   - 清除所有可驱散的Buff、Debuff，保留剧情/机制Tag
   - 触发「单位死亡」事件，击杀者触发击杀被动
   - 单位标记为死亡，移除实体控制权
4. **复活规则**：复活后仅恢复基础血量，不恢复之前的Buff、冷却、能量

#### 架构价值
对应Lite-GAS的**Trigger层 + Pipeline层**：明确死亡的时序窗口，避免“死了还能触发被动”“击杀回血没加上”这类时序Bug。

## 三、资源与时序体系补全
### 8. 双资源+冷却的管控与修改规则
#### 铃兰实装规则
技能资源不是只有“冷却”，而是「行动点数+能量+冷却回合」三层管控，修改规则严格统一：
1. **行动点（AP）**：每回合每单位默认1点，普攻/主动技能消耗1点，支援技能不消耗
   - 「再行动」效果是额外增加1点行动点，不是直接再动一次
   - 单回合行动点有上限，避免无限再行动
2. **能量（CP）**：技能的主要消耗资源，每回合固定回复，击杀/受击额外回复
   - 能量有上限，溢出作废
   - 「能量回复提升」效果只影响回合固定回复，不影响击杀额外回复
3. **冷却回合（CD）**：技能释放后，需要等待的回合数
   - 冷却减少效果：直接减少剩余回合数，多个冷却减少效果加算
   - 冷却刷新：直接清零冷却，可被「禁止刷新冷却」Tag屏蔽
   - 冷却增加：延长技能冷却，属于减益效果

#### 架构价值
对应Lite-GAS的**Ability层**：统一资源修改规则，避免后期出现“冷却减少能不能叠加”“能量溢出算不算”这类无明确规则的问题。

### 9. 行动顺序与速度调度的底层规则
#### 铃兰实装规则
回合内单位的出手顺序不是随机的，由速度属性严格决定：
1. **基础排序**：按单位的「速度」属性从高到低排序，速度相同按阵营、站位固定排序
2. **插队规则**
   - 「立即行动」效果：将单位插入到当前行动位的下一位，不是直接放到队首
   - 「延迟行动」效果：将单位移到当前回合队尾
3. **回合边界**：所有单位行动完毕，才算回合结束；回合内新增的召唤物，当回合不行动，下回合按速度插入排序
4. **再行动规则**：单位行动结束后获得再行动，会再次插入到当前回合的队尾，不是立刻再动

#### 架构价值
对应Lite-GAS的**Pipeline层**：明确回合调度的底层逻辑，是战斗确定性和回放的重要基石，也避免“再行动打乱回合顺序”的问题。

### 10. 召唤物实体的从属与联动规则
#### 铃兰实装规则
召唤物不是简单的额外单位，有完整的从属、继承、联动规则：
1. **属性继承**：召唤物的基础属性按百分比继承召唤者的属性，受召唤者的Buff影响吗？—— 只继承施加召唤时的快照属性，召唤者后续Buff变化不影响召唤物
2. **行为规则**：召唤物有独立AI，自动行动，不受玩家控制
3. **归属判定**
   - 召唤物击杀单位，击杀者算召唤者本体，触发本体的击杀被动
   - 召唤物死亡，不视为召唤者死亡，不触发“友军死亡”类被动
   - 召唤物数量有上限，召唤新的会顶替最早的
4. **生命周期**：召唤者死亡，所有召唤物立即消失；召唤持续回合结束，自动消失

#### 架构价值
对应Lite-GAS的**Effect层 + Tag层**：召唤是一类特殊Effect，需要实体从属关系（Bevy Relationship）和归属Tag支撑，明确规则后不会出现“召唤物击杀不算数”的逻辑问题。

## 四、数值与AI底层补全
### 11. 数值边界与取整的强制规则
#### 铃兰实装规则
这是最细节但最影响稳定性的部分，所有数值计算都遵循统一的边界规则：
1. **取整规则**：所有伤害、治疗、属性值统一向下取整，最小为1（真实伤害可到0）
2. **属性下限**：防御最低为0，不会出现负数；移动范围最低为1，不会0移动
3. **概率上限**：暴击率上限95%，闪避率上限80%，命中率上限100%，不存在100%必闪
4. **百分比边界**：减伤最高90%，不会出现无敌减伤；易伤无上限但乘算递减
5. **结算顺序**：所有百分比计算完成后，再统一取整，避免分步取整导致的数值偏差

#### 架构价值
对应Lite-GAS的**Attribute层 + Execution层**：从底层锁死数值边界，避免后期出现“防御为负伤害越打越高”“满暴满闪破坏平衡”的数值灾难。

### 12. AI决策的输入边界与仿真对齐
#### 铃兰实装规则
敌方AI的决策输入有严格边界，既保证策略性，又不会“作弊”，且和玩家侧的规则完全对齐：
1. **AI可读取信息**：所有可见单位的位置、血量、属性、Buff、技能冷却，和玩家看到的信息完全一致
2. **AI不可读取信息**：玩家下一步操作、未暴露的单位、随机数种子，禁止作弊式预判
3. **决策层级**：优先级固定为「保命>完成目标>输出>辅助」，不同单位有不同的行为权重
4. **仿真决策**：AI决策时会模拟技能释放的结果（伤害、命中），和真实战斗使用同一套核心规则，不会出现“AI算的伤害和实际不一样”的情况

#### 架构价值
对应Lite-GAS的**仿真层 + Replay层**：核心规则天然支持AI离线仿真，和玩家侧共用同一套执行管线，不需要单独写AI逻辑，完美契合“仿真优先”的架构原则。
