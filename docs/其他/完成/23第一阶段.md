# 第一优先级：打通完整战斗闭环

先确保玩家可以：

```text
进入战斗
 ↓
轮到角色
 ↓
移动
 ↓
选择技能
 ↓
选择目标
 ↓
造成伤害
 ↓
Buff生效
 ↓
结束回合
 ↓
敌人行动
 ↓
战斗胜利/失败
```

如果还没有完全跑通：

```text
移动
攻击
伤害
死亡
回合切换
胜负判定
```

那么其它功能全部暂停。

因为这是：

```text
Vertical Slice（纵向切片）
```

而不是：

```text
Horizontal Slice（横向堆系统）
```

对于独立开发最重要。

---

# 第二优先级：Character → Skill → Buff 联动

你的目录已经有：

```text
character
skill
buff
```

下一步应该让三者真正连接起来。

例如：

```rust
Character
 ├─ Attributes
 ├─ Skills
 └─ Buffs
```

技能释放：

```rust
FireBall
    ↓
ApplyDamage
    ↓
ApplyBurnBuff
```

Buff：

```rust
Burn
    ↓
TurnStart
    ↓
Damage
```

此时形成：

```text
Character
 ↕
Skill
 ↕
Buff
```

核心战斗玩法开始出现。

---

# 第三优先级：数据驱动

这是我认为你项目接下来最重要的一步。

不要再写：

```rust
if skill_id == 1 {
    ...
}
```

或者：

```rust
match skill {
    FireBall => ...
}
```

开始建立：

```text
content/
 ├─ skills/
 ├─ buffs/
 ├─ characters/
 └─ maps/
```

例如：

```yaml
fireball:
  cost: 2
  range: 3
  effects:
    - damage: 120%
    - add_buff: burn
```

或者：

```json
{
  "id":"fireball",
  "range":3,
  "effects":[]
}
```

因为以后：

```text
100角色
500技能
300Buff
```

不可能靠硬编码维护。

---

# 第四优先级：AI行动系统

当战斗闭环完成后。

加入：

```text
AI Turn
```

最简单版本：

```rust
攻击范围内有敌人
    -> 攻击

否则
    -> 靠近最近敌人
```

不要上来：

```text
GOAP
BT
Utility AI
HTN
```

简单规则就够了。

你现在最需要的是：

```text
能玩
```

而不是：

```text
很聪明
```

---

# 第五优先级：胜负系统

很多人会忘。

其实非常重要。

新增：

```text
battle_result
```

例如：

```rust
Victory
Defeat
Retreat
```

触发：

```text
全部敌人死亡
↓
Victory
↓
奖励
↓
返回地图
```

这样第一次形成完整游戏循环。

# 第七优先级：地图内容编辑

你前面说：

> 目前棋子还是图块硬编码

我认为这一步应该在战斗闭环完成后立刻做。

目标：

```text
地图配置
↓
出生点
↓
敌人配置
↓
胜利条件
```

例如：

```yaml
battle_001:

  player_spawn:
    - [3,5]
    - [4,5]

  enemies:
    - goblin
    - goblin
    - boss

  victory:
    kill_all
```

这样你以后制作关卡：

```text
新增数据
=
新增关卡
```

而不是：

```text
新增代码
=
新增关卡
```
