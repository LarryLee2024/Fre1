如果你的目标是：

Bevy 0.18+
长期维护（3~5年以上）
AI大量参与开发
SRPG（类似《铃兰之剑》《FFT》《八方旅人》）
已经有几百个测试
后续会有数万行代码

那么我要先说一个很多人忽略的结论：

architecture.md、domain_rules.md、test_spec.md 不是给人看的。

它们首先是给 AI 看的，其次才是给人看的。

所以很多网上的示例其实不合格。

因为它们写成：

# Architecture

本项目采用DDD思想。

本项目采用ECS架构。

本项目遵循SOLID原则。

这种东西对于 AI 几乎没有约束力。

AI真正需要的是：

允许做什么
禁止做什么
发现什么情况必须停止
修改代码时优先遵守什么

这种规则文档。

一、architecture.md（架构圣经）

这是最高优先级文档。

所有 Agent 都必须读。

# Architecture

Version: 1.0

本文件优先级高于任何代码实现。

当代码与本文件冲突时：
优先认为代码错误，而不是文档错误。

# 模块边界

Battle：

负责：

- 战斗
- 伤害
- 回合

禁止：

- 修改背包
- 修改任务
- 修改UI

Inventory：

负责：

- 物品
- 背包

禁止：

- 直接修改角色属性

必须通过：

EquipmentMessage
ItemConsumedMessage

通知其他模块。

# Definition / Instance

Definition：

不可变配置

例如：

JobDefinition
SkillDefinition
ItemDefinition

Instance：

运行时状态

例如：

CharacterStats
SkillCooldown
Inventory

禁止修改 Definition。

---

# Rule / Content

代码：

负责规则

配置：

负责内容

允许：

新增配置

禁止：

修改伤害代码

1 Architecture
2 Domain Rules
3 Test Spec
