对于你的项目（Bevy 0.18 + 长期连载SRPG + 数据驱动 + AI协作 + 未来可能几十万字文本）来说：

> **Fluent（fluent-rs）目前仍然是 Rust/Bevy 生态最接近“最佳实践”的国际化方案。**

# 为什么 Fluent 是目前最好的

## Fluent支持复杂语法

例如：

```ftl
hp-low =
    { $value ->
        [0] 已死亡
        [1] 仅剩1点生命
       *[other] 剩余 {$value} 点生命
    }
```

代码：

```rust
tr("hp-low", fluent_args! {
    "value" => hp
})
```

---

传统JSON：

```json
{
  "hp_low": "剩余 {value} 点生命"
}
```

做不了：

```text
复数
性别
语法变体
地区差异
```

---

# Fluent特别适合JRPG/SRPG

因为你未来一定会有：

```text
角色名
技能描述
Buff说明
任务文本
剧情对话
UI
系统提示
```

---

例如：

```ftl
skill-fireball-desc =
    对目标造成 {$damage} 点火焰伤害。
```

---

代码：

```rust
tr(
    "skill-fireball-desc",
    fluent_args! {
        "damage" => damage
    }
)
```

---

这是现代游戏行业主流做法。

---

# 但是 Fluent 最大问题

不是技术。

而是：

```text
翻译人员不会写FTL
```

---

例如策划看到：

```ftl
quest-complete =
    恭喜完成任务 {$quest_name}
```

还能接受。

---

看到：

```ftl
quest-complete =
    { $count ->
        [1] 完成一个任务
       *[other] 完成 {$count} 个任务
    }
```

开始崩溃。

---

所以真正的大项目不会直接让策划维护FTL。
