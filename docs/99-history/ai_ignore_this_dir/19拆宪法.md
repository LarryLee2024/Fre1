# 1. 文档分级（非常重要）

很多大型项目最后失败不是因为文档少。

而是：

```text
规则
建议
经验

混在一起
```

导致没人知道什么必须遵守。

建议增加：

```text
P0 = Constitution
P1 = ADR
P2 = Standard
P3 = Guide
P4 = Notes
```

例如：

```text
docs/

00-governance/      # P0

adr/                # P1

standards/          # P2

guides/             # P3

notes/              # P4
```

这样 AI 和人都知道：

```text
Constitution > ADR > Standard > Guide
```

---

# 2. 建立 Rule ID 体系

```text
禁止XXX
禁止YYY
```

# 我的建议

分级处理。

## 第一类：必须写 YAML

这些是「治理类文档」。

例如：

```text
00-governance/
adr/
standards/
```

必须有。

例如：

```yaml
---
id: ADR-065
title: Map Pipeline
status: accepted
owner: architect
created: 2026-06-22
updated: 2026-06-22
tags:
  - map
  - content
  - architecture
supersedes:
  - ADR-041
---
```

因为需要：

* 搜索
* 索引
* 生命周期管理
* AI检索
* ADR关系图

---

## 第二类：建议写极简 YAML

例如：

```text
capabilities/
domains/
```

不要写一堆。

只保留：

```yaml
---
id: DOM-COMBAT
status: stable
---
```

最多：

```yaml
---
id: DOM-COMBAT
title: Combat Domain
status: stable
depends_on:
  - CAP-ATTRIBUTE
  - CAP-EFFECT
---
```

结束。

例如：

```text
guides/
tutorials/
notes/
examples/
```

直接正文。

例如：

```markdown

...
```

即可。
