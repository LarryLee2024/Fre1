# 第一原则：代码中禁止出现用户可见文本

禁止：

```rust
Text::new("Fireball")
```

```rust

```

```rust
error!("角色不存在")
```

---

必须：

```rust
TextKey("ui.battle.end_turn")
```

```rust
TextKey("ability.fireball.name")
```

```rust

```

---

代码里只能出现：

```text
Key
```

```text
中文

```

```text
content/
├── ability/
├── buff/
├── item/
├── quest/
└── story/

localization/
├── zh-CN/
├── en-US/
└── ja-JP/

infrastructure/
└── localization/
    ├── LocalizationPlugin
    ├── LocalizationDatabase
    ├── LocalizationLoader
    ├── LocalizationValidator
    ├── LocalizationAudit
    └── LocalizationReport
```

而且对于你的项目，我还会额外定一条架构宪法：

```text
任何用户可见文本禁止进入Rust代码。

代码层只能出现 LocalizationKey。

所有文本均视为 Content Asset。
```

这一条能避免未来几十万行代码中最难清理的国际化技术债。真正的大型 RPG（尤其是剧情量大的 SRPG）最后本质上是在管理几十万到上百万行文本资产，而不是在管理代码。国际化系统应该按“内容资产管理系统”设计，而不是按“字符串翻译工具”设计。

有，而且下面这些反而是很多「架构文章不写，但项目做到 20~50 万行一定会踩」的东西。

我不重复 Key、Fluent、LocalizationPlugin 这些基础内容，只讲更深层的设计。

# 1. 不要把文本Key写成字符串

很多项目：

```rust
tr("ability.fireball.name")
```

看起来没问题。

50万行以后：

```rust
tr("ability.firebal.name")
```

少一个 `l`

编译通过。

运行时爆炸。

---

应该生成：

```rust
LocKey::AbilityFireballName
```

或者：

```rust
loc::ability::fireball::NAME
```

---

来源：

```yaml
ability.fireball.name
```

自动代码生成：

```rust
pub const ABILITY_FIREBALL_NAME: LocKey;
```

---

这样：

```rust
loc::ability::fireball::NAME
```

编译期检查。

---

# 2. Localization必须纳入CI

```ftl

  Deal {$damage} damage
```

# 我觉得最容易被忽略，但价值最高的三项

如果让我在你的项目里额外强制写进《AI开发宪法》：

```text
1. Localization Key 自动生成 Rust 常量
2. Fake Locale 自动检测硬编码文本
3. Replay/Event 只存 Key+参数，不存最终翻译文本
```

这三个东西，前期看不到收益。

但等你做到：

```text
10万+ Localization Key
几十万行剧情
多语言
DLC
Mod
Replay
```

的时候，会直接决定国际化系统能不能继续维护下去。
最后一次补充的话，我不会再讲技术实现，而是讲 **真正决定 50 万行项目能不能活到第 5 年的国际化治理问题**。

这些东西很多团队做到上线都没意识到。

# 13. 最重要的一条：Localization 视为代码

很多团队：

```text
代码
严肃管理

翻译
随便改
```

---

结果：

```text
线上Bug
```

最后来源是：

```ftl
变量名写错
Key删错
格式损坏
```

---

对于大型项目：

```text
Localization
Content
Config
Code
```

四者地位应该完全平等。

---

如果让我给一个 **50万行 Bevy SRPG 项目国际化宪法最终版**，最后会多一条很多人想不到的规则：

```text
Localization 是一种可执行资源（Executable Asset），
而不是文本文件。
```

因为它会影响：

```text
编译
CI
运行时
UI
Replay
存档
Mod
DLC
配音
AI翻译流水线
```

当你用这个视角看待国际化时，很多设计决策会自然变得正确。真正的大项目里，Localization 更像一个独立子系统，而不是一个翻译目录。
