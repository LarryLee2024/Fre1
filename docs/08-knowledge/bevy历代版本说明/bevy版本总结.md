# Bevy 版本总结

## 内容编写标准

| 标准项 | 要求 |
| --- | --- |
| **术语使用** | 保留英文技术术语（如 ECS、Query、Transform、GLTF、PBR 等），中文描述核心概念 |
| **描述深度** | 每个特性需包含：定义/概念、核心组件、使用方式、API 示例（如有） |
| **分类粒度** | 大类 → 细类 → 子项，三级分类，覆盖引擎所有核心模块 |
| **应用建议** | 具体到 SRPG 场景，说明如何用于战棋开发，而非泛泛而谈 |
| **版本对比** | 标注相对于前一版本的新增/变更/废弃，便于追踪演进 |

## Bevy 0.1 — 首次发布：开创性的 ECS 游戏引擎

| 大类 | 项目 | 详细描述 | SRPG项目应用建议 |
| --- | --- | --- | --- |
| **核心架构** | ECS架构体系 | **实体组件系统（Entity Component System）**：<br>- **实体（Entity）**：唯一 ID，作为组件容器，不包含数据或逻辑<br>- **组件（Component）**：普通 Rust 结构体，存储数据，如 `Position(f32, f32)`、`Velocity(f32)`<br>- **系统（System）**：普通 Rust 函数，在特定组件集上运行逻辑，自动推断读/写依赖<br>**架构优势**：<br>- 数据与逻辑分离，强制解耦设计<br>- 缓存友好：组件紧密打包（Struct of Arrays 布局）<br>- 天然并行：系统声明依赖后自动调度<br>**对比其他 ECS**：<br>- Specs：复杂生命周期和宏<br>- Shipyard：更多抽象层<br>- Bevy：纯 Rust 数据类型，零额外学习成本 | **战棋实体建模**：<br>- 角色实体：`Unit`（阵营）、`Position`（格子坐标）、`Health`（生命值）、`ActionPoints`（行动点）<br>- 地图实体：`Tile`（地形类型）、`Terrain`（属性）、`Occupant`（占用者）<br>- 技能实体：`Skill`（效果）、`Cooldown`（冷却）、`Range`（范围）<br>**系统设计**：<br>- `movement_system`：处理单位移动<br>- `combat_system`：计算伤害和状态<br>- `ai_system`：敌方单位决策 |
| **核心架构** | 函数系统机制 | **函数系统（Function Systems）**：<br>- 直接将 Rust 函数作为系统，无需实现 trait 或宏<br>- `.system()` 方法将函数指针转换为 `Box<dyn System>`<br>- 编译期通过 `IntoQuerySystem` trait 推断依赖<br>**系统签名**：<br>- 无参系统：`fn system()`<br>- 查询系统：`fn system(query: Query<(&A, &mut B)>)`<br>- 资源系统：`fn system(time: Res<Time>, score: ResMut<Score>)`<br>- 混合系统：同时访问查询和资源 | **战棋系统实现**：<br>- `fn move_unit(mut query: Query<(&mut Position, &MovementSpeed)>, input: Res<Input<KeyCode>>)`<br>- `fn apply_damage(mut query: Query<(&mut Health, &DamageReceiver)>)`<br>- `fn update_turn(timer: Res<TurnTimer>, mut state: ResMut<GameState>)` |
| **核心架构** | 查询系统 | **Query<T> 查询**：<br>- 精确控制迭代范围和访问权限<br>- `&T`：只读访问<br>- `&mut T`：可变访问<br>- `Query<Entity>`：获取实体 ID<br>**过滤器**：<br>- `With<T>`：包含组件 T<br>- `Without<T>`：不包含组件 T<br>- `Added<T>`：本次更新新添加<br>- `Mutated<T>`：本次更新已修改<br>- `Changed<T>`：添加或修改<br>**多查询**：<br>- 同一系统可持有多个 Query<br>- 避免借用冲突 | **战棋查询**：<br>- 查询所有可移动单位：`Query<(&Unit, &mut Position), With<ActionPoints>>`<br>- 查询攻击范围内敌人：`Query<&Unit, Without<Ally>>`<br>- 检测新回合：`Query<Added<TurnStarted>>`<br>- 碰撞检测：多查询遍历攻击者和防御者 |
| **核心架构** | 阶段系统 | **阶段（Stages）**：<br>- 按顺序运行系统组的机制<br>- 默认阶段："update"<br>- 自定义阶段：`add_stage_after("update", "do_things")`<br>**系统执行顺序**：<br>- 同一阶段内系统可并行<br>- 不同阶段按顺序执行<br>- 阶段间可设置依赖 | **战棋回合流程**：<br>- `input_stage`：处理玩家输入<br>- `movement_stage`：执行移动逻辑<br>- `combat_stage`：结算战斗<br>- `ai_stage`：敌方 AI 决策<br>- `render_stage`：更新渲染 |
| **核心架构** | 命令系统 | **Commands**：<br>- 延迟执行的 World 和 Resource 更改<br>- 在当前阶段结束时统一应用<br>- 避免系统间直接借用冲突<br>**常用命令**：<br>- `commands.spawn((ComponentA, ComponentB))`<br>- `commands.despawn(entity)`<br>- `commands.insert_resource(resource)` | **战棋实体操作**：<br>- 生成新单位：`commands.spawn((Unit::new(), Position::default()))`<br>- 移除死亡单位：`commands.despawn(dead_entity)`<br>- 状态变更：`commands.insert_resource(TurnState::PlayerTurn)` |
| **渲染系统** | 渲染图架构 | **渲染图（Render Graph）**：<br>- 节点化渲染逻辑，编码原子单元<br>- 支持多线程并行渲染<br>**内置节点**：<br>- `CameraNode`：相机处理<br>- `PassNode`：渲染通道<br>- `RenderResourcesNode`：资源绑定<br>- `SharedBuffersNode`：共享缓冲区<br>- `TextureCopyNode`：纹理拷贝<br>- `WindowSwapChainNode`：交换链<br>- `WindowTextureNode`：窗口纹理<br>**子图**：<br>- 2D 渲染子图<br>- 3D 渲染子图<br>- UI 渲染子图 | **战棋渲染管线**：<br>- 地形层：瓦片渲染<br>- 单位层：精灵/模型渲染<br>- 高亮层：移动范围、攻击范围<br>- UI 层：界面元素<br>- 自定义节点：添加战棋特效节点 |
| **渲染系统** | 数据驱动着色器 | **着色器绑定**：<br>- 组件派生 `RenderResources`，自动复制到 GPU<br>- 支持 GLSL 着色器<br>**着色器定义**：<br>- `ShaderDefs` 派生宏<br>- `#[shader_def]` 属性标记<br>- 条件编译：`#ifdef MYMATERIAL_ALWAYS_BLUE`<br>**布局反射**：<br>- 从 SpirV 自动反射数据布局<br>- 简化自定义着色器创建 | **战棋自定义着色器**：<br>- 移动范围高亮：蓝色半透明覆盖<br>- 攻击范围高亮：红色半透明覆盖<br>- 选中单位：边框闪烁效果<br>- 地形效果：森林、山脉、水域着色器 |
| **渲染系统** | 3D渲染特性 | **GLTF 模型加载**：<br>- 加载 GLTF 文件作为网格资源<br>- 支持材质和纹理绑定<br>**绘制顺序**：<br>- 不透明材质：前→后（Early-Z 优化）<br>- 透明材质：后→前（正确混合）<br>**父子层级**：<br>- 父级变换传递到后代<br>- `with_children` 构建层级树<br>**MSAA 抗锯齿**：<br>- 多重采样抗锯齿<br>- 可配置采样数（如 8x） | **战棋3D扩展**：<br>- 3D 单位模型：角色、怪物<br>- 3D 地形：山丘、建筑<br>- 父子层级：武器挂载点<br>- 抗锯齿：提升视觉质量 |
| **渲染系统** | 2D精灵系统 | **精灵渲染**：<br>- 单个图像渲染为精灵<br>- `SpriteComponents` 组件<br>**精灵表（Texture Atlas）**：<br>- 纹理图集存储多个帧<br>- `SpriteSheetComponents` 组件<br>- `TextureAtlas::from_grid` 从网格生成<br>**动态图集生成**：<br>- 运行时合并单独精灵文件<br>- `TextureAtlasBuilder` 构建器 | **战棋2D渲染**：<br>- 单位精灵：每个角色独立纹理<br>- 地图瓦片：精灵表存储地形贴图<br>- 动画帧：攻击、死亡动画<br>- 动态合并：运行时构建图集 |
| **UI系统** | Bevy UI架构 | **ECS驱动UI**：<br>- 直接使用核心 ECS/层级/变换/事件/资源系统<br>- 自动获得：场景热重载、异步纹理加载、变更检测<br>**构建块**：<br>- `Node`：矩形区域（宽度/高度）<br>- `Transform`：定位组件<br>- `Style`：Flex 属性控制<br>**组件Bundle**：<br>- `NodeComponents`：完整节点组件包<br>- `TextComponents`：文本组件包<br>- `ImageComponents`：图像组件包 | **战棋UI框架**：<br>- 回合信息面板：当前回合、行动点数<br>- 单位状态面板：生命值、攻击力、防御力<br>- 技能面板：技能图标和冷却<br>- 地图信息：地形属性显示 |
| **UI系统** | Flexbox布局 | **布局引擎**：<br>- 基于 Stretch 库的纯 Rust Flexbox 实现<br>- 符合 Flexbox 规范的 2D 定位算法<br>**定位方式**：<br>- 相对定位：默认相对于父节点<br>- 绝对定位：`PositionType::Absolute`<br>- 父子层级：子节点相对于父节点缩放<br>**Flex属性**：<br>- `justify_content`：主轴对齐<br>- `align_items`：交叉轴对齐<br>- `flex_direction`：主轴方向<br>- `size`：宽度/高度 | **战棋UI布局**：<br>- 主菜单：居中对齐<br>- 战斗界面：顶部回合信息，左侧单位列表，底部技能栏<br>- 弹窗：居中显示<br>- 适配不同分辨率 |
| **UI系统** | 交互事件 | **Interaction组件**：<br>- 跟踪交互状态<br>- `Interaction::Clicked`：已点击<br>- `Interaction::Hovered`：悬停<br>- `Interaction::None`：无交互<br>**变更检测**：<br>- `Mutated<Interaction>`：状态变化时触发<br>- 仅在状态变化时运行系统 | **战棋按钮交互**：<br>- 攻击按钮：点击后进入攻击模式<br>- 移动按钮：点击后显示移动范围<br>- 待机按钮：结束当前单位行动<br>- 悬停提示：显示技能描述 |
| **场景系统** | 场景概念 | **场景（Scene）**：<br>- 实体和组件的集合<br>- 可多次生成到 World<br>**生成方式**：<br>- `load`：保留实体 ID（用于存档）<br>- `instance`：使用新 ID（用于多实例）<br>**场景文件**：<br>- RON（Rusty Object Notation）格式<br>- 扁平实体列表<br>- 实体 ID 可选，加载时自动生成 | **战棋配置存储**：<br>- 地图配置：地形、单位初始位置<br>- 单位数据：属性、技能、装备<br>- 关卡配置：胜利条件、敌人配置<br>- 模板系统：可复用的地图模板 |
| **场景系统** | 场景热重载 | **热重载机制**：<br>- 运行时自动应用场景文件变更<br>- 无需重启或重新编译<br>- 即时反馈<br>**工作原理**：<br>- 基于属性系统（Properties）的序列化/反序列化<br>- 支持运行时打补丁 | **战棋调试**：<br>- 修改地图文件后实时看到效果<br>- 调整单位属性后立即测试<br>- 快速迭代关卡设计 |
| **场景系统** | 场景序列化 | **保存到场景**：<br>- `Scene::from_world(&world, &registry)`<br>- `scene.serialize_ron(&registry)`<br>**加载场景**：<br>- `asset_server.load("scene.scn")`<br>- `scene_spawner.load(handle)` / `scene_spawner.instance(handle)` | **战棋存档系统**：<br>- 保存当前游戏状态到场景文件<br>- 读档恢复游戏状态<br>- 多存档槽位管理<br>- 自动存档功能 |
| **属性系统** | 反射机制 | **bevy_property crate**：<br>- 为 Rust 增加动态反射能力<br>- 通过字符串字段名获取/设置属性值<br>**派生宏**：<br>- `#[derive(Properties)]` 派生属性<br>- 字段必须实现 `Property` trait<br>**核心方法**：<br>- `set_prop_val::<T>("field", value)`：类型安全设置<br>- `set_prop("field", &dyn Property)`：动态设置<br>- `prop_val::<T>("field")`：获取属性值<br>- `apply(&patch)`：应用属性补丁 | **战棋配置系统**：<br>- 单位属性动态访问：运行时修改生命值、攻击力<br>- 配置补丁：用新值修补现有配置<br>- 技能效果：动态修改单位属性<br>- 状态效果：临时属性修改 |
| **属性系统** | 序列化支持 | **serde 集成**：<br>- 实现 Properties 的类型可序列化<br>- DynamicProperties 可反序列化<br>**往返一致性**：<br>- 序列化 → 反序列化 → 属性一致<br>- 支持存档/读档 | **战棋存档**：<br>- 单位状态序列化<br>- 游戏进度保存<br>- 配置文件序列化<br>- 网络同步数据序列化 |
| **事件系统** | 双缓冲事件 | **事件机制**：<br>- 双缓冲 EventBuffer 实现高效生产/消费<br>- 零分配消费（EventReader 仅跟踪数组索引）<br>**使用方式**：<br>- `app.add_event::<MyEvent>()` 注册事件<br>- `ResMut<Events<MyEvent>>` 发送事件<br>- `EventReader<MyEvent>` 消费事件<br>**限制**：<br>- 每个系统只有一次机会接收事件<br>- 下次更新时丢失未消费事件 | **战棋事件**：<br>- `UnitMoved`：单位移动事件<br>- `UnitAttacked`：单位攻击事件<br>- `UnitDied`：单位死亡事件<br>- `TurnEnded`：回合结束事件<br>- `SkillUsed`：技能使用事件<br>**事件驱动解耦**：<br>- 移动系统发送 UnitMoved<br>- 音效系统监听播放音效<br>- UI 系统监听更新显示 |
| **资源系统** | Assets集合 | **Assets<T>**：<br>- 类型化资源集合<br>- Handle<T> 引用机制<br>- 支持创建、访问、事件通知<br>**资源事件**：<br>- `AssetEvent::Created`：资源创建<br>- `AssetEvent::Modified`：资源修改<br>- `AssetEvent::Removed`：资源移除 | **战棋资源管理**：<br>- 地图数据：`Assets<MapData>`<br>- 单位配置：`Assets<UnitConfig>`<br>- 技能配置：`Assets<SkillConfig>`<br>- 资源热重载：修改配置后自动更新 |
| **资源系统** | 异步加载 | **AssetServer**：<br>- 异步并行加载文件<br>- `load("file.png")` 返回 Handle<br>- Handle 可立即使用，资源就绪后自动可用<br>**同步加载**：<br>- `load_sync()` 阻塞当前系统<br>- 适用于必须立即访问资源的场景<br>**文件夹加载**：<br>- `load_asset_folder()` 递归加载整个文件夹 | **战棋资源加载**：<br>- 地图数据异步加载<br>- 单位精灵/模型异步加载<br>- 音效资源异步加载<br>- 启动时预加载常用资源 |
| **资源系统** | 热重载 | **热重载机制**：<br>- `watch_for_changes()` 启用资源变更检测<br>- 文件修改后自动重新加载<br>**扩展性**：<br>- 实现 `AssetLoader` trait 添加新资源类型<br>- `app.add_asset_loader::<T, Loader>()` 注册加载器 | **战棋开发**：<br>- 修改单位配置后自动更新<br>- 调整地图数据后实时预览<br>- 自定义资源类型：技能配置、装备配置 |
| **音效系统** | 音频播放 | **基础音频**：<br>- 加载音频文件（如 MP3）<br>- 异步播放：加载完成后自动播放<br>- 立即播放：已有 AudioSource 时直接播放<br>**AudioOutput**：<br>- `play(handle)`：异步播放<br>- `play_source(source)`：立即播放 | **战棋音效**：<br>- 攻击音效：刀剑、魔法<br>- 移动音效：脚步声<br>- UI 反馈音效：按钮点击、菜单切换<br>- 背景音乐：战斗、探索 |
| **插件架构** | 模块化设计 | **Plugin trait**：<br>- 所有功能实现为 Plugin<br>- `build(&mut App)` 方法注册系统和资源<br>**默认插件**：<br>- `CorePlugin`：核心功能<br>- `InputPlugin`：输入处理<br>- `WindowPlugin`：窗口管理<br>- `RenderPlugin`：渲染<br>- `UiPlugin`：UI<br>- `add_default_plugins()` 添加所有默认插件<br>**自定义插件**：<br>- 创建 struct 实现 Plugin trait<br>- 按需添加/移除 | **战棋模块化**：<br>- `CombatPlugin`：战斗系统<br>- `MovementPlugin`：移动系统<br>- `AiPlugin`：AI 系统<br>- `UiPlugin`：UI 系统<br>- `AudioPlugin`：音频系统<br>- 可独立开发、测试、替换<br>- 不同项目可组合不同插件 |
| **编译优化** | 快速编译配置 | **编译时间目标**：<br>- 0-1 秒：理想<br>- 1-3 秒：不错<br>- 3-5 秒：烦人<br>- 5-10 秒：痛苦<br>- 10+ 秒：不可用<br>**配置三要素**：<br>- **LLD 链接器**：比默认链接器快很多<br>- **Nightly Rust 编译器**：最新性能优化<br>- **泛型共享**：预编译泛型代码，避免重复编译<br>**迭代编译时间**：<br>- Linux：~0.8-3.0 秒<br>- Windows：~1.5-3.0 秒<br>- MacOS：较慢（LLD 版本限制） | **开发效率**：<br>- 快速迭代：修改代码后秒级编译<br>- 多平台开发：Linux 优先，Windows 次之<br>- CI/CD：快速构建<br>- 原型开发：快速验证想法 |
| **跨平台** | 平台支持 | **当前支持**：<br>- Windows、MacOS、Linux<br>**底层依赖**：<br>- winit：跨平台窗口和输入<br>- wgpu：跨平台渲染（Vulkan/DX12/Metal）<br>**未来计划**：<br>- Android<br>- iOS<br>- Web（WebAssembly） | **战棋发布**：<br>- PC 平台首发（Windows/Mac/Linux）<br>- 移动端后续支持（iOS/Android）<br>- Web 版本可能性<br>- 跨平台输入适配 |
| **未来规划** | 后续版本预告 | **PBR 渲染**：<br>- 物理基础渲染着色器<br>- 阴影系统<br>- 更多光照选项<br>**编辑器**：<br>- 在引擎内部构建<br>- 可视化场景编辑<br>- 属性动画工具<br>**动画系统**：<br>- 代码优先动画<br>- 时间轴系统<br>**物理引擎**：<br>- 可插拔物理接口<br>- nphysics/ncollide 后端<br>**其他**：<br>- Canvas 绘图 API<br>- 更好的场景格式<br>- 文档完善 | **战棋扩展方向**：<br>- 3D 战棋的 PBR 渲染<br>- 可视化地图编辑器<br>- 单位骨骼动画<br>- 碰撞检测和物理效果<br>- 动画时间轴系统<br>- 动态插件加载（热重载） |
