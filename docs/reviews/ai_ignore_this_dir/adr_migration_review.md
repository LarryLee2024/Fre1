## 问题A — 阻塞性：ADR-001 与 ADR-004 中 `assets/rules/*.ron` 的目标路径冲突

**严重程度**：阻塞性
**受影响ADR**：ADR-001（第5阶段）、ADR-004（第3.2节）
**问题类型**：跨ADR矛盾

### 问题描述
ADR-001 第5阶段迁移对照表（第1070行）将 `assets/rules/*.ron` 映射至 `content/formulas/*.ron`：

```
| `assets/rules/*.ron` | `content/formulas/*.ron` | 1 | ModifierRuleRegistry | → 权威路径参见 ADR-004 |
```

ADR-004 第3.2节的目标目录结构（第129行）将同源文件映射至另一目标路径：

```
├── modifiers/        ← assets/rules/*.ron（修饰规则）
```

两处路径存在冲突：`content/formulas/` 与 `content/modifiers/` 不一致。第5阶段全局替换规则（第5阶段，第1099行）会将 `assets/rules/` 替换为 `content/formulas/`，但若以ADR-004的路径为准，则替换目标应为 `content/modifiers/`。

### 违规项
- **宪法第1.1.4条**：文档之间不得存在冲突
- **架构原则**：所有ADR文档必须构成一致、无矛盾的整体

### 升级上报
→ **请 @architect 介入**，确定标准目标路径，并更新其中一份ADR以保持一致。

## 问题B — 高优先级：全部8份ADR引用了过时的宪法文件（`docs/AI开发宪法.md`）

**严重程度**：高优先级
**受影响ADR**：ADR-001（第63行）、ADR-002（第38行）、ADR-003（第1510行）、ADR-004（第677行）、ADR-005、ADR-006（第41行）、ADR-007、ADR-008（第41行），以及所有 `ai_ignore_this_dir/` 目录下的ADR
**问题类型**：引用过时

### 问题描述

若构建人员或审查人员仅查阅ADR中的宪法引用，会读取到错误文档，可能遗漏已更新的规则，或引用已变更的条款。

### 违规项
- **宪法第1.1.1条**：文档引用必须指向最新版本

### 升级上报
→ **请 @architect 介入**，将所有ADR的宪法引用更新为 `docs/AI开发宪法完整版.md`。

## 问题C — 高优先级：宪法章节编号过时（v1.6语境下使用v1.4编号）

**严重程度**：高优先级
**受影响ADR**：ADR-002（第972行）、ADR-008（第925行）
**问题类型**：编号不匹配

### 问题描述
ADR-002 的合规性总结（第972行）引用了宪法章节 `14.8.1`、`14.8.2`、`14.9.1`、`14.9.3`、`14.10.1`、`14.10.3`。ADR-008 的宪法引用同理使用了 `14.x.x` 编号体系。

在 `docs/AI开发宪法完整版.md`（v1.6版本）中，对应章节的实际编号如下：

| ADR中使用的v1.4编号 | v1.6权威实际编号 |
|---|---|
| §14.8.1 | §13.8.1 |
| §14.8.2 | §13.8.2 |
| §14.9.1 | §13.9.1 |
| §14.9.3 | §13.9.3 |
| §14.10.1 | §13.10.1 |
| §14.10.3 | §13.10.3 |

由于v1.6版本的结构重组，章节编号整体前移了1位（第14编变为第13编）。查阅者追踪这些引用时，会定位到错误或不存在的章节。

### 违规项
- **宪法第1.1.1条**：文档引用必须匹配权威版本的章节编号

### 升级上报
→ **请 @architect 介入**，将所有宪法章节引用更新为v1.6的编号（`13.x.x`格式）。

## 问题D — 高优先级：引用不存在的文件路径（`.trae/` 与 `.qoder/` 不一致）

**严重程度**：高优先级
**受影响ADR**：ADR-002（第41行）、ADR-006（第42行、第992行）
**问题类型**：引用失效

### 问题描述
ADR-002 引用了 `.trae/rules/coding_rules.md` 作为参考文件。ADR-006 引用了 `.trae/rules/test_spec精简版.md`。但本项目使用 `.qoder/` 作为规则目录，而非 `.trae/`。这些文件在引用路径下并不存在。

2. `.qoder/rules/coding_rules.md`（或等效文件）中的实际编码规则，可能与ADR-002的假设存在差异
3. ADR-006引用的测试规范可能根本不存在

### 违规项
- **宪法第1.1.1条**：所有文档引用必须指向实际存在的文件

### 升级上报
→ **请 @architect 介入**，确认正确文件路径，并更新ADR-002与ADR-006。

## 问题E — 中优先级：`attribute_def.rs` / `tag_def.rs` 存放位置矛盾

**严重程度**：中优先级
**受影响ADR**：ADR-001（第292-293行）、ADR-002（债务4，第344-345行）
**问题类型**：跨ADR矛盾

### 问题描述
ADR-001 第2阶段核心层目标结构（第292-293行）将 `attribute_def.rs` 与 `tag_def.rs` 保留在 `core/` 目录下：

```
src/core/
├── tag_def.rs                 # 现有 TagDefPlugin（保留）
└── attribute_def.rs           # 现有 AttributeDefPlugin（保留）
```

ADR-002 的债务4项（第344-345行）明确指出这些文件应迁移至 `content/` 目录：

```
| `core/attribute_def.rs` | `content/attribute_def.rs` | 内容层 |
| `core/tag_def.rs` | `content/tag_def.rs` | 内容层 |
```

两处均未给出解决方案或交叉引用说明孰是孰非。若保留在 `core/`，则债务4提出的“core/ 职责清晰”目标将部分落空；若迁移至 `content/`，则ADR-001第2阶段的结构定义有误。

### 违规项
- **宪法第1.1.4条**：文档之间不得存在冲突

### 升级上报
→ **请 @architect 介入**，确定标准存放位置，并协调两份ADR保持一致。

## 问题F — 中优先级：ADR-001 内部阶段编号不一致

**严重程度**：中优先级
**受影响ADR**：ADR-001（第8、71、649、1182、1197、1302、1307行）
**问题类型**：内部自相矛盾

### 问题描述
ADR-001 中关于阶段总数的表述多处矛盾：

| 表述 | 位置 | 阶段数量 |
|---|---|---|
| "30 个工作日（8 个 Phase）" | 第8行 | 8个阶段 |
| "共 8 个 Phase" | 第71行 | 8个阶段 |
| 阶段列表：0、1、2、3、4、5、6、7、8 | 各章节 | 9个阶段 |
| 第7阶段 = Modding（"未来"） | 第1182行 | 未来阶段 |
| 第8阶段 = Tools（"未来"） | 第1197行 | 未来阶段 |
| "第7-8阶段 = 未来阶段" | 第1302行 | 未来阶段 |
| 执行链路：第0阶段 → ... → 第6阶段 | 第1307行 | 7个执行阶段 |

矛盾点：
- 声称共8个阶段，实际定义了9个（0-8）
- 声称“8个阶段（0-7）”，但第7、第8阶段均标注为“未来”
- 实际执行链路（0→1→2→3→4→5→6）仅包含7个阶段
- 若第0阶段算作脚手架搭建（非迁移阶段），则有效迁移阶段为1-6，共6个

### 违规项
- **宪法第1.1.4条**：同一文档内部不得自相矛盾

### 升级上报
→ **请 @architect 介入**，明确阶段编号规则（第0阶段是否计入脚手架、未来阶段如何标注），并更新阶段总数的表述。

**严重程度**：中优先级
**受影响ADR**：ADR-001（第176-177行）、ADR-003（第159行起）
