# ADR-005: 插件与通信迁移方案

## 状态: Proposed

**版本**: 1.0
**日期**: 2026-06-14
**范围**: Plugin 重组 + Message 迁移 + 通信契约重构
**依赖**: ADR-001（迁移计划）、ADR-003（分层契约）

---

## 背景

### 问题

当前 `src_待迁移` 目录的 Plugin 体系存在以下结构性问题：

1. **Plugin 层级混乱**：39 个 Plugin 未按七层架构分组注册，注册顺序违反依赖方向
2. **Message 分散**：35+ Message 类型散布在 6 个模块中，无统一归属规则
3. **循环依赖**：CharacterPlugin 注册 MovementIntent（定义在 ui/events.rs），造成 Core → UI 依赖
4. **职责越界**：BattlePlugin 注册 CombatLog 显示系统（UI 关注点），TurnPlugin cleanup 手动重置 13 个跨模块 Resource
5. **双事件反模式**：LogPlugin 注册 12 个日志专用 Message（ConfigLoaded、BuffApplied 等），这些只是原始事件的日志镜像

### 决策目标

定义完整的 Plugin 重组与通信迁移方案，确保：
- 所有 Plugin 按七层架构分组注册，依赖方向正确
- Message 归属清晰：核心域事件在 core，跨层事件在 shared，基础设施事件在 infra
- 消除循环依赖和职责越界
- 为后续迁移提供可执行的分步操作指南

## 决策

**采用分组注册 + Message 归属重构方案，将 39 个 Plugin 整合为 22 个注册组，同时将 35+ Message 类型迁移到正确的架构层。**

### 1.3 当前注册顺序问题

| 问题 | 违反规则 | 涉及 Plugin |
|------|----------|-------------|
| 数据层（SkillPlugin 等）在核心层（EffectPlugin）之前注册 | Core 应在 Content 之前 | SkillPlugin, BuffPlugin 等 |
| CharacterPlugin 注册 MovementIntent（定义在 ui/events.rs） | Core → UI 循环依赖 | CharacterPlugin |
| BattlePlugin 注册 CombatLogPlugin（日志显示 UI） | Core 包含 UI 关注点 | BattlePlugin |
| TurnPlugin cleanup 重置 13 个跨模块 Resource | Core 包含 UI/Infra 资源重置 | TurnPlugin |
| DebugPlugin 无条件注册 | 应 `#[cfg(feature = "dev")]` | DebugPlugin |
| 无 SharedPlugin | 缺少零依赖基础层 | — |
| 无 ContentPlugin | 数据加载分散在各业务模块 | AssetsPlugin |

## 第二节：目标 Plugin 注册（22 组，有序）

### 2.1 最终注册顺序

```rust
// src/app/app_plugin.rs — AppPlugin 统一组装
impl Plugin for AppPlugin {
    fn build(&self, app: &mut App) {
        app.add_plugins((
            DefaultPlugins
                .set(WindowPlugin { primary_window: Some(Window { ... }) })
                .set(AssetPlugin { file_path: asset_path })
        ))
        .add_plugins(EguiPlugin::default())

        // ── Group 1: Shared（零依赖）──
        .add_plugins(SharedPlugin)

        // ── Group 2: Infrastructure（依赖: shared）──
        .add_plugins((LogPlugin, AuditPlugin))

        // ── Group 3: Core Foundations（依赖: shared）──
        .add_plugins((
            EffectPlugin,
            ModifierRulePlugin,
            AttributeDefPlugin,
            TagDefPlugin,
        ))

        // ── Group 4: Core Domains（依赖: core foundations）──
        .add_plugins((
            BattlePlugin,
            TurnPlugin,
            CharacterPlugin,
            SkillPlugin,
            BuffPlugin,
            EquipmentPlugin,
            InventoryPlugin,
            AiPlugin,
            CampaignPlugin,
            MapPlugin,
        ))

        // ── Group 5: Content（依赖: core + infra + shared）──
        .add_plugins(ContentPlugin)

        // ── Group 6: UI（依赖: core read-only via ViewModel）──
        .add_plugins((UiPlugin, InputPlugin))

        // ── Group 7: Debug（依赖: core read-only，仅 dev 模式）──
        #[cfg(feature = "dev")]
        .add_plugins(DebugPlugin)

        // ── Group 8: Modding（依赖: core + shared + infra + content，可选）──
        #[cfg(feature = "modding")]
        .add_plugins(ModdingPlugin);
    }
}
```

### 2.2 分组依赖理由

| Group | 名称 | Plugin 数量 | 依赖说明 |
|-------|------|-------------|----------|
| 1 | Shared | 1 | 零依赖，提供强类型 ID、共享事件、数学工具 |
| 2 | Infrastructure | 2 | 依赖 Shared；提供日志、审计技术实现 |
| 3 | Core Foundations | 4 | 依赖 Shared；提供效果管线、修饰规则、属性/标签定义 |
| 4 | Core Domains | 10 | 依赖 Core Foundations；提供战斗、回合、角色等游戏规则 |
| 5 | Content | 1 | 依赖 Core + Infra；统一加载 RON 配置 → Registry |
| 6 | UI | 2 | 依赖 Core（只读 ViewModel）；用户界面与输入 |
| 7 | Debug | 1 | 依赖 Core（只读）；仅 `#[cfg(feature = "dev")]` |
| 8 | Modding | 1 | 依赖 Core + Shared + Infra + Content；可选 |

### 2.3 新增 Plugin

| Plugin | 层 | 文件路径 | 职责 |
|--------|-----|----------|------|
| SharedPlugin | shared | `src/shared/shared_plugin.rs` | 注册共享 ID 类型、事件、工具 |
| ContentPlugin | content | `src/content/content_plugin.rs` | 统一加载所有 RON 配置 → Registry |
| ModdingPlugin | modding | `src/modding/mod_plugin.rs` | MOD 支持（可选） |

```rust
    mut commands: Commands,
    // ... 8 个 Query ...

    // ... 重置 13 个 Resource（跨模块！） ...

    // ... 等 8 个 UI/Battle 资源

// AFTER: core/turn/plugin.rs cleanup_ingame
fn cleanup_ingame(
    mut commands: Commands,
    cameras: Query<Entity, With<CameraController>>,
    tiles: Query<Entity, With<TileSprite>>,
    units: Query<Entity, With<Unit>>,
) {
    // 只清理 Turn 自己管理的实体
    for e in &cameras { commands.entity(e).try_despawn(); }
    for e in &tiles { commands.entity(e).try_despawn(); }
    for e in &units { commands.entity(e).try_despawn(); }

    // 只重置 Turn 自己的资源
    commands.insert_resource(TurnState::default());
    commands.insert_resource(TurnOrder::default());
    commands.insert_resource(GameOverState::default());
    commands.insert_resource(NeedsResolve::default());
    commands.insert_resource(AiTimer::default());
    // UI/Battle 资源重置由各自的 Plugin 在 OnExit(AppState::InGame) 中处理
}
```

## 第四节：Message 迁移计划

### 4.1 Message 归属规则

| 规则 | 说明 | 示例 |
|------|------|------|
| 核心域事件 → core | 定义在 core/battle/events.rs | DamageApplied, CharacterDied |
| 跨层事件 → shared | 定义在 shared/events/ | UiCommand（UI→Core 通道） |
| 基础设施事件 → infra | 仅当 infra 有独立语义时 | （当前无此类型） |
| 删除双事件 | infra 日志事件直接监听核心域事件 | ConfigLoaded → 删除 |

### 5.1 跨模块通信契约表

| 生产者 | 消费者 | 机制 | 数据载体 | 迁移变更 |

### 5.2 通信方式选择矩阵

| 场景 | 推荐方式 | 理由 |
|------|----------|------|
| 战斗事件通知 UI | Message | 跨层广播，一对多 |
| 回合事件通知 UI | Message | 跨层广播 |
| Dead 组件添加触发死亡通知 | Observer | 同 Feature 局部响应 |
| 组件生命周期副作用 | Hook | 组件固有行为 |
| UI → Core 用户意图 | Message (UiCommand) | 跨层通信 |
| Core 内部同模块 | 函数调用 | 同模块最高效 |
| Core 跨模块 | Message | 模块边界解耦 |
| Entity 增删改 | Command | ECS 延迟执行 |
