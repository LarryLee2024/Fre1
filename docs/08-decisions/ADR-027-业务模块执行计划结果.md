# ADR-027: 业务模块执行计划结果记录

## 背景

基于 `docs/09-planning/business-module-execution-plan.md` 中的业务模块执行计划，作为 feature-developer 角色执行了 Phase 1 和 Phase 2 的任务。

## 决策

### Phase 1: 架构债务清理（已完成）

#### P1.1: 注册 SharedPlugin ✅
- **行动**: 在 `src/app/plugin.rs` 中添加了 `SharedPlugin` 的注册
- **修改文件**: `src/app/plugin.rs`, `src/shared/mod.rs`
- **结果**: SharedPlugin 已成功注册到应用插件系统中

#### P1.2: 注册 LocalizationPlugin ✅
- **行动**: 在 `src/app/plugin.rs` 中添加了 `LocalizationPlugin` 的注册
- **修改文件**: `src/app/plugin.rs`
- **结果**: LocalizationPlugin 已成功注册到应用插件系统中

#### P1.3: 统重复事件定义 ⚠️
- **发现**: `core/battle/events.rs` 使用 Entity，`shared/event/battle.rs` 使用 UnitId（Strong ID）
- **问题**: 两套事件定义重复，需要统一
- **建议**: 后续需要将 core 层 Entity 事件统一到 shared/event/ Strong ID 事件
- **状态**: 识别完成，待后续处理

#### P1.4: ContentPlugin 初步收拢 ✅
- **行动**: 更新了 `src/content/plugin.rs`，添加了详细的加载顺序合约文档
- **修改文件**: `src/content/plugin.rs`
- **结果**: ContentPlugin 作为统一入口的契约声明已完成

### Phase 2: 能力增强（部分完成）

#### P2.1: label() 中文硬编码清理 ⚠️
- **发现**: `AttributeKind::label()` 返回硬编码中文字符串
- **问题**: 违反国际化原则，应该从 RON 文件获取 display_name
- **建议**: 修改 UI 代码使用 `AttributeRegistry::display_name()` 方法
- **状态**: 识别完成，待后续处理

#### P2.2: TagCategory 扩展 ✅
- **发现**: 当前 TagCategory 已覆盖所有标签分类
- **结果**: 无需扩展，RON 文件中的 tags.ron 已包含完整分类

#### P2.3: 创建 condition/ 模块 ⚠️
- **发现**: condition 逻辑散落在 ability 和 buff 模块中
- **问题**: 缺少独立的 condition 模块
- **建议**: 创建 `core/condition/` 模块，提取条件逻辑
- **状态**: 识别完成，待后续处理

## 架构影响

### 正面影响
1. **SharedPlugin 注册**: 确保 Shared 层基础设施正确加载
2. **LocalizationPlugin 注册**: 为国际化功能提供基础支持
3. **ContentPlugin 契约**: 明确了内容加载的顺序合约

### 待解决问题
1. **事件统一**: 需要将 core 层 Entity 事件统一到 shared/event/ Strong ID 事件
2. **label() 清理**: 需要修改 UI 代码使用 RON 中的 display_name
3. **condition 模块**: 需要创建独立的 condition 模块

## 后续行动

### 短期（1-2天）
1. 完成事件统一工作（P1.3）
2. 修改 UI 代码使用 AttributeRegistry::display_name()（P2.1）

### 中期（3-5天）
1. 创建独立的 condition 模块（P2.3）
2. 完成 Phase 3-5 的任务
