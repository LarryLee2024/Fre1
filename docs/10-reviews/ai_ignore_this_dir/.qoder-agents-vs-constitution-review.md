## 三、Constitution v5.0 内容冲突

### 3.1 通信类型：缺少 Trigger

Constitution v5.0 定义了 4 种通信类型：**Hook / Trigger / Observer / Message**

多个 Agent 仍使用旧的 3 类型模型。

| Agent | 当前引用 | 应改为 |
|-------|---------|--------|
| architect (行 20-21) | Message/Observer/Hook | Message/Trigger/Observer/Hook |
| architect (行 88-91) | ADR 模板 Communication Design 缺 Trigger | 添加 Trigger 行 |
| code-reviewer (行 41) | "Message/Observer/Command" | "Message/Trigger/Observer/Hook" |
| feature-developer (行 48-49) | Hook/Observer/Message | Hook/Trigger/Observer/Message |

### 3.2 Capabilities/Domains 双轴架构缺失

Constitution v5.0 核心架构是"纵向三层 + 横向四层"，Core 层内部分为 Capabilities（能力轴）和 Domains（领域轴）。

| Agent | 缺失内容 |
|-------|---------|
| architect | 未提及双轴架构、未在 ADR 模板中体现 Capabilities/Domains |
| code-reviewer | 审查清单无双轴边界违规检查 |
| domain-designer | 领域分类未使用 Capabilities/Domains 框架 |
| refactor-guardian | 扫描目标无双轴边界违规检查 |

### 3.3 测试架构过时（test-guardian）

Constitution v5.0 §12.2 定义了**领域内聚 4 层测试**：unit / integration / invariant / fixtures

test-guardian.md 仍使用旧的测试金字塔：
```
当前（过时）：           应改为：
Unit 70%               unit (纯函数)
Integration 20%        integration (最小 Bevy App)
Replay 8%              invariant (业务不变量，最高优先级)
E2E 2%                 fixtures (测试数据)
```

关键变更：
- **不变量测试 (invariant)** 应为最高优先级
- 回放测试 (replay) 不再是独立层级
- E2E 测试不在 4 层体系中

### 3.4 AI 自检格式过时（feature-developer）

Constitution v5.0 将 AI 自检从代码块输出改为文档引用。

feature-developer.md 行 112-122 仍输出代码块格式的自检结果：
```
Feature First: PASS/FAIL
Definition/Instance: PASS/FAIL
...
```

应改为：引用 `docs/00-governance/ai-constitution-complete.md` 中的自检清单。

### 3.5 data-architect.md 内容对齐

data-architect.md 内容与 v5.0 **基本对齐**：
- ✅ Definition/Instance 强制分离
- ✅ Rule/Content 强制分离
- ✅ Replay 优先于便利
- ✅ Data Laws 体系完整 (001-010)
- ✅ 四层数据模型 (Definition/Instance/Runtime/Persistence)
- ✅ Schema Design 职责明确

仅需修正目录路径（`docs/domain/` → `docs/02-domain/`）。
