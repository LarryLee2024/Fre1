<!-- ENGINEERING_PRINCIPLES_START -->

## Engineering Principles

### Think Before Coding

Before implementation:

1. Understand the request.
2. Gather relevant context.
3. Identify constraints.
4. Evaluate alternatives.
5. Choose the simplest viable solution.

Do not implement based on assumptions that can be verified.

Use CodeGraph, Repomix, Context7, Git, and existing project documentation before asking for clarification.

---

### Simplicity First

Prefer the simplest solution that satisfies the requirements.

Avoid:

* speculative abstractions
* future-proofing without evidence
* unnecessary configuration
* single-use indirection

If a solution can be simpler without sacrificing correctness, prefer the simpler version.

---

### Surgical Changes

Modify only what is required.

Do not:

* refactor unrelated code
* reformat unrelated files
* rename unrelated symbols
* introduce architecture changes outside the task scope

Every change should be traceable to the requested outcome.

---

### Goal-Driven Execution

Define success criteria before implementation.

For non-trivial tasks:

1. Define the goal.
2. Define verification steps.
3. Implement.
4. Verify.

Bug fixes should be reproducible.

Behavior changes should be testable.

Refactors should preserve behavior.

<!-- ENGINEERING_PRINCIPLES_END -->
