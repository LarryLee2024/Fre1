# 二、重点检查项

## 1 Observer

0.19新增：

```rust
app.add_observer(
    on_damage.run_if(...)
);
```

以前：

```rust
app.add_observer(on_damage);
```

现在可以：

```rust
app.add_observer(
    on_damage
        .run_if(not_paused)
        .run_if(has_battle)
);
```

对于你的战棋项目特别有价值。

例如：

```rust
DamageApplied
```

观察器：

```rust
GamePhase == Battle
```

时才触发。

---

建议：

把大量：

```rust
if battle_state != BattleState::Running {
    return;
}
```

逐步迁移到：

```rust
run_if(...)
```

---

# 2 Delayed Commands

这是 0.19 最适合 SRPG 的新功能之一。

以前：

```rust
Timer
```

写一堆：

```rust
AnimationTimer
BuffTimer
EffectTimer
```

---

现在：

```rust
commands
    .delayed()
    .secs(1.5)
    .entity(target)
    .insert(Dead);
```

---

例如：

死亡动画：

```rust
播放死亡动画
↓
1.5秒后移除实体
```

直接：

```rust
commands
    .delayed()
    .secs(1.5)
    .entity(entity)
    .despawn();
```

---

DOT伤害：

```rust
3秒后触发
```

也能用。

---

对于你的：

```text
Ability
Effect
Buff
Animation
VFX
```

会减少大量 Timer 样板代码。

这是整个 0.19 最大更新。

但我要给你一个非常谨慎的建议。

---

原因：

BSN 目前相当于：

```text
下一代 Spawn DSL
```

不是 ECS 核心。

---

你的项目已经大量使用：

```rust

    Character,
    Health,
    Position,
));
```

这种写法。

没必要重构。

---

特别适合：

```text

```

以前：

```rust

    ...
});
```

很恶心。

---

BSN：

```rust
bsn! {
    Node
    Children [
        Text("Attack")
    ]
}
```

明显更好。

# 20. BSN 最危险的地方

最后说一个风险。

很多人看到 BSN：

```rust
bsn! {
    Character
    Children [
        Weapon
    ]
}
```

会兴奋。

---

但对于大型项目：

要警惕：

```text
业务逻辑进入场景定义
```

---

例如：

坏：

```rust
bsn! {
    Character
    if level > 10 {
        LegendaryWeapon
    }
}
```

---

这种很快变成：

```text
场景脚本
```

---

对于你的项目。

我建议未来即使采用 BSN：

也坚持：

```text
BSN负责结构

System负责行为

Domain负责规则
```

---

这是很多大型项目最终会踩的坑。

---

如果站在**3~5年维护周期**看，而不是看一个版本升级：

我认为 0.19 最值得你额外关注的其实只有三个隐藏信号：

```text
1.
ECS正在向“数据批处理”方向发展
(contiguous_iter)

2.
Observer正在成为一级公民
(事件驱动越来越强)

3.
Bevy正在朝“编辑器平台”演化
(BSN + Feathers + Gizmo + Settings)
```

这三个信号，会比具体 API 名字更影响你未来几十万行 SRPG 项目的架构决策。
再往下挖，其实还有一些**架构层面的信号**，发布说明没直接说，但从 Bevy 核心团队这一年的改动方向能看出来。

这些前面都没提过。

# 22. 反射（Reflect）地位进一步上升

# 30. 对你最现实的一条

你的项目有大量：

```text
Ability
Effect
Buff
Character
```

领域。

---

0.19 出来以后。

如果让我选一个最值得提前布局的东西，不是 BSN。

而是：

```rust
Relationship
+
Observer
```

组合。

---

因为这两者其实已经越来越接近：

```text
UE GAS
Unity DOTS
Mass Framework
```

的设计思路。

---

举例：

未来完全可能变成：

```rust
CasterOf(Target)
```

↓

```rust
DamageApplied
```

↓

Observer

↓

```rust
Has(BurningBuff)
```

↓

Observer

↓

```rust
Death
```

---

整个链路几乎没有传统 OOP 调度器。

而是纯 ECS 关系驱动。

这条路线和你的 SRPG 领域模型是高度契合的。

---

所以如果问我：

**除了前面所有内容之外，还有什么最值得记在架构笔记里？**

```text
0.19 真正值得关注的不是 BSN。

而是 Bevy 官方正在把 ECS 从
“组件存储系统”
升级成
“关系 + 生命周期 + 观察器驱动的数据模型”。

这会直接影响未来 0.20~0.22 的架构方向。
```

这一点很多人读完整个 Release Note 都未必能看出来。
