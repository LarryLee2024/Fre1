# 第一章：架构目标

本项目不是传统游戏项目。

本项目目标：

1. 长期维护（5年以上）
2. 数十万行代码规模
3. 支持持续扩展内容
4. 支持确定性回放
5. 支持自动化测试
6. 支持战斗模拟器
7. 支持未来服务器模拟
8. 支持未来Mod生态

因此：

架构优先级如下：

```text
正确性
>
可维护性
>
可扩展性
>
开发效率
>
性能
```

性能优化必须基于Profile。

禁止凭感觉优化。

---

# 第二章：最高原则

## P0-1 Feature First

永远按业务领域组织代码。

禁止：

```text

events/
utils/
helpers/
```

允许：

```text

```

---

## P0-2 Data Driven First

新增内容优先新增数据。

禁止：

```rust
match skill_id {
    Fireball => ...
}
```

允许：

```ron
(
    ability: "fireball"
)
```

---

## P0-3 Replay First

所有核心战斗逻辑必须可重放。

任何随机行为：

必须来自统一随机源。

禁止：

```rust
rand::thread_rng()
```

允许：

```rust
BattleRng
```

---

## P0-4 Logic Presentation Separation

业务逻辑与表现彻底分离。

禁止：

```rust
apply_damage();
play_animation();
```

同系统混写。

必须：

```text
Core
↓
Cue
↓
Presentation
```

---

## P0-5 Composition Over Inheritance

禁止继承驱动设计。

角色差异：

通过：

```text

Modifier
Effect
Ability
```

组合实现。

# 第六章：ECS使用规范

允许：

```rust
#[derive(Component)]
pub struct Health;
```

定义于Core。

---

允许：

```rust
Query
Component
Resource
Event
Observer
Schedule
```

出现在Core。

---

禁止：

```rust
AssetServer
Image
AudioPlayer
UiNode
```

出现在Core。

---

原则：

ECS属于架构模型。

不是技术实现。

---

# 第七章：通信规范

## Hook

固有行为。

例如：

```text
Component Hook
```

---

## Observer

局部响应。

例如：

```text
EffectApplied
```

---

## Message

跨领域广播。

例如：

```text
BattleEnded
QuestCompleted
```

---

规则：

```text
Hook
>
Observer
>
Message
```

优先级递减。

禁止事件滥用。

# 第十章：性能例外机制

原则：

禁止提前优化。

---

申请例外必须满足：

1. Profile证据
2. ADR记录
3. 有效期
4. 重构计划

格式：

```rust
// ARCH_EXCEPTION
// ADR-042
// expires: 2027-01-01
```

# 第十一章：禁止事项

```text

common.rs
```

垃圾桶模块。

---

禁止：

```text

```

替代Tag。

