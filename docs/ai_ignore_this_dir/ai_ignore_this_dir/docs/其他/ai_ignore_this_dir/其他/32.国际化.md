对于你的项目（Bevy 0.18+、长期SRPG、未来MOD、未来多语言、AI协作），我不推荐很多教程里的简单方案：

```rust
HashMap<String, String>
```

或者：

```text
assets/i18n/en.json
assets/i18n/zh.json
```

直接到处查字符串。

因为做到后面会出现：

```text
几千条文本
几十个章节
上百个任务
MOD文本
动态变量
```

然后彻底失控。

---

# 先说结论

目前 Bevy 生态里，长期项目最推荐的是：

```text
bevy + fluent(Ftl)
```

即：

```text
fluent-rs
+
bevy_fluent
(或自己封装)
```

而不是：

```text
ron
json
yaml
HashMap
```

直接做国际化。

---

# 为什么 Fluent 是目前最优解

Fluent 是：

Mozilla

设计的现代国际化系统。

---

它天然支持：

```text
变量

复数

性别

条件文本

参数插值

多语言语法差异
```

---

例如：

```ftl
battle-damage =
    { $attacker } 对 { $target } 造成了 { $damage } 点伤害
```

---

运行：

```text
attacker=Knight
target=Goblin
damage=50
```

得到：

```text
Knight 对 Goblin 造成了 50 点伤害
```

---

英文：

```ftl
battle-damage =
    { $attacker } dealt { $damage } damage to { $target }
```

---

代码完全不用改。

---

# 不推荐 JSON 原因

很多人这样：

```json
{
  "battle_damage": "{attacker} dealt {damage}"
}
```

---

然后自己实现：

```rust
replace("{attacker}")
replace("{damage}")
```

---

做到后面：

```text
复数
语法
条件
```

全部要自己写。

---

Fluent 已经解决了。

很多项目这样：

```text

```

---

几年后：

```text
没人知道是什么
```

---

推荐：

```text
battle.turn.start

battle.turn.end

battle.damage

battle.heal

skill.fireball.name

skill.fireball.desc

quest.chapter1.start

quest.chapter1.end
```

---

# Content层怎么做

这是你项目最关键的地方。

---

不要：

```json
{
  "name": "火球术"
}
```

---

应该：

```json
{
  "name_key": "skill.fireball.name",
  "desc_key": "skill.fireball.desc"
}
```

---

技能配置永远保存：

```text
Key
```

---

不是：

```text
文本
```

---

这样：

```text
新增语言
MOD
DLC
```

全部不影响数据。

---

# UI层

不要：

```rust
Text::new("开始战斗")
```

---

应该：

```rust
LocalizedText {
    key: "battle.start"
}
```

---

切换语言：

```text
自动刷新
```

# 对话系统

推荐：

```text
content/dialogues
```

只保存：

```json
{
  "text_key": "dialogue.chapter1.knight.001"
}
```

---

真正文本：

```ftl
dialogue-chapter1-knight-001 =
    我们必须守住这座城。
```

---

# 技能描述

很多项目踩坑。

---

不要：

```text
造成100点伤害
```

写死。

---

应该：

```ftl
skill-fireball-desc =
    对目标造成 { $damage } 点火焰伤害
```

---

然后：

```rust
damage=150
```

动态生成。

---

这样平衡修改后：

```text
150 → 180
```

描述自动更新。

---

# 语音

```text
voice/
```

推荐：

```json
{
  "text_key": "...",
  "voice_key": "..."
}
```

---

而不是直接绑定文件名。
