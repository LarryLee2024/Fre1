## 3. 逐文件迁移映射表

> **Action 说明**：
> - `Move` = 直接移动，路径变化
> - `Rename+Move` = 重命名后移动
> - `Merge` = 与现有文件合并
> - `Split` = 拆分为多个文件
> - `Delete` = 删除（功能已被拆分覆盖）
> - `Rewrite` = 重写内容
> - `Keep` = 保持原位（目标路径 = 当前路径）

## 7. 双兼容 Re-export 模式详解

### 设计原则

每个迁移阶段保留旧路径的兼容 shim，确保：
1. **阶段内可编译**：每次迁移后 `cargo check` 必须通过
2. **渐进式更新**：其他模块可以在不同阶段更新 use 路径
3. **最终清理**：Phase 7 统一删除所有 shim

### Re-export 模式模板

```rust
// ===== 模块级 re-export =====
// 文件：src/{旧路径}/mod.rs
// 用途：整个模块迁移后，旧路径仍可用

/// 兼容层 — 已迁移到 crate::{新路径}
/// TODO: Phase 7 清理时删除此文件
pub use crate::{新路径}::*;
```

```rust
// ===== 文件级 re-export =====
// 文件：src/{旧路径}/{旧文件名}.rs
// 用途：单文件迁移后，旧路径仍可用

/// 兼容层 — 已迁移到 crate::{新路径}::{新文件名}
/// TODO: Phase 7 清理时删除此文件
pub use crate::{新路径}::{新文件名}::*;
```
