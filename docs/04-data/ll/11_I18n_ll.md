# I18n 领域 — 铃兰之剑数据提取（国际化补充）

> 领域：I18n | 来源：ADR-017 + 78铃兰.md | 数据层：Definition + Instance

---

## 一、国际化架构总览

### 1.1 核心原则

| 原则 | 说明 | 来源 |
|------|------|------|
| Key 唯一永久 | 命名空间 + 永久唯一 ID，一旦创建永不修改 | ADR-017 §4 |
| 配置只存 Key | 所有 Content 配置中的 name/description 替换为 name_key/desc_key | ADR-017 §2 |
| 禁止语义化 Key | 禁止使用 `skill.fireball.name`，必须用 `skill.s_1001.name` | ADR-017 §4 |
| 回退链保障 | 请求语言 → 项目默认(zh-CN) → 英语(en-US) → Key本身 | ADR-017 §6 |

### 1.2 语言回退链

```
请求语言 → zh-CN（项目默认）→ en-US（兜底）→ Key本身(Debug)/空字符串(Release)
```

MOD 回退链：`MOD_zh-CN → 本体_zh-CN → 本体_en-US`

## 二、Key 命名规范

### 2.1 Key 结构

```
<namespace>.<permanent_id>.<field>
```

| 组成部分 | 说明 | 示例 |
|----------|------|------|
| namespace | 业务域命名空间 | skill, buff, attr, tag, ability |
| permanent_id | 永久唯一 ID（创建后永不修改） | s_1001, b_2001, a_001 |
| field | 字段类型 | name, desc, tip |

### 2.2 Key 命名空间映射

| 命名空间 | 领域 | 说明 |
|----------|------|------|
| `skill` | Ability | 技能相关文本 |
| `buff` | Effect | Buff/Debuff相关文本 |
| `attr` | Attribute | 属性相关文本 |
| `tag` | Tag | 标签/状态相关文本 |
| `ui` | UI | 界面相关文本 |
| `battle` | Execution | 战斗相关文本 |

### 2.3 Key 示例

| 领域 | 数据元素 | 旧格式 | 新格式（Key） |
|------|----------|--------|---------------|
| Attribute | 物理攻击 | `name: "物理攻击"` | `name_key: "attr.a_001.name"` |
| Attribute | 暴击率 | `name: "暴击率"` | `name_key: "attr.a_006.name"` |
| Tag | 眩晕状态 | `name: "眩晕"` | `name_key: "tag.t_001.name"` |
| Effect | 中毒 Buff | `name: "中毒"` | `name_key: "buff.b_001.name"` |
| Ability | 火球术 | `name: "火球术"` | `name_key: "skill.s_1001.name"` |
| Ability | 火球术描述 | `description: "发射火球..."` | `desc_key: "skill.s_1001.desc"` |

## 五、数据层划分

| 数据层 | 国际化相关内容 | 说明 |
|--------|----------------|------|
| Definition | FTL 文件、Key 配置 | 翻译表和本地化 Key |
| Instance | FluentBundle、LocalizedText 组件、CurrentLocale | 运行时缓存和状态 |
| Runtime | 文本解析结果（临时） | 解析后的文本，不持久化 |
| Persistence | 仅存储 Key，不存储文本 | 存档中只保存 Key |

---

## 六、LocalizedText 组件

```rust
#[derive(Component)]
pub struct LocalizedText {
    pub key: String,           // 本地化 Key
    pub args: Option<FluentArgs>, // 变量参数（如伤害值、持续时间）
}
```

### 6.1 变量参数类型

| 参数类型 | 说明 | 示例 |
|----------|------|------|
| `$damage` | 伤害数值 | `{ $damage }` |
| `$heal_amount` | 治疗数值 | `{ $heal_amount }` |
| `$duration` | 持续回合数 | `{ $duration }` |
| `$multiplier` | 技能倍率 | `{ $multiplier }` |
| `$count` | 数量（支持复数） | `{ $count -> [one]... [other]... }` |

## 七、性能策略

| 策略 | 说明 |
|------|------|
| 预解析缓存 | FluentBundle 启动时预解析并缓存 |
| 惰性解析 | 仅语言切换时重新格式化 |
| LRU 缓存 | `LocalizedTextCache` 按 Key + 参数哈希缓存（上限 2048 条） |
| 异步加载 | 大体积 FTL 文件（>100KB）异步加载 |

---

## 八、边界定义

| 规则 | 说明 |
|------|------|
| 允许 | `content/` 和 `ui/` 依赖 `infrastructure/localization/`（解析文本） |
| 禁止 | `core/` 直接调用 `LocalizationService`（Core 只存 Key） |
| 禁止 | 运行时修改 FTL 翻译表（Definition 不可变） |
| 禁止 | MOD 翻译覆盖原版 Key（破坏原版文本完整性） |

---

## 九、禁止事项

| 禁止项 | 理由 |
|--------|------|
| Content 配置中硬编码文本 | 违反宪法 §17.2.2 |
| UI 中使用 `Text::new("硬编码")` | 语言切换时无法自动刷新 |
| 使用语义化名称作为 Key | 改名后批量替换灾难 |
| 使用无意义编号 Key | 维护时无法识别内容 |
| Core 层直接调用 LocalizationService | Core 只存 Key，解析在上层 |
| 每帧解析 Fluent AST | CPU 开销导致帧率暴跌 |
| UI 容器固定像素宽度 | 德语/俄语比中文长 30%~50% |

## 十、国际化改造影响评估

### 10.1 需要修改的配置

| 配置文件 | 修改内容 | 工作量 |
|----------|----------|--------|
| `attribute_config.ron` | `name` → `name_key` | 中 |
| `tag_config.ron` | `name` → `name_key` | 中 |
| `effect_config.ron` | `name`/`description` → `name_key`/`desc_key` | 高 |
| `ability_config.ron` | `name`/`description` → `name_key`/`desc_key` | 高 |

### 10.2 需要创建的 FTL 文件

| 文件 | 条目数估计 | 说明 |
|------|-----------|------|
| `zh-CN/attr.ftl` | 11 | 属性名称 |
| `zh-CN/tag.ftl` | 50+ | 标签名称 |
| `zh-CN/buff.ftl` | 100+ | Buff名称和描述 |
| `zh-CN/skill.ftl` | 200+ | 技能名称和描述 |
| `zh-CN/ui.ftl` | 500+ | 界面文本 |
| `zh-CN/battle.ftl` | 100+ | 战斗文本 |

## 十一、与原数据提取的兼容性

| 原字段 | 新字段 | 迁移策略 |
|--------|--------|----------|
| `name` | `name_key` | 一次性替换，生成对应 FTL 条目 |
| `description` | `desc_key` | 一次性替换，生成对应 FTL 条目 |
| 数值字段 | 保持不变 | 数值无需国际化 |

## 十二、Data Laws 合规

| Law | 状态 | 说明 |
|-----|------|------|
| 001 | ✅ | Definition(FTL)与Instance(FluentBundle)分离 |
| 002 | ✅ | 翻译规则在代码中，文本内容在配置(FTL)中 |
| 003 | ✅ | 配置只引用Key，不重复定义文本 |
| 010 | ✅ | 翻译文本不影响Replay，Key固定不变 |
