# 调试工具操作说明

本文档说明项目中所有调试功能的使用方法和预期效果。

---

## 快捷键总览

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| 按钮 | 切换调试面板 | 点击屏幕右下角 "Show Debug" / "Hide Debug" 按钮 |
| F1 | 切换调试面板 | 快捷键方式（如可用） |
| F2 | Buff 视图 | 切换到 Buff 查看器 |
| F3 | Overlay 视图 | 切换到 Gizmos 可视化开关面板 |
| F4 | Damage 视图 | 切换到伤害分解与属性修饰面板 |
| F5 | Turn 视图 | 切换到回合队列查看器 |
| F6 | Stepping 暂停 | 切换 Debug Stepping 启用/禁用 |
| F7 | Stepping 单步 | 执行下一个 System |
| F12 | World Inspector | 打开 bevy-inspector-egui 全局检查器 |

## 打开调试面板

### 方式一：点击按钮（推荐）

游戏启动后，在屏幕**右下角**会显示一个按钮：

```
┌─────────────────────────────────────────────┐
│                                             │
│              游戏画面                        │
│                                             │
│                                             │
│                                   [Show Debug]
└─────────────────────────────────────────────┘
```

- 点击 **Show Debug** → 打开调试面板
- 点击 **Hide Debug** → 关闭调试面板

### 方式二：快捷键 F1

按 **F1** 键可切换调试面板显隐（需要窗口聚焦）。

## 统一调试面板

启动游戏后，调试面板默认显示在屏幕左上角。

### 面板布局

```

│  导航栏   │              内容区域                        │
```

## 各视图详细说明

### 1. Battle（战斗调试器）

**快捷键**：F1 切换面板后点击 Battle，或面板已打开时直接查看

**显示内容**：
- 当前回合数
- 当前行动单位（阵营 + 名称 + Entity ID）
- 队列进度（当前索引 / 总数）
- 事件统计：
  - 总事件数
  - 伤害次数
  - 治疗次数
  - DoT（持续伤害）次数
  - 死亡次数

**预期效果**：
```
回合: 3
当前行动: [友]战士 (e:0)
队列进度: 2/5
───────────────
总事件数: 12
  伤害: 8  治疗: 2
  DoT: 1  死亡: 1
```

---

### 2. Buff（Buff 查看器）

**快捷键**：F2

**显示内容**：
- 所有单位的 Buff 列表（按阵营分组）
- 每个 Buff 的详细信息：
  - 类型图标：▲增益 ▼减益
  - Buff 名称和 ID
  - 剩余回合数
  - DoT/HoT 数值
  - 晕眩状态标记

**预期效果**：
```
[友]战士 (e:0)
  ▲ 攻击增强 (id:attack_up) 剩余:2回合
  ▼ 中毒 (id:poison) DoT:5 剩余:3回合

[敌]哥布林 (e:3)
  (无 Buff)
```

---

### 3. Overlay（调试覆盖层）

**快捷键**：F3

**功能**：控制游戏内 Gizmos 可视化的开关

**可开关项**：
| 开关 | 颜色 | 显示内容 |
|------|------|----------|
| 寻路路径 | 绿色(起点) → 青绿(中间) → 黄色(终点) | 移动单位的路径轨迹 |
| AI 决策 | 蓝色(攻击者) + 红色(目标) | AI 的攻击意图 |
| 占用网格 | 蓝色(玩家) / 红色(敌方) / 灰色(未知) | 被占据的格子标记 |
| 范围轮廓 | 蓝色(移动) / 红色(攻击) | 攻击/移动范围线框 |

**操作方式**：
- 点击复选框单独开关
- 按 F3 全部切换（开→关 或 关→开）

**预期效果**：
```
Debug Overlay
Gizmos 可视化开关 (F3 全部切换)
───────────────
☑ 寻路路径
☐ AI 决策
☑ 占用网格
☐ 范围轮廓
```

### 4. Damage（伤害与属性调试器）

**快捷键**：F4

**功能**：查看伤害计算详情和属性修饰来源

**子面板切换**：
- **Damage Breakdown**：伤害计算全链路
- **Attribute Modifier**：属性修饰符来源

#### Damage Breakdown 子面板

显示最近 20 条伤害记录，每条可展开查看详情：
- 原始效果值
- 修饰符列表（每步的 before/after/rule_name）
- 修饰后伤害
- 实际扣血值

**预期效果**：
```
Damage & Attribute Debugger
[Damage Breakdown] [Attribute Modifier]
───────────────
#1 战士 → 哥布林  伤害:25 [技能]
  ▸ 展开查看详情
#2 法师 → 哥布林  伤害:18
  ▸ 展开查看详情
```

#### Attribute Modifier 子面板

显示所有单位的属性值和修饰来源：
- HP/MP 当前值/最大值
- 8 维核心属性（力量/敏捷/智力等）
- 每个属性的修饰符列表：
  - 来源（Trait/装备/Buff）
  - 操作类型（+加法 / x乘法）
  - 数值变化（绿色=增益，红色=减益）

**预期效果**：
```
[友]战士 (e:0)
HP: 85 / 100
MP: 20 / 30
───────────────
力量 = 8
  基础: 5
  +3 (装备(铁剑))
体质 = 7
  基础: 5
  +2 (Trait(warrior_mastery))
```

### 6. Stepping（调试单步）

**快捷键**：F6 暂停/继续，F7 单步执行

**功能**：System 级单步调试，用于调试 Buff 链/Observer 链/回合流程

**操作方式**：
- **F6**：暂停/继续（切换 Stepping 启用状态）
- **F7**：单步执行（执行下一个 System）
- **面板按钮**：
  - ▶ 继续：恢复执行
  - ⏭ 单步：执行下一个 System
  - ⏹ 停用：禁用 Stepping
  - ⏸ 启用 Stepping：启用单步模式

**预期效果**：
```
Debug Stepping
System 单步调试 (F6 暂停/继续, F7 单步)
───────────────
⏸ 已暂停

[▶ 继续] [⏭ 单步] [⏹ 停用]

当前: Update[buff_tick_system]
───────────────
统计: 切换5次, 单步12步
```

## Gizmos 可视化详细说明

Gizmos 是在游戏画面上叠加的调试可视化层，每帧自动清除并重绘。

### 寻路路径（show_pathfinding）

当单位移动时显示路径：
- **起点**：绿色方块 `(0.2, 1.0, 0.2)`
- **中间节点**：青绿色方块 `(0.0, 1.0, 0.5)`
- **终点**：黄色方块 `(1.0, 1.0, 0.0)`

### AI 决策（show_ai_intent）

显示 AI 的攻击意图：
- **攻击者位置**：蓝色方块（敌方单位）
- **攻击目标**：红色方块（双层轮廓增强可见性）

### 占用网格（show_occupancy）

显示被占据的格子：
- **玩家单位**：蓝色小方块 `(0.3, 0.6, 1.0)`
- **敌方单位**：红色小方块 `(1.0, 0.3, 0.2)`
- **未知/中立**：灰色小方块 `(0.7, 0.7, 0.7)`
- 标记位于格子右上角，不遮挡格子中心内容

### 范围轮廓（show_range_outline）

显示攻击/移动范围的线框轮廓：
- **移动范围**：蓝色线框 `(0.3, 0.6, 1.0)`
- **攻击范围**：红色线框 `(1.0, 0.3, 0.2)`
- 与 Sprite 填充高亮互补，提供清晰边界

## Debug Stepping 详细说明

### 工作原理

Debug Stepping 基于 Bevy 的 `Stepping` Resource，允许逐个 System 执行进行调试。

### 支持的 Schedule

- **Update**：主逻辑更新
- **FixedUpdate**：固定步长更新
- **PostUpdate**：后处理更新

### 使用场景

1. **调试 Buff 链**：观察 Buff 应用/移除的顺序
2. **调试 Observer 链**：追踪事件触发的系统执行顺序
3. **调试回合流程**：逐步执行回合状态转换
4. **定位 Bug**：暂停后检查中间状态

### 注意事项

- Stepping 启用时所有 Schedule 同时暂停
- 单步执行每次只执行一个 System
- 调试面板显示当前游标位置（Schedule + NodeId）

## World Inspector

**快捷键**：F12

由 `bevy-inspector-egui` 提供的全局 ECS 世界检查器。

### 功能

- 查看所有 Entity 的组件
- 查看所有 Resource 的状态
- 实时修改 Resource 值
- 搜索和过滤 Entity

### 使用场景

- 检查 Entity 组件状态
- 调试 Resource 值
- 查看系统注册状态

## bevy_remote

已注册 `RemotePlugin`，提供 BRP（Bevy Remote Protocol）协议核心能力。

### 功能

- 通过网络查询/修改 Entity 和 Resource
- 未来可连接 Bevy Editor

### 注意

当前 Bevy 0.18.1 的 bevy_remote 未启用 HTTP 传输层。如需 HTTP 远程访问，需在 `Cargo.toml` 中为 bevy_remote 单独启用 http feature。

## 故障排查

### 调试面板不显示

1. 确认使用 `--features dev` 编译运行
2. 按 F1 切换面板显隐
3. 检查 `DebugPanelState.show_panel` 是否为 true

### F1 快捷键无响应

1. 确保游戏窗口已聚焦
2. 检查是否有其他程序占用 F1
3. 尝试点击导航栏切换视图

### Gizmos 不显示

1. 打开 Overlay 面板（F3）
2. 确认对应开关已启用
3. 确保有相应的游戏状态（如移动中的单位）

### 面板内容为空

1. 确保游戏已进入 InGame 状态
2. 检查是否有对应的数据（如 BattleRecord）
3. 查看控制台是否有错误输出

## 开发者说明

### 面板位置规划

所有调试面板按区域分布，避免重叠：

| 区域 | 面板 | 位置 |
|------|------|------|
| 左侧 | Battle Debugger | [10, 10] |
| 左侧 | Buff Viewer | [10, 200] |
| 中间 | Damage & Attribute | [370, 10] |
| 右侧 | Debug Overlay | [740, 10] |
| 右侧 | Debug Stepping | [740, 200] |
| 底部 | Turn Queue | [10, 960] |

### 添加新调试视图

1. 在 `DebugView` 枚举中添加新变体
2. 在 `DebugView::all()` 中添加标签和快捷键
3. 在 `unified_debug_panel` 的 match 中添加渲染逻辑
4. 在 `viewers/` 下创建新的 viewer 模块

### 遵循的规则

- 调试面板只读取数据，不修改业务状态
- Gizmos 只读取数据，不触发业务逻辑
- 所有调试功能仅在开发模式下启用
- 快捷键在 PreUpdate 中处理，使用 just_pressed
