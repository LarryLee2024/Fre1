# 随机数/RNG 系统激进重构计划

> **扫描日期**: 2026-06-21 | **范围**: ADR-041 + 宪法 §8.11 + shared/random/ + core/replay/ + infra/replay/
> **发现**: 1 个 P0 架构违规 + 2 个 P1 设计问题

---

## 一、评审结论

### 文档一致性

| 检查项 | 结果 | 说明 |
|--------|------|------|
| ADR-041 vs 宪法 §8.11 | ✅ 一致 | RNG 流分离、禁止 rand::random() |
| ADR-041 vs SRPG规则 §8.3 | ✅ 一致 | 四流分离（Combat/Drop/AI/World） |
| ADR-041 vs Data Law 010 | ✅ 一致 | Replay First |
| 宪法 §2.4 vs 实际代码 | ❌ **违规** | shared/random/ 应放基础 RNG，但 4 流 DeterministicRng 在 core/ |

### P0: 架构违规

```
┌──────────────────┐    宪法 §2.4: shared/random/ 是"确定性随机数基础接口"
│ shared/random/   │    ✅ 正确位置
│  └── SeededRng   │    ChaCha12 包装器
│  └── GameRng     │    弃用（僵尸代码）
└──────────────────┘
         ↑ 应放在这里
┌──────────────────┐    
│ core/replay/     │    ❌ 架构违规
│  └── DeterministicRng  │    4 流 RNG 放在 core 层
│       （自制 hash PRNG）│    违反 Shared ← Core ← Infra
└──────────────────┘
         ↓ 错误依赖
┌──────────────────┐
│ infra/replay/    │    
│  └── DeterministicRng  │    ECS 包装器
│  └── recording.rs      │    Core 域竟然依赖 Infra！
└──────────────────┘
```

**具体违规点**:
1. `core/domains/combat/integration/replay/recording.rs:34` — 直接 `use crate::infra::replay::resources::DeterministicRng` (Core → Infra)
2. `core/capabilities/runtime/replay/foundation/values.rs` — DeterministicRng（4 流 RNG）在 core 层，但基础 RNG 工具应归 shared/

### P1: 自制 PRNG 替换 ChaCha12

`DeterministicRng::next_u64()` 当前使用自制 MurmurHash3 风格混合器：
```rust
let mut x = state;
x = x.wrapping_mul(0x9E37_79B9_7F4A_7C15);
x ^= x >> 27;
x = x.wrapping_mul(0xBF58_4766_71CE_4E5B);
x ^= x >> 31;
```
而 `shared/random/` 的 `SeededRng` 已经包装了 `ChaCha12Rng`——经 NIST 认证的 CSPRNG。
**应该复用 SeededRng 而非自创 PRNG。**

### P2: GameRng 僵尸代码

`GameRng` 已弃用，无业务代码使用，但仍注册为 Resource。应删除。

## 二、执行计划

### Phase 1: 根治架构违规（P0）

**目标**: 将 4 流 DeterministicRng 从 core/replay/ 移到 shared/random/，复用 ChaCha12

#### 1.1 重构 shared/random/
**文件**: `src/shared/random/mod.rs`

将 RngStream、RngSeeds、DeterministicRng 移入 shared/random/，基于 SeededRng(ChaCha12) 实现：

```rust
/// RNG 流枚举
pub enum RngStream { Combat, Drop, AI, World }

/// 4 流确定性 RNG
pub struct DeterministicRng {
    combat: SeededRng,
    drop: SeededRng,
    ai: SeededRng,
    world: SeededRng,
}

impl DeterministicRng {
    pub fn stream(&mut self, stream: RngStream) -> &mut SeededRng;
    pub fn with_seed(seed: u64) -> Self;  // 统一种子
    pub fn set_seeds(&mut self, seeds: RngSeeds);
    pub fn get_seeds(&self) -> RngSeeds;
}
```

关键 Design Decision: 每流独立 ChaCha12 实例，而非自制 hash。
每流独立种子 = 独立 ChaCha12 实例 = 真正的独立随机序列。

#### 1.2 删除 core/replay 中的 DeterministicRng
**文件**: `src/core/capabilities/runtime/replay/foundation/values.rs`

删除 DeterministicRng 结构体及其方法（next_u64、next_f32、gen_bool、gen_range）。
core/replay/ 只保留 ReplayLog、ReplayMode、ReplayRecorder、ReplayPlayer、ReplayValidator、ReplayModeGuard。

#### 1.3 更新 infra/replay/resources.rs
**文件**: `src/infra/replay/resources.rs`

将包装的 CoreDeterministicRng 改为 shared::random::DeterministicRng：
```rust
use crate::shared::random::DeterministicRng;
pub struct DeterministicRng(pub shared::random::DeterministicRng);
```

#### 1.4 修复 Core→Infra 依赖
**文件**: `src/core/domains/combat/integration/replay/recording.rs`

将 `use crate::infra::replay::resources::DeterministicRng` 改为使用 shared 层的 RNG。

### Phase 2: 删除 GameRng 僵尸代码（P2）

#### 2.1 删除 GameRng
**文件**: `src/shared/random/mod.rs`

删除 `GameRng` 结构体、所有方法、Default impl。SeededRng 保留。

#### 2.2 清理 shared_plugin.rs
**文件**: `src/shared/shared_plugin.rs`

删除 `app.insert_resource(GameRng::default())`。

### Phase 3: 文档对齐

#### 3.1 ADR-041
更新 Module Design 章节：RNG 位置从 infra/replay/ → shared/random/

#### 3.2 宪法 §8.11
确认"统一 RNG 服务"与共享层一致。
