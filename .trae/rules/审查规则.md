## 自动化检查清单（与 Fitness Function 对齐）

### Fitness Function 四项检查映射
`tools/check-architecture-budget.sh` 执行以下四项自动化检查，每项对应审查规则中的手工检查项：

| # | 自动化检查 | 对应手工检查项 | 说明 |
|---|-----------|---------------|------|
| 1 | 函数行数 ≤ 50 | Architecture（模块规模） | 超限即违规，CI 拒绝 |
| 2 | 文件行数 ≤ 500 | Architecture（文件规模） | 超限即违规，CI 拒绝 |
| 3 | 导入方向合规 | Architecture（DDD 边界） | 禁止 Core→Infra 等反向依赖 |
| 4 | 无硬编码 Localization Key | Error handling（Localization First） | 用户可见文本必须使用 Key |

### 新增审查项

- **Integration layer used?**
  - 跨 Capability 访问必须通过 `integration/` 层的 Facade，禁止直接引用对方内部类型
  - 确认使用 ReadFacade/WriteFacade 模式，非裸 Query

- **Explain trait considered?**
  - 所有复杂计算（属性公式、伤害计算、状态转换）应考虑实现 `Explain` trait
  - Explain 提供逐步推导输出，便于 Debug 和 Replay 验证
