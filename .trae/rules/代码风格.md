## Naming

Type      -> PascalCase
Trait     -> Verb/Capability
Function  -> snake_case
Constant  -> SCREAMING_SNAKE_CASE

### 测试函数命名
测试函数名用英文 snake_case，使用业务术语如 `damage_applies_armor_reduction`、`buff_removed_on_expiry`：
```rust
#[test]
fn damage_applies_armor_reduction_correctly() { ... }

#[test]
fn buff_removed_on_expiry() { ... }
```

## File

One file = one topic

Avoid:
- systems.rs
- components.rs
- utils.rs

Note: Domain 内部按需拆分，详见 `ai-constitution-complete.md` §3.4

## Function

- Single responsibility
- Early return
- Max nesting <= 3

## Module

- private by default
- expose minimal API

## Rust

- avoid clone()
- avoid unwrap()
- prefer iterator
- prefer Result

## Macro 使用边界
- 🟩 `macro_rules!` 声明式宏：允许用于消除重复模式（如 `define_stat! { Health, Mana }`）
- 🟥 禁止用宏生成业务逻辑
- 🟥 过程宏（proc_macro）需 ADR 审批
- 与 ECS 规则 §3.7 BSN 宏规范保持一致
