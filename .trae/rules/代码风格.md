## Naming

Type      -> PascalCase
Trait     -> Verb/Capability
Function  -> snake_case
Constant  -> SCREAMING_SNAKE_CASE

### 测试函数命名
测试函数名用英文 snake_case，使用业务术语如 `damage_applies_armor_reduction`、`buff_removed_on_expiry`：
```rust
#[test]
fn damage_applies_armor_reduction_correctly() { ... }

#[test]
fn buff_removed_on_expiry() { ... }
```

## Macro 使用边界
- 🟩 `macro_rules!` 声明式宏：允许用于消除重复模式（如 `define_stat! { Health, Mana }`）
- 🟥 禁止用宏生成业务逻辑
- 🟥 过程宏（proc_macro）需 ADR 审批
- 与 ECS 规则 §3.7 BSN 宏规范保持一致

### PhantomData 使用规范
- 🟩 当需要编译期类型区分但运行时不需要数据时使用 PhantomData
- 🟩 与 Newtype 组合实现类型安全 ID（`InstanceId<T>`、`SpecId<D>`）
- 🟩 Reflect 场景下用 `#[reflect(ignore)]` 标注 PhantomData 字段
- 🟩 PhantomData 在编译期被优化掉，零运行时开销

### 字符串所有权策略
- 🟩 静态文本（编译期常量、LocalizationKey、Def的name_key/desc_key）用 `Cow<'static, str>`
- 🟩 动态文本（运行时生成、用户输入）用 `String`
- 🟨 跨层数据传递优先 `Cow<'static, str>`——调用方决定所有权
- 🟥 禁止在热路径 clone String
