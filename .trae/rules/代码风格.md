## Naming

Type      -> PascalCase
Trait     -> Verb/Capability
Function  -> snake_case
Constant  -> SCREAMING_SNAKE_CASE

### 测试函数命名
测试函数名用英文 snake_case，使用业务术语如 `damage_applies_armor_reduction`、`buff_removed_on_expiry`：
```rust
#[test]
fn damage_applies_armor_reduction_correctly() { ... }

#[test]
fn buff_removed_on_expiry() { ... }
```

## Macro 使用边界
- 🟩 `macro_rules!` 声明式宏：允许用于消除重复模式（如 `define_stat! { Health, Mana }`）
- 🟥 禁止用宏生成业务逻辑
- 🟥 过程宏（proc_macro）需 ADR 审批
- 与 ECS 规则 §3.7 BSN 宏规范保持一致

### PhantomData 使用规范
- 🟩 当需要编译期类型区分但运行时不需要数据时使用 PhantomData
- 🟩 与 Newtype 组合实现类型安全 ID（`InstanceId<T>`、`SpecId<D>`）
- 🟩 Reflect 场景下用 `#[reflect(ignore)]` 标注 PhantomData 字段
- 🟩 PhantomData 在编译期被优化掉，零运行时开销

### 字符串所有权策略
- 🟩 静态文本（编译期常量、LocalizationKey、Def的name_key/desc_key）用 `Cow<'static, str>`
- 🟩 动态文本（运行时生成、用户输入）用 `String`
- 🟨 跨层数据传递优先 `Cow<'static, str>`——调用方决定所有权
- 🟥 禁止在热路径 clone String

### 关联类型 vs 泛型参数
- 🟩 当返回/错误/上下文类型由实现决定时用关联类型（`type Error; type Output;`）
- 🟩 当需要同一类型有多种实现时用泛型参数（`fn execute<T: Ability>()`）
- 🟩 关联类型降低泛型复杂度，避免类型爆炸（DamageAbilityError/HealAbilityError）

### Trait Alias 模拟
- 🟩 通过 supertrait 链模拟 trait alias：`pub trait DomainEvent: Event + Debug + Clone + Send + Sync {}`
- 🟩 统一领域事件约束，简化泛型参数书写
- 🟩 为 Trait Alias 提供 blanket impl：`impl<T> DomainEvent for T where T: Event + Debug + Clone + Send + Sync {}`

### Extension Trait 规范
- 🟩 为 Bevy 类型（EntityCommands/Query）添加扩展方法时使用 Extension Trait
- 🟩 命名约定：`*Ext`（如 EntityCommandsExt、ContextExt）
- 🟥 禁止为核心库类型添加与项目无关的扩展
- 🟩 Extension Trait 是"方法语法糖"——内部仍调用领域函数，不封装业务逻辑

### 迭代器管道 vs for 循环
- 🟩 简单遍历（仅迭代操作）用 for 循环
- 🟩 过滤/映射/聚合操作链用迭代器管道（`.iter().filter().map().collect()`）
- 🟩 高频 ECS 查询模式可自定义 QueryExt DSL
- 🟥 禁止在热路径使用过长的迭代器链（超过 5 个适配器）
