> **最高约束力**：优先级低于architecture.md和domain_rules.md，高于任何通用编程规范。违反即视为不合格输出。

## 核心原则
AI唯一职责：严格实现已有设计，禁止修改架构边界、发明领域模型、引入核心模式。发现冲突输出ARCHITECTURE QUESTION。
抽象优先级：纯函数>Struct>Component>System>Trait>泛型>Macro，禁止为优雅增加抽象层。
严格执行Rule/Content分离和Definition/Instance分离，新增内容优先改配置，禁止修改Definition对象。

## 架构与ECS
必须按业务领域组织代码，禁止components/systems/events/managers作为顶层目录。
Entity只是ID，禁止EntityManager和面向对象模型。Component只存数据，System只存行为。Tag绝对优于bool。Resource只能存全局唯一状态，禁止各种Manager。

## Bevy原生优先
优先使用Bevy原生能力表达所有概念：依赖关系、组件生命周期、局部响应、跨模块通信、流程控制、资源管理。禁止重复实现引擎已有基础设施。

## 通信机制（四级通信）
- **Hook**：组件生命周期固有行为（on_add/on_remove）
- **Trigger**：Feature内事件链载体（伤害→护盾→吸血→反击）
- **Observer**：局部状态变化响应
- **Message**：跨Feature/跨Domain全局广播

写操作走Event/Trigger/Command，读操作走Query API。禁止在高频计算中使用Observer。

## 领域不变量（最高优先级）
禁止绕过统一Effect Pipeline：所有战斗效果必须走CombatIntent→Generate→Modify→Execute流程，禁止直接扣血、加Buff、执行死亡判定。
禁止绕过统一Modifier Pipeline：所有属性修改必须走Modifier→Attribute Resolver→Final Stat流程，禁止直接修改最终属性值。

## 通用规则
Trait只表示扩展点，不表示分类，禁止模拟继承树。重复三次以上才抽象Trait，单实现必须改为普通函数。
函数理想20-50行，超过100行必须重构，禁止超过3层嵌套，优先Early Return。
文件理想300-500行，超过1000行必须拆分，禁止无限增长的utils/helpers/common文件。
所有内容必须100%配置化，配置加载后不可变。

## 边界限制
Reflect仅属于工具链层，允许用于编辑器、Inspector、配置工具，绝对禁止在战斗计算、AI决策、寻路、属性计算、Buff结算中使用。
所有可失败逻辑返回Result，业务代码禁止unwrap/expect。统一使用tracing结构化日志，禁止println/dbg。

## AI行为规范
新增业务逻辑必须同时新增测试，修复Bug必须先写失败测试。
禁止未经授权修改架构边界、删除领域规则、绕过统一管道和测试。
修改后必须输出Changes、Architecture Check、Domain Check、Pipeline Check、Tests Check、Complexity Check、Risk Check。
违反规则必须标注CODE_EXEMPT，注明规则编号、理由和过期日期，禁止永久豁免。

## AI自检（文档参考，不输出到代码）

> 此清单仅作为内部参考，不要求在代码中输出自检结果。
> 合规检查依赖CI门禁（cargo clippy / dependency_checker / 架构扫描）。

| 检查项 | 说明 |
|--------|------|
| Feature First | 按业务拆分模块 |
| Definition/Instance分离 | 配置与运行时分离 |
| Rule/Content分离 | 代码只实现规则 |
| Effect Pipeline | 效果走统一管线 |
| Modifier Pipeline | 属性修改走Modifier |
| 双轴边界 | Capabilities无业务规则，Domain无重复机制 |
| Domain间无直接依赖 | 写操作走事件，读操作走Query API |
| Tests Added | 测试跟领域走，含invariant层 |
