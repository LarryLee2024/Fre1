# 日志专项规则（AI提示词精简版）v5.1
> 与《SRPG 项目总宪法 v5.0》配套，优先级等同
> 统一使用 `tracing`，**绝对禁止** `println!` / `dbg!`

---

## 核心定位
日志 = **领域事件履历**，不是技术流水账
- 只记录业务事实，不记录函数进入退出、系统执行流程
- 一个完整业务动作最多输出 **1条INFO** 日志；DEBUG/TRACE可细化，但发布版默认关闭
- 玩家可见的战斗履历用独立 `BattleLogEvent` 体系，不走 tracing
- **INFO级业务日志必须通过领域事件 + 统一Observer输出，业务代码禁止直接写`info!`**

## 分级标准
- **ERROR**：程序异常（Entity丢失、配置缺失、非法状态、存档损坏），必须附带完整复现上下文
- **WARN**：可继续运行的异常（配置不全、数据异常、回退逻辑触发）
- **INFO**：核心业务事件边界（战斗开始/结束、回合切换、单位移动/攻击/死亡、Buff增减、任务完成）
- **DEBUG**：开发调试细节（寻路路径长度、Modifier计算过程），发布版默认关闭
- **TRACE**：极细算法细节（A*节点探索），仅专项调试时临时开启

## 强制格式

### 1. Span 字段与 Event 字段分离（核心规则）
`#[instrument]` 的 `fields()` 和 `info!()` 的参数有严格分工，**不要重复**：

| 位置 | 放什么 | 典型字段 |
|------|--------|----------|
| `#[instrument(fields(...))]` | 不变量 — 本次调用中所有日志共用的固定值 | `code` (LogCode), `event` (事件名) |
| `info!(...)` / `warn!(...)` | 变量 — 只有本次调用才有的动态数据 | `entity`, `spec_id`, `old`, `new`, `amount` |

```rust
// ✅ 正确：span 放不变量，info! 放变量
#[tracing::instrument(skip_all, fields(code = ?LogCode::PRG002, event = "level_up"))]
fn on_level_up(trigger: On<LevelUp>) {
    let e = trigger.event();
    info!(
        entity = ?e.entity,      // 变量 — 每次调用不同
        old = e.old_level,
        new = e.new_level,
        "level_up"
    );
}

// ❌ 错误：code/event 重复出现在 info!() 中（已包含在 span 里）
info!(
    code = ?LogCode::PRG002,     // ❌ 重复 — span 已有
    event = "level_up",          // ❌ 重复 — span 已有
    entity = ?e.entity,
    ...
);
```

### 2. 字段语言规范
- **`event` 字段值**：必须用英文（`"level_up"`、`"battle_started"`），因为结构化日志是机器消费的，后续会被 LogCode 取代
- **`message`（日志的字符串消息）**：可以用中文或英文，这是给人看的
- **`LogCode::description()`**：可以用中文（例如 `"角色升级"`），AI 搜索和人类阅读皆可

```rust
// ✅ 正确：event 英文，description 中文
#[tracing::instrument(skip_all, fields(code = ?LogCode::PRG002, event = "level_up"))]
fn on_level_up(trigger: On<LevelUp>) {
    // ... info!("角色升级");  // message 可以是中文
}
```

### 3. LogCode 是事件唯一标识
- 所有 INFO 日志必须携带 `code = ?LogCode::XXX` 字段
- `event` 字段是 LogCode 的**可选人类别名**，不是第二个标识
- 后续计划通过 `telemetry::emit(LogCode::XXX)` 统一日志 + 度量 + trace，届时 `event` 字段从 LogCode 自动派生

### 4. 结构化字段必须低基数
- 所有结构化字段必须使用 ID（`entity_id`、`spec_id`、`item_id`），**禁止使用自然语言描述**
- `context_desc` 等自由文本字段高基数，长期存在会破坏日志聚合分析，应改用 `context_id` + `LocalizationKey`
- 示例对比：
  ```rust
  // ❌ 高基数：自然语言文本
  info!(context_desc = "caster's level 3 fireball", ...);

  // ✅ 低基数：使用 ID
  info!(spec_id = ?spec_id, entity = ?entity, ...);
  ```

### 5. `target` 命名规范
日志 `target` 必须遵循 `domain.module.submodule` 层级格式，支持按精确粒度过滤：

| 模块 | target |
|------|--------|
| 战斗核心 | `domain.combat` |
| 技能激活 | `domain.ability.activation` |
| 效果应用 | `domain.effect` |
| 回合流转 | `domain.tactical.turn` |
| 存档 | `infra.save` |
| 内容加载 | `infra.content` |

`#[instrument]` 用法：
```rust
#[tracing::instrument(skip_all, target = "domain.ability.activation", fields(...))]
```

## Observer 模板（推荐模式）

```rust
use bevy::prelude::*;
use crate::core::domains::progression::events::LevelUp;
use crate::infra::logging::metrics;
use crate::shared::diagnostics::LogCode;

// ❗ span：放不变量（code + event）
#[tracing::instrument(skip_all, target = "domain.progression", fields(
    code = ?LogCode::PRG002,
    event = "level_up",
))]
pub(crate) fn on_level_up(trigger: On<LevelUp>) {
    // 1. 度量计数
    metrics::record(LogCode::PRG002);

    // 2. 取事件数据
    let event = trigger.event();

    // 3. 输出 — 只放变量字段，不放 span 已覆盖的不变量
    info!(
        target = "domain.progression",        // 需要显式传递（tracing instrument 的 target 不传给 info!）
        entity = ?event.entity,
        old = event.old_level,
        new = event.new_level,
        "角色升级",
    );
}
```

> **注意**：`#[instrument]` 的 `target` 只在 span 层生效，`info!()` 内部的日志仍需要使用自己的 `target` 参数。后续可以通过 `telemetry::emit` 简化此模式。

## 例外范围
以下场景可直接调用tracing宏，无需走事件链路：
- ERROR / WARN 级别异常日志
- DEBUG / TRACE 级别调试日志
- 基础设施、工具层代码
- 测试代码

## 绝对禁令
❌ 记录系统运行、函数进出等技术流水账
❌ 每帧执行的系统输出 INFO/DEBUG 日志（仅允许 ERROR）
❌ 循环/迭代器内部输出 INFO 日志
❌ 堆砌日志替代 Inspector/Replay 等调试工具
❌ 绕过 Modifier/Effect 管道时打日志掩盖违规
❌ 业务代码直接手写 `info!` 输出核心业务事件
❌ `event` 字段用中文（机器可读字段必须英文）
❌ `context_desc` 等自然语言文本作为结构化字段（改用 ID）
❌ 在 `info!()` 中重复 span 已覆盖的 `code`/`event` 字段

## 生成后自检
✅ 记录的是领域事件，不是技术流程
✅ 日志级别使用正确
✅ 带 `code = ?LogCode::XXX` 字段
✅ `event` 字段值是英文（机器可读）
✅ 结构化字段使用 ID，没有自然语言文本
✅ Span 字段与 info! 字段无重复
✅ `target` 使用 `domain.xxx` 层级格式
✅ INFO级日志走事件链路，未直接手写info!
✅ 无违禁日志
