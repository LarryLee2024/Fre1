# Bevy SRPG测试宪法v3.1 纯提示词版
> **最高约束力**：优先级高于任何通用测试实践，与AI开发宪法v1.1配套执行。违反即视为不合格输出。

## 核心规则优先级（绝对不可违反）
domain_rules.md > architecture.md > test_spec.md

## 测试哲学
测试验证**行为**，不验证实现；关注**What**，不关注How。

## 测试分类与优先级
1. **Replay测试（最高）**：所有战斗Bug必须转化为此类
2. 回归测试：所有已修复Bug必须对应，永久保留
3. 单元测试（70%）：不启动App，验证单一纯规则
4. 集成测试（20%）：最小化Bevy App，验证多模块协作
5. E2E测试（2%）：仅验证核心主流程，数量最少

## 强制要求
- 所有测试必须确定性，随机数固定Seed=42
- 测试用例必须遵循：Test ID → Title → Given → When → Then → Assertions
- 必须使用标准测试单位：Unit_001(战士100/30/10/10)、Unit_002(法师80/40/5/12)、Unit_003(坦克150/20/20/5)
- 覆盖目标：100%核心领域规则，追求规则覆盖而非代码覆盖
- 发现Bug必须先写失败测试，再修复代码

## AI决策规则（测试失败时严格执行）
1. 先查domain_rules.md
2. 再查architecture.md
3. 最后检查测试本身
- 测试违反领域规则→改测试；代码违反领域规则→改代码；都符合→更新领域规则
- 禁止为过测试改业务逻辑，禁止为过代码改测试

## AI约束与自检
禁止测试私有实现、内部缓存、System顺序、组件布局。生成测试后必须确认：
✅ 测行为不测实现 ✅ 符合领域规则 ✅ 确定性 ✅ 使用标准数据 ✅ 无越界测试

## 最终原则
Domain Rules First. Tests Second. Implementation Third.
